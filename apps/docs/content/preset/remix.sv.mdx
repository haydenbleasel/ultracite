Remix-konfigurationen har filnamnskonventioner för ruttfiler. Denna konfiguration är också kompatibel med TanStack Router, React Router och andra filbaserade routing-system.

## Installation

Lägg till Remix-konfigurationen i din `biome.jsonc`:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/remix"]
}
```

## Översikt

Denna konfiguration ger särskild hantering för ruttfiler som använder filsystembaserade routingkonventioner:

- **Namn för ruttfiler**: Inaktiverar strikta filnamnskonventioner för ruttfiler
- **Flexibla mönster**: Stöder Remixes rutt-namngivningsmönster (punkter, dollar, understreck)
- **Ramverksoberoende**: Fungerar med Remix, TanStack Router, React Router och liknande system

## Överstyrningar av filmönster

Konfigurationen tillämpar överstyrningar på ruttfiler i dessa mönster:

- `**/routes/**/*.{tsx,ts}` - Remix v1 routes
- `**/app/routes/**/*.{tsx,ts}` - Remix v2 routes

### Inaktiverade regler för ruttfiler

| Regel | Inställning | Beskrivning |
|------|---------|-------------|
| `useFilenamingConvention` | `off` | Inaktivera tvång att använda kebab-case för filnamn för ruttfiler. Tillåter speciala rutt-namngivningsmönster. |

## Varför den här konfigurationen finns

Filbaserade routing-system använder specialtecken i filnamn för att representera routing-mönster:

- `.` (punkt) - Representerar sökvägssegment (t.ex. `posts.new.tsx` → `/posts/new`)
- `$` (dollar) - Representerar dynamiska segment (t.ex. `posts.$id.tsx` → `/posts/:id`)
- `_` (understreck) - Representerar layoutrutter eller väglösa rutter

Dessa namngivningskonventioner kolliderar med Ultracites standardkrav på kebab-case för filnamn, så denna konfiguration inaktiverar den regeln specifikt för ruttfiler.
