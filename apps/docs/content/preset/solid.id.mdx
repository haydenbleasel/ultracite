Konfigurasi Solid memiliki aturan linting khusus Solid.js untuk reaktivitas, pola komponen, dan penanganan JSX.

## Instalasi

Tambahkan konfigurasi Solid ke `biome.jsonc` Anda:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/solid"]
}
```

## Ikhtisar

Konfigurasi ini menambahkan aturan yang spesifik untuk pengembangan Solid.js:

- **Aturan Reaktivitas**: Menegakkan penggunaan yang tepat dari primitif reaktif Solid
- **Pola Komponen**: Pastikan komponen Solid mengikuti praktik terbaik
- **Penanganan Props**: Mencegah destrukturisasi props (merusak reaktivitas)
- **Props Khusus React**: Melarang props khusus React di komponen Solid

## Aturan Khusus Solid

### Kebenaran

| Aturan | Pengaturan | Deskripsi |
|------|---------|-------------|
| `noSolidDestructuredProps` | `error` | Mencegah destrukturisasi props di komponen Solid. Destrukturisasi merusak sistem reaktivitas Solid. Akses props secara langsung sebagai gantinya. |

### Kinerja

| Aturan | Pengaturan | Deskripsi |
|------|---------|-------------|
| `useSolidForComponent` | `error` | Menegakkan penggunaan komponen `<For>` untuk mengiterasi array di Solid. Komponen `<For>` dioptimalkan untuk reaktivitas Solid. |

### Mencurigakan

| Aturan | Pengaturan | Deskripsi |
|------|---------|-------------|
| `noReactSpecificProps` | `error` | Melarang props khusus React seperti `className` dan `htmlFor` di Solid. Gunakan `class` dan `for` sebagai gantinya. |
