La configuración de Qwik incluye reglas de linting específicas para la serialización, las tareas visibles, el alcance léxico y el uso de métodos.

## Instalación

Agregue la configuración de Qwik a su `biome.jsonc`:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/qwik"]
}
```

## Descripción general

Esta configuración añade reglas específicas para el desarrollo con Qwik:

- **Serialización**: Hacer cumplir el uso apropiado del modelo de reanudabilidad de Qwik
- **Tareas visibles**: Evitar el uso inapropiado de `useVisibleTask$`
- **Props específicos de React**: Prohibir props específicos de React en componentes de Qwik

## Reglas específicas de Qwik

### Corrección

| Regla | Configuración | Descripción |
|------|---------------|-------------|
| `noQwikUseVisibleTask` | `error` | Evitar el uso de `useVisibleTask$`. Este hook ejecuta el código de forma inmediata en el cliente, lo que perjudica la reanudabilidad de Qwik. Utilice `useTask$` o `useResource$` en su lugar cuando sea posible. |
| `useQwikClasslist` | `error` | Imponer el uso de la sintaxis de objeto `class` en lugar de la concatenación de cadenas para clases dinámicas en Qwik. |

### Sospechosas

| Regla | Configuración | Descripción |
|------|---------------|-------------|
| `noReactSpecificProps` | `error` | Prohibir props específicos de React como `className` y `htmlFor` en Qwik. Utilice `class` y `for` en su lugar. |
