React yapılandırması, JSX, hook'lar ve bileşen desenleri için React'e özgü lint kuralları içerir.

## Kurulum

React yapılandırmasını `biome.jsonc` dosyanıza ekleyin:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/react"]
}
```

## Genel Bakış

Bu yapılandırma React geliştirme için şu kuralları ekler:

- **JSX Kuralları**: Doğru JSX sözdizimi ve desenlerini zorunlu kılar
- **Hook Kuralları**: React hook'larının doğru kullanıldığından emin olur
- **Bileşen Kuralları**: React bileşenleri için en iyi uygulamaları zorunlu kılar
- **Children İşleme**: children prop kullanımının hatalı olmasını önler

## React'e Özgü Kurallar

### Doğruluk

| Rule | Setting | Description |
|------|---------|-------------|
| `noChildrenProp` | `error` | children'ın props olarak geçirilmesini engelleyin. children, açılış ve kapanış etiketleri arasında yer almalıdır. |
| `noNestedComponentDefinitions` | `error` | Bir bileşenin içinde başka bileşen tanımlamalarını engelleyin. Bu gereksiz yeniden renderlara neden olur. |
| `noReactPropAssignments` | `error` | React bileşenlerinde props'un yeniden atanmasını engelleyin. Props değiştirilemez olarak ele alınmalıdır. |
| `noRenderReturnValue` | `error` | `ReactDOM.render()`'ın dönüş değerinin kullanılmasını engelleyin. |
| `noVoidElementsWithChildren` | `error` | Void elementlerin (ör. `<img>`, `<br>`) children sahibi olmasını engelleyin. |
| `useExhaustiveDependencies` | `error` | React hook'larında (useEffect, useCallback, useMemo) tüm bağımlılıkların doğru şekilde belirtilmesini zorunlu kılar. |
| `useHookAtTopLevel` | `error` | Tüm React hook'larının bileşen fonksiyonlarının üst düzeyinden çağrılmasını zorunlu kılar; döngüler, koşullar veya iç içe fonksiyonlar içinde olmamalıdır. |
| `useJsxKeyInIterable` | `error` | İterasyonlardaki öğelerin React'in reconciliation'ı için bir `key` prop'una sahip olmasını zorunlu kılar. |

### Stil

| Rule | Setting | Description |
|------|---------|-------------|
| `useReactFunctionComponents` | `error` | React'te sınıf bileşenleri yerine fonksiyon bileşenlerinin kullanılmasını zorunlu kılar. |

### Şüpheli

| Rule | Setting | Description |
|------|---------|-------------|
| `noArrayIndexKey` | `error` | Dizi indekslerinin key olarak kullanılmasını engelleyin. Dizi indeksleri kararlı tanımlayıcılar değildir ve bileşen durumunda sorunlara yol açabilir. |
| `noDuplicateJsxProps` | `error` | JSX'te yinelenen özelliklerin kullanılmasını engelleyin. |
| `noSuspiciousSemicolonInJsx` | `error` | JSX öğelerinin anlamsal yapısını değiştiren noktalı virgülleri engelleyin. |
| `noReactSpecificProps` | `off` | `className` ve `htmlFor` gibi React'e özgü prop'lara izin verin. |
