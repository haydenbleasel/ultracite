La configuración de Astro contiene ajustes de reglas de lint específicos de Astro para manejar la estructura única de los componentes de Astro, particularmente en lo relativo a los scripts de frontmatter y la sintaxis de componentes.

## Instalación

Agrega la configuración de Astro a tu `biome.jsonc`:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/astro"]
}
```

## Resumen

Esta configuración ajusta reglas para el desarrollo con Astro:

- **Variables/Importaciones no usadas**: Permite variables e importaciones no usadas en archivos Astro debido a las restricciones del frontmatter donde las variables pueden ser usadas solo en la plantilla
- **Declaraciones de variables**: Relaja la exigencia de `const` ya que el frontmatter de Astro puede requerir variables reasignables
- **Tipos de importación**: Desactiva los requisitos explícitos de tipo de importación para una mejor compatibilidad
- **Props específicos de React**: Señala props específicos de React como `className` ya que Astro utiliza atributos HTML estándar

## Reglas específicas de Astro

### Corrección

| Regla | Ajuste | Descripción |
|------|---------|-------------|
| `noUnusedVariables` | `off` | Permitir variables no usadas en archivos Astro. Las variables declaradas en el frontmatter pueden usarse solo en la sección de plantilla. |
| `noUnusedImports` | `off` | Permitir importaciones no usadas en archivos Astro. Las importaciones en el frontmatter pueden usarse solo en la sección de plantilla. |

### Estilo

| Regla | Ajuste | Descripción |
|------|---------|-------------|
| `useConst` | `off` | Permitir declaraciones `let` y `var`. El frontmatter de componentes Astro puede requerir variables reasignables. |
| `useImportType` | `off` | Deshabilita los requisitos explícitos de tipo de importación para una mejor compatibilidad con el sistema de compilación de Astro. |

### Sospechosas

| Regla | Ajuste | Descripción |
|------|---------|-------------|
| `noReactSpecificProps` | `error` | Prohíbe props específicos de React como `className` y `htmlFor` en Astro. Usa `class` y `for` en su lugar. |
