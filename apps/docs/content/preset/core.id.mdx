Konfigurasi inti Ultracite menyediakan serangkaian aturan linting dan pengaturan pemformatan yang komprehensif dan tidak bergantung pada kerangka kerja. Ini adalah dasar yang diperluas oleh semua konfigurasi spesifik kerangka kerja lainnya.

## Instalasi

Konfigurasi inti otomatis disertakan ketika Anda menjalankan:

```bash
npx ultracite init
```

Atau tambahkan secara manual ke `biome.jsonc` Anda:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core"]
}
```

## Gambaran Umum

Konfigurasi inti mencakup ratusan aturan linting ketat yang meliputi:

- **Aksesibilitas (a11y)**: Menegakkan atribut ARIA, HTML semantik, dan navigasi keyboard
- **Kompleksitas**: Mengurangi kompleksitas kognitif dan menegakkan praktik terbaik
- **Ketepatan**: Mencegah kesalahan umum dan menegakkan keselamatan tipe
- **Performa**: Mengoptimalkan kode untuk performa runtime yang lebih baik
- **Keamanan**: Mencegah kerentanan keamanan
- **Gaya**: Menegakkan gaya kode yang konsisten
- **Mencurigakan**: Menangkap pola mencurigakan yang mungkin menunjukkan bug

## Pengaturan Formatter

- **Indentasi**: 2 spasi
- **Lebar Baris**: 80 karakter
- **Akhir Baris**: LF (gaya Unix)
- **Titik koma**: Selalu diperlukan
- **Tanda kutip**: Tanda kutip ganda untuk JavaScript/TypeScript dan JSON
- **Koma akhir**: Gaya ES5 (array/objek multi-baris)
- **Kurung untuk Arrow**: Selalu sertakan kurung

## Fitur Utama

### Keketatan TypeScript

Konfigurasi inti mengaktifkan pemeriksaan ketat TypeScript:

- Menganjurkan untuk tidak menggunakan `any` (`noExplicitAny: error`)
- Memerlukan penanganan `null/undefined`
- Memilih tipe eksplisit dalam situasi tertentu
- Menegakkan keselamatan tipe di seluruh basis kode

### Padanan yang Direkomendasikan ESLint

Aturan praktik terbaik umum disertakan:

- Tidak ada variabel tak terpakai (`noUnusedVariables: error`)
- Tidak ada impor tak terpakai (`noUnusedImports: error`)
- Tidak ada eval eksplisit (`noGlobalEval: error`)
- Tidak ada prototype pollution (`noPrototypeBuiltins: error`)

### Aksesibilitas Secara Default

Aturan yang setara dengan standar aksesibilitas:

- Atribut ARIA harus valid
- Elemen HTML semantik lebih disarankan
- Dukungan navigasi keyboard diperlukan
- Teks alt untuk gambar diperlukan

### Dukungan Node.js

Penanganan khusus lingkungan:

- Mengenali global Node.js (`module`, `process`)
- Menegakkan protokol `node:` untuk modul bawaan
- Mendukung baik CommonJS maupun ESM

## Catatan

Beberapa aturan dinonaktifkan karena terlalu lambat dijalankan dengan Scanner. Seperti yang dicatat dalam [Dokumentasi Biome](https://biomejs.dev/guides/investigate-slowness/#first-things-first), aturan proyek yang diaktifkan diketahui dapat menyebabkan overhead performa sebagai pertukaran untuk analisis yang lebih mendalam.

Dalam pengujian kami, kami menemukan aturan-aturan ini dapat membuat perbedaan antara 300ms dan 13s untuk linting.
