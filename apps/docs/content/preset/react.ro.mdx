Configurația React conține reguli de lint specifice pentru JSX, hooks și tipare de componente.

## Instalare

Adaugă configurația React în fișierul tău `biome.jsonc`:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/react"]
}
```

## Prezentare generală

Această configurație adaugă reguli specifice dezvoltării cu React:

- **Reguli JSX**: Impun sintaxă și tipare JSX corecte
- **Reguli pentru hooks**: Asigură utilizarea corectă a hook-urilor React
- **Reguli pentru componente**: Impun cele mai bune practici pentru componentele React
- **Gestionarea copiilor (Children)**: Previne utilizarea incorectă a prop-ului children

## Reguli specifice React

### Corectitudine

| Regulă | Setare | Descriere |
|------|---------|-------------|
| `noChildrenProp` | `error` | Împiedică transmiterea children ca prop. Children ar trebui să fie plasate între tag-urile de deschidere și de închidere. |
| `noNestedComponentDefinitions` | `error` | Împiedică definirea componentelor în interiorul altor componente. Aceasta cauzează re-renderizări inutile. |
| `noReactPropAssignments` | `error` | Împiedică reatribuirile prop-urilor în componentele React. Prop-urile trebuie tratate ca imutabile. |
| `noRenderReturnValue` | `error` | Împiedică utilizarea valorii returnate de `ReactDOM.render()`. |
| `noVoidElementsWithChildren` | `error` | Împiedică elementele void (de ex. `<img>`, `<br>`) să aibă children. |
| `useExhaustiveDependencies` | `error` | Forțează ca toate dependențele să fie specificate corect în hook-urile React (useEffect, useCallback, useMemo). |
| `useHookAtTopLevel` | `error` | Impune ca toate hook-urile React să fie apelate la nivelul superior al funcțiilor componentă, nu în interiorul buclelor, condițiilor sau funcțiilor imbricate. |
| `useJsxKeyInIterable` | `error` | Impune ca elementele din iterable să aibă prop-ul `key` pentru reconcilierea React. |

### Stil

| Regulă | Setare | Descriere |
|------|---------|-------------|
| `useReactFunctionComponents` | `error` | Impune utilizarea componentelor funcție în locul componentelor clasă în React. |

### Suspecte

| Regulă | Setare | Descriere |
|------|---------|-------------|
| `noArrayIndexKey` | `error` | Împiedică folosirea indicilor din array ca și chei. Indicii unui array nu sunt identificatori stabili și pot cauza probleme cu starea componentelor. |
| `noDuplicateJsxProps` | `error` | Împiedică proprietăți duplicate în JSX. |
| `noSuspiciousSemicolonInJsx` | `error` | Împiedică punct și virgulă care schimbă semantica elementelor JSX. |
| `noReactSpecificProps` | `off` | Permite prop-uri specifice React cum ar fi `className` și `htmlFor`. |
