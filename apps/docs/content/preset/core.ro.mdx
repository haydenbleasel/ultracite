Configurația principală Ultracite oferă un set cuprinzător de reguli de linting și setări de formatare care nu depind de un framework. Aceasta este fundația pe care se bazează toate celelalte configurații specifice framework-ului.

## Instalare

Configurația de bază este inclusă automat când rulezi:

```bash
npx ultracite init
```

Sau adaug-o manual în `biome.jsonc`:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core"]
}
```

## Prezentare generală

Configurația de bază include sute de reguli stricte de linting care acoperă:

- **Accesibilitate (a11y)**: Impune atribute ARIA valide, HTML semantic și navigare de la tastatură
- **Complexitate**: Reduce complexitatea cognitivă și impune bune practici
- **Corectitudine**: Previne erorile comune și impune siguranța tipurilor
- **Performanță**: Optimizează codul pentru o performanță mai bună la rulare
- **Securitate**: Previne vulnerabilități de securitate
- **Stil**: Impune un stil de cod consistent
- **Suspicios**: Depistează modele suspecte care ar putea indica bug-uri

## Setări formatare

- **Indentare**: 2 spații
- **Lățime linie**: 80 de caractere
- **Sfârșit de linie**: LF (stil Unix)
- **Punct și virgulă**: Întotdeauna obligatorii
- **Ghilimele**: Ghilimele duble pentru JavaScript/TypeScript și JSON
- **Virgule finale**: Stil ES5 (array/obiecte multi-linie)
- **Paranteze pentru arrow**: Include întotdeauna parantezele

## Caracteristici cheie

### Rigorozitate TypeScript

Configurația de bază activează verificările stricte TypeScript:

- Descurajează utilizarea lui `any` (`noExplicitAny: error`)
- Solicită tratarea valorilor `null/undefined`
- Preferă tipuri explicite în anumite situații
- Impune siguranța tipurilor în întregul codebase

### Echivalente recomandate ESLint

Reguli comune de bune practici sunt incluse:

- Fără variabile neutilizate (`noUnusedVariables: error`)
- Fără importuri neutilizate (`noUnusedImports: error`)
- Fără eval explicit (`noGlobalEval: error`)
- Fără poluare a prototipurilor (`noPrototypeBuiltins: error`)

### Accesibilitate implicită

Reguli echivalente standardelor de accesibilitate:

- Atributele ARIA trebuie să fie valide
- Se preferă elementele HTML semantice
- Suport pentru navigare de la tastatură necesar
- Text alternativ pentru imagini obligatoriu

### Suport Node.js

Tratament specific pentru mediu:

- Recunoaște globale Node.js (`module`, `process`)
- Impune protocolul `node:` pentru modulele încorporate
- Suportă atât CommonJS, cât și ESM

## Note

Anumite reguli sunt dezactivate deoarece sunt prea lente pentru rulare cu Scanner-ul. Așa cum este menționat în documentația [Biome docs](https://biomejs.dev/guides/investigate-slowness/#first-things-first), regulile activate la nivel de proiect sunt cunoscute că pot cauza overhead de performanță în schimbul unei analize mai avansate.

În testările noastre, am constatat că aceste reguli pot face diferența între 300ms și 13s de lint.
