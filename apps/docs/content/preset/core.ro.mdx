Configurația de bază Ultracite oferă un set cuprinzător de reguli de linting independente de framework și setări de formatare. Aceasta este fundația pe care se extind toate celelalte configurații specifice framework-urilor.

## Instalare

Configurația de bază este inclusă automat când rulezi:

```bash
npx ultracite init
```

Sau o poți adăuga manual în `biome.jsonc`:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core"]
}
```

## Prezentare generală

Configurația de bază include sute de reguli stricte de linting care acoperă:

- **Accesibilitate (a11y)**: Impune atribute ARIA valide, HTML semantic și navigare prin tastatură
- **Complexitate**: Reduce complexitatea cognitivă și impune bune practici
- **Corectitudine**: Previne erori comune și impune siguranța tipurilor
- **Performanță**: Optimizează codul pentru performanță mai bună la rulare
- **Securitate**: Previne vulnerabilități de securitate
- **Stil**: Impune un stil de cod consecvent
- **Suspecte**: Detectează tipare suspecte care pot indica bug-uri

## Setări de formatare

- **Indentare**: 2 spații
- **Lățime linie**: 80 de caractere
- **Sfârșit de linie**: LF (stil Unix)
- **Punct și virgulă**: Întotdeauna obligatorii
- **Ghilimele**: Ghilimele duble pentru JavaScript/TypeScript și JSON
- **Virgule finale**: Stil ES5 (pentru array-uri/obiecte multi-linie)
- **Paranteze pentru arrow**: Include întotdeauna paranteze

## Caracteristici cheie

### Verificări stricte pentru TypeScript

Configurația de bază activează verificările stricte ale TypeScript:

- Descurajează utilizarea lui `any` (`noExplicitAny: error`)
- Cere tratarea valorilor `null/undefined`
- Preferă tipuri explicite în anumite situații
- Impune siguranța tipurilor în întregul cod

### Echivalente recomandate ESLint

Reguli comune de bune practici incluse:

- Nici variabile neutilizate (`noUnusedVariables: error`)
- Nici importuri neutilizate (`noUnusedImports: error`)
- Nici eval explicit (`noGlobalEval: error`)
- Nici prototype pollution (`noPrototypeBuiltins: error`)

### Accesibilitate implicită

Reguli echivalente standardelor de accesibilitate:

- Atributele ARIA trebuie să fie valide
- Se preferă elemente HTML semantice
- Suport pentru navigare prin tastatură este necesar
- Text alternativ pentru imagini este obligatoriu

### Suport Node.js

Tratament specific mediului:

- Recunoaște globale Node.js (`module`, `process`)
- Impune protocolul `node:` pentru modulele încorporate
- Suportă atât CommonJS, cât și ESM

## Note

Anumite reguli sunt dezactivate deoarece sunt prea lente pentru a rula cu Scanner-ul. Așa cum este menționat în [documentația Biome](https://biomejs.dev/guides/investigate-slowness/#first-things-first), regulile activabile în proiect sunt cunoscute pentru a cauza overhead de performanță în schimbul unei analize mai avansate.

În testele noastre, am găsit că aceste reguli pot face diferența între 300ms și 13s de lint.
