Configurația de bază Ultracite oferă un set cuprinzător de reguli stricte de linting și setări de formatare care sunt agnostice față de framework. Aceasta este fundația pe care se extind toate celelalte configurații specifice framework-urilor.

## Instalare

Configurația de bază este inclusă automat când rulezi:

```bash
npx ultracite init
```

Sau adaug-o manual în `biome.jsonc`:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core"]
}
```

## Prezentare generală

Configurația de bază include sute de reguli stricte de linting care acoperă:

- **Accesibilitate (a11y)**: Impune atribute ARIA valide, HTML semantic și navigare prin tastatură
- **Complexitate**: Reduce complexitatea cognitivă și impune bune practici
- **Corectitudine**: Previne erorile comune și impune siguranța tipurilor
- **Performanță**: Optimizează codul pentru performanță la rulare
- **Securitate**: Previne vulnerabilitățile de securitate
- **Stil**: Impune un stil de cod consistent
- **Suspecte**: Prinde modele suspecte care ar putea indica bug-uri

## Setări pentru formatare

- **Indentare**: 2 spații
- **Lățime linie**: 80 de caractere
- **Sfârșit de linie**: LF (stil Unix)
- **Punct și virgulă**: Solicită întotdeauna
- **Ghilimele**: Ghilimele duble pentru JavaScript/TypeScript și JSON
- **Virgule finale**: Stil ES5 (array-uri/obiecte pe mai multe linii)
- **Paranteze pentru funcțiile arrow**: Include întotdeauna parantezele

## Caracteristici cheie

### Strictitatea TypeScript

Configurația de bază activează verificările stricte ale TypeScript:

- Descurajează folosirea lui `any` (`noExplicitAny: error`)
- Solicită tratarea valorilor `null/undefined`
- Preferă tipuri explicite în anumite situații
- Impune siguranța tipurilor în întregul cod

### Echivalente recomandate ESLint

Reguli comune de bune practici sunt incluse:

- Fără variabile nefolosite (`noUnusedVariables: error`)
- Fără importuri nefolosite (`noUnusedImports: error`)
- Fără eval explicit (`noGlobalEval: error`)
- Fără pollutare de prototip (`noPrototypeBuiltins: error`)

### Accesibilitate implicită

Reguli echivalente cu standardele de accesibilitate:

- Atributele ARIA trebuie să fie valide
- Se preferă elementele HTML semantice
- Suport pentru navigare prin tastatură necesar
- Text alternativ (alt) pentru imagini necesar

### Suport Node.js

Tratament specific mediului:

- Recunoaște globale Node.js (`module`, `process`)
- Impune protocolul `node:` pentru modulele builtin
- Suportă atât CommonJS cât și ESM

## Note

Anumite reguli sunt dezactivate pentru că sunt prea lente pentru a rula cu Scanner-ul. Așa cum este menționat în [documentația Biome](https://biomejs.dev/guides/investigate-slowness/#first-things-first), regulile activate la nivel de proiect sunt cunoscute că pot cauza un overhead de performanță în schimbul unei analize mai avansate.

În testele noastre, am constatat că aceste reguli pot face diferența între un lint de 300ms și unul de 13s.
