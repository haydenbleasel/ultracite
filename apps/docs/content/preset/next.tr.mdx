Next.js yapılandırması, görüntü optimizasyonu, belge yapısı ve App Router kalıpları için Next.js'e özgü lint kuralları içerir.

## Kurulum

Next.js yapılandırmasını `biome.jsonc` dosyanıza ekleyin:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/next"]
}
```

## Genel Bakış

Bu yapılandırma Next.js geliştirmeye özgü kurallar ekler:

- **Görüntü Optimizasyonu**: Next.js Image bileşeninin kullanılmasını zorunlu kılar
- **Belge Yapısı**: Next.js özel bileşenlerinin uygunsuz kullanımını önler
- **Yapılandırma Dosyaları**: `next.config.*` dosyaları için özel işlem

## Next.js'e Özgü Kurallar

### Performans

| Kural | Ayar | Açıklama |
|------|---------|-------------|
| `noImgElement` | `error` | HTML `<img>` öğesinin kullanımına izin vermez. Otomatik görüntü optimizasyonu için bunun yerine Next.js `<Image>` bileşenini kullanın. |

### Stil

| Kural | Ayar | Açıklama |
|------|---------|-------------|
| `noHeadElement` | `error` | HTML `<head>` öğesinin kullanımına izin vermez. Bunun yerine Next.js `next/head` veya App Router metadata API'sini kullanın. |

### Şüpheli

| Kural | Ayar | Açıklama |
|------|---------|-------------|
| `noDocumentImportInPage` | `error` | Sayfa dosyalarında `next/document` içe aktarımını engeller. `_document.tsx` yalnızca `pages/_document.tsx` içinde bulunmalıdır. |
| `noHeadImportInDocument` | `error` | `_document.tsx` içinde `next/head` içe aktarımını engeller. Bunun yerine `next/document`'ten `<Head>` kullanın. |

## Yapılandırma Dosyası Geçersiz Kılmaları

Next.js önayarları, yapılandırma dosyaları ve App Router bileşenleri için özel işlem içerir:

### Next.js Yapılandırma Dosyaları

```jsonc
{
  "overrides": [
    {
      "includes": ["next.config.*"],
      "linter": {
        "rules": {
          "suspicious": {
            "useAwait": "off"
          }
        }
      }
    }
  ]
}
```

Bu, genellikle lint kurallarını geçemeyecek Next.js yapılandırma kalıplarına izin verir.

### App Router Sayfaları ve Düzenleri

```jsonc
{
  "overrides": [
    {
      "includes": ["**/page.{ts,tsx,js,jsx}", "**/layout.{ts,tsx,js,jsx}"],
      "linter": {
        "rules": {
          "suspicious": {
            "useAwait": "off"
          }
        }
      }
    }
  ]
}
```

App Router sayfa ve düzen bileşenlerinin `await` gerektirmeden async olmalarına izin verilir; Next.js bunları sunucu tarafı render ve veri getirme için özel olarak işler.
