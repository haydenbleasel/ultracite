Die Qwik-Konfiguration enthält Qwik-spezifische Linting-Regeln für Serialisierung, sichtbare Tasks, lexikalischen Gültigkeitsbereich und Methodenverwendung.

## Installation

Fügen Sie die Qwik-Konfiguration zu Ihrer `biome.jsonc` hinzu:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/qwik"]
}
```

## Übersicht

Diese Konfiguration fügt Qwik-spezifische Regeln für die Entwicklung hinzu:

- **Serialisierung**: Erzwingt die richtige Verwendung von Qwiks Resumability-Modell
- **Sichtbare Tasks**: Verhindert die ungeeignete Verwendung von `useVisibleTask$`
- **React-spezifische Props**: Verhindert die Verwendung von React-spezifischen Props in Qwik-Komponenten

## Qwik-spezifische Regeln

### Korrektheit

| Regel | Einstellung | Beschreibung |
|------|-------------|-------------|
| `noQwikUseVisibleTask` | `error` | Verhindert die Verwendung von `useVisibleTask$`. Dieser Hook führt Code sofort auf dem Client aus und untergräbt damit Qwiks Resumability. Verwenden Sie stattdessen, wenn möglich, `useTask$` oder `useResource$`. |
| `useQwikClasslist` | `error` | Erzwingt die Verwendung der `class`-Objektsyntax statt String-Konkatenation für dynamische Klassen in Qwik. |

### Verdächtig

| Regel | Einstellung | Beschreibung |
|------|-------------|-------------|
| `noReactSpecificProps` | `error` | Verbietet React-spezifische Props wie `className` und `htmlFor` in Qwik. Verwenden Sie stattdessen `class` und `for`. |
