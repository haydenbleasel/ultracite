La configuración de Vue incluye reglas de linting específicas de Vue.js para la estructura de componentes, las opciones de datos, las propiedades computadas y las palabras clave reservadas. También incluye anulaciones para evitar falsos positivos de linting en archivos `.vue`.

## Instalación

Agrega la configuración de Vue a tu `biome.jsonc`:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/vue"]
}
```

## Visión general

Esta configuración añade reglas específicas para el desarrollo con Vue.js:

- **Soporte HTML**: Habilita soporte HTML experimental completo con indentación de las etiquetas `<script>` y `<style>`
- **Props específicas de React**: Prohíbe props específicas de React en componentes Vue
- **Anulaciones para archivos .vue**: Reglas relajadas para archivos `.vue` para manejar la estructura de componentes de un solo archivo

## Configuración del formateador HTML

El preset de Vue habilita soporte HTML experimental con un formateo especial para scripts y estilos embebidos:

```jsonc
{
  "html": {
    "experimentalFullSupportEnabled": true,
    "formatter": {
      "indentScriptAndStyle": true
    }
  }
}
```

Esta configuración:
- **Soporte experimental completo**: Habilita las capacidades experimentales de Biome para el parseo y formateo HTML en archivos `.vue`
- **Indentar script y style**: Indenta correctamente el código dentro de las etiquetas `<script>` y `<style>` en componentes Vue de un solo archivo

## Reglas específicas de Vue

### Sospechosas

| Rule | Setting | Description |
|------|---------|-------------|
| `noReactSpecificProps` | `error` | Prohibir props específicas de React como `className` y `htmlFor` en Vue. Usa `class` y `for` en su lugar. |

## Anulaciones para archivos Vue

Las siguientes reglas están deshabilitadas para archivos `.vue` para evitar falsos positivos debido al soporte parcial de Biome para componentes de un solo archivo de Vue:

### Corrección

| Rule | Setting | Description |
|------|---------|-------------|
| `noUnusedVariables` | `off` | Permitir variables sin usar en archivos `.vue`. Las variables en la sección `script` pueden usarse únicamente en la plantilla. |
| `noUnusedImports` | `off` | Permitir imports sin usar en archivos `.vue`. Las importaciones pueden utilizarse únicamente en la sección de plantilla. |

### Estilo

| Rule | Setting | Description |
|------|---------|-------------|
| `useConst` | `off` | Permitir declaraciones `let` y `var` en archivos `.vue` para patrones de datos reactivos. |
| `useImportType` | `off` | Deshabilitar requisitos explícitos de tipo de importación para una mejor compatibilidad con el sistema de compilación de Vue. |
