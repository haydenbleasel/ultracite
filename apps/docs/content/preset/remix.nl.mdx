De Remix-configuratie heeft conventies voor bestandsnamen van routebestanden. Deze configuratie is ook compatibel met TanStack Router, React Router en andere op bestanden gebaseerde routersystemen.

## Installatie

Voeg de Remix-configuratie toe aan je `biome.jsonc`:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/remix"]
}
```

## Overzicht

Deze configuratie biedt speciale behandeling voor routebestanden die bestandssysteem-gebaseerde routerconventies gebruiken:

- **Naamgeving van routebestanden**: Schakelt strikte conventies voor bestandsnamen van routebestanden uit
- **Flexibele patronen**: Ondersteunt Remix' route-naamgevingspatronen (punten, dollartekens, underscores)
- **Framework-onafhankelijk**: Werkt met Remix, TanStack Router, React Router en vergelijkbare systemen

## Overrides voor bestandspatronen

De configuratie past overrides toe op routebestanden met deze patronen:

- `**/routes/**/*.{tsx,ts}` - Remix v1 routes
- `**/app/routes/**/*.{tsx,ts}` - Remix v2 routes

### Uitgeschakelde regels voor routebestanden

| Regel | Instelling | Beschrijving |
|------|-----------|-------------|
| `useFilenamingConvention` | `off` | Schakelt afdwingen van kebab-case bestandsnamen voor routebestanden uit. Staat speciale route-naamgevingen toe. |

## Waarom deze configuratie bestaat

Bestandsgebaseerde routersystemen gebruiken speciale tekens in bestandsnamen om routeringspatronen aan te duiden:

- `.` (punt) - Geeft padsegmenten aan (bijv. `posts.new.tsx` → `/posts/new`)
- `$` (dollarteken) - Geeft dynamische segmenten aan (bijv. `posts.$id.tsx` → `/posts/:id`)
- `_` (underscore) - Staat voor layoutroutes of padloze routes

Deze naamgevingsconventies conflicteren met de standaardvereiste van Ultracite voor kebab-case bestandsnamen, dus deze configuratie schakelt die regel specifiek uit voor routebestanden.
