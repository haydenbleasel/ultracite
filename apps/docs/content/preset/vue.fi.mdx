Vue-konfiguraatio sisältää Vue.js-kohtaisia linttaussääntöjä komponenttirakenteelle, data-optioille, laskennallisille ominaisuuksille ja varatuille avainsanoille. Se sisältää myös poikkeukset väärien positiivisten linttivirheiden estämiseksi `.vue`-tiedostoissa.

## Asennus

Lisää Vue-konfiguraatio `biome.jsonc`-tiedostoosi:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/vue"]
}
```

## Yleiskatsaus

Tämä konfiguraatio lisää Vue.js-kehitykseen liittyviä sääntöjä:

- **HTML-tuki**: Mahdollistaa kokeellisen täydellisen HTML-tuen, joka sisältää skriptien ja tyylien sisennyksen
- **Komponenttien nimeäminen**: Varmistaa, että komponenttien nimet ovat monisanaisia
- **Datan rakenne**: Estää virheelliset `data`-optiomäärittelyt
- **Varaetut avainsanat**: Estää Vue-varattujen propien ja avainten käytön
- **Kaksoisavaimet**: Estää päällekkäisten avainten käytön komponenttioptiokentissä
- **React-spesifiset propit**: Estää React-spesifisten propien käytön Vue-komponenteissa
- **Vue-tiedostojen poikkeukset**: Löyhempinä sääntöinä `.vue`-tiedostoille Single File Component -rakenteen käsittelyyn

## HTML-muotoilijan asetukset

Vue-esiasetus ottaa käyttöön kokeellisen HTML-tuen erityisellä muotoilulla upotetuille skripteille ja tyyleille:

```jsonc
{
  "html": {
    "experimentalFullSupportEnabled": true,
    "formatter": {
      "indentScriptAndStyle": true
    }
  }
}
```

Tämä konfiguraatio:
- **Kokeellinen täydellinen tuki**: Mahdollistaa Biomen kokeellisen HTML-jäsennyksen ja muotoilun `.vue`-tiedostoille
- **Sisennä skripti ja tyyli**: Sisentää oikein koodin `<script>`- ja `<style>`-tageissa Vue Single File Componenteissa

## Vue-kohtaiset säännöt

### Kehitteillä

| Sääntö | Asetus | Kuvaus |
|-------:|:------:|--------|
| `noVueDataObjectDeclaration` | `error` | Estää `data`-option objektimäärittelyn käytön Vue-komponenteissa. `data`-optiion tulee olla funktio, joka palauttaa objektin. |
| `noVueDuplicateKeys` | `error` | Estää päällekkäisten avainten käytön Vue-komponenttien optioissa. Avainten `data`, `computed`, `methods` jne. tulee olla yksilöllisiä. |
| `noVueReservedKeys` | `error` | Kieltää Vue:n varattujen avainten kuten `$data`, `$props`, `$el` jne. käytön komponenttien optioissa. |
| `noVueReservedProps` | `error` | Kieltää Vue-varattujen propien kuten `key`, `ref` ja `is` käytön omina komponenttiproppeina. |
| `noVueSetupPropsReactivityLoss` | `error` | Estää käyttötapoja, jotka aiheuttavat reaktiivisuuden menetyksen Vue `setup`-funktion propeissa. |
| `noVueVIfWithVFor` | `error` | Kieltää `v-for` ja `v-if` saman elementin kohdalla. |
| `useVueHyphenatedAttributes` | `error` | Kannustaa käyttämään attribuuttien nimissä kebab-casea Vue-tyyliohjeen suositusten mukaisesti. |
| `useVueMultiWordComponentNames` | `error` | Varmistaa monisanaiset komponenttien nimet HTML-elementtien konfliktien välttämiseksi. Yksisanaisia nimiä kuten `Button` tai `Card` tulee välttää. |
| `useVueValidVBind` | `error` | Varmistaa `v-bind`-direktiivien oikeellisuuden Vue-tiedostoissa. |
| `useVueValidVElseIf` | `error` | Varmistaa `v-else-if`-direktiivien pätevyyden Vue-templaatissa. Raportoi virheellisistä direktiiveistä, joilta puuttuu ehtolauseke tai joita ei edellä `v-if`- tai `v-else-if`-direktiivi. |
| `useVueValidVElse` | `error` | Varmistaa `v-else`-direktiivien pätevyyden Vue-templaatissa. Raportoi direktiivejä, joita ei edellä `v-if` tai `v-else-if`. |
| `useVueValidVHtml` | `error` | Varmistaa `v-html`-direktiivin oikean käytön Vue-templaatissa. Raportoi direktiivejä, joilta puuttuu lauseke, joilla on odottamattomia argumentteja tai modifioijia. |
| `useVueValidVIf` | `error` | Varmistaa `v-if`-direktiivien pätevyyden Vue-templaatissa. Kieltää argumenttien ja modifioijien käytön ja varmistaa, että arvo on annettu. |
| `useVueValidVOn` | `error` | Varmistaa `v-on`-direktiivien pätevyyden Vue-templaatissa. Raportoi virheellisistä `v-on` / lyhennetystä `@`-syntaksista, joilta puuttuu tapahtuman nimi, joissa on virheellisiä modifioijia tai puuttuu käsittelijälauseke. |
| `useVueValidVText` | `error` | Varmistaa `v-text`-direktiivien pätevyyden Vue-templaatissa. Sääntö raportoi, kun `v-text`illa on argumentti, modifioijia tai arvo puuttuu. |

### Epäilyttävät

| Sääntö | Asetus | Kuvaus |
|-------:|:------:|--------|
| `noReactSpecificProps` | `error` | Kieltää React-spesifisten propien kuten `className` ja `htmlFor` käytön Vue:ssa. Käytä sen sijaan `class` ja `for`. |

## Vue-tiedostojen poikkeukset

Seuraavat säännöt on poistettu käytöstä `.vue`-tiedostoissa virheellisten positiivisten ilmoitusten ehkäisemiseksi Biomen osittaisen tuen vuoksi Vue Single File Componenteille:

### Oikeellisuus

| Sääntö | Asetus | Kuvaus |
|-------:|:------:|--------|
| `noUnusedVariables` | `off` | Salli käyttämättömät muuttujat `.vue`-tiedostoissa. Skriptiosion muuttujia voidaan käyttää vain templaattiosiossa. |
| `noUnusedImports` | `off` | Salli käyttämättömät importit `.vue`-tiedostoissa. Importteja saatetaan käyttää vain templaattiosiossa. |

### Tyyli

| Sääntö | Asetus | Kuvaus |
|-------:|:------:|--------|
| `useConst` | `off` | Salli `let`- ja `var`-määrittelyt `.vue`-tiedostoissa reaktiivisia datamalleja varten. |
| `useImportType` | `off` | Poista käytöstä eksplisiittiset import-tyyppivaatimukset paremman yhteensopivuuden vuoksi Vuen build-järjestelmän kanssa. |
