Die Vue-Konfiguration enthält Vue.js-spezifische Linting-Regeln für Komponentenstruktur, Datenoptionen, berechnete Eigenschaften und reservierte Schlüsselwörter. Sie enthält außerdem Überschreibungen, um falsch-positive Linting-Fehler in `.vue`-Dateien zu verhindern.

## Installation

Fügen Sie die Vue-Konfiguration zu Ihrer `biome.jsonc` hinzu:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/vue"]
}
```

## Überblick

Diese Konfiguration fügt Regeln hinzu, die speziell für die Vue.js-Entwicklung gelten:

- **HTML-Unterstützung**: Aktiviert experimentelle vollständige HTML-Unterstützung mit Einrückung von Skript- und Style-Bereichen
- **React-spezifische Props**: Verbot von React-spezifischen Props in Vue-Komponenten
- **Überschreibungen für `.vue`-Dateien**: Gelockerte Regeln für `.vue`-Dateien, um mit der Struktur von Single File Components umzugehen

## Konfiguration des HTML-Formatierers

Das Vue-Preset aktiviert experimentelle HTML-Unterstützung mit spezieller Formatierung für eingebettete Skripte und Styles:

```jsonc
{
  "html": {
    "experimentalFullSupportEnabled": true,
    "formatter": {
      "indentScriptAndStyle": true
    }
  }
}
```

Diese Konfiguration:
- **Experimentelle vollständige Unterstützung**: Aktiviert Biome's experimentelle HTML-Parsing- und Formatierungsfunktionen für `.vue`-Dateien
- **Einrückung von Skript und Style**: Bezieht die Einrückung von Code innerhalb von `<script>`- und `<style>`-Tags in Vue Single File Components korrekt mit ein

## Vue-spezifische Regeln

### Verdächtig

| Regel | Einstellung | Beschreibung |
|------|---------|-------------|
| `noReactSpecificProps` | `error` | Verbot von React-spezifischen Props wie `className` und `htmlFor` in Vue. Verwenden Sie `class` und `for` stattdessen. |

## Überschreibungen für Vue-Dateien

Die folgenden Regeln sind für `.vue`-Dateien deaktiviert, um falsch-positive Meldungen aufgrund von Biome's teilweiser Unterstützung für Vue Single File Components zu vermeiden:

### Korrektheit

| Regel | Einstellung | Beschreibung |
|------|---------|-------------|
| `noUnusedVariables` | `off` | Erlaubt ungenutzte Variablen in `.vue`-Dateien. Variablen im Script-Bereich können ausschließlich im Template verwendet werden. |
| `noUnusedImports` | `off` | Erlaubt ungenutzte Importe in `.vue`-Dateien. Importe können ausschließlich im Template-Bereich verwendet werden. |

### Stil

| Regel | Einstellung | Beschreibung |
|------|---------|-------------|
| `useConst` | `off` | Erlaubt `let`- und `var`-Deklarationen in `.vue`-Dateien für reaktive Datenmuster. |
| `useImportType` | `off` | Deaktiviert die Anforderungen an explizite Importtypen für eine bessere Kompatibilität mit dem Build-System von Vue. |
