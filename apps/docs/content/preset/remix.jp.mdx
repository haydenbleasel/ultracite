The Remix 設定はルートファイルのファイル命名規則を持っています。この設定は TanStack Router、React Router、その他のファイルベースのルーティングシステムとも互換性があります。

## インストール

Remix 設定を `biome.jsonc` に追加します:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/remix"]
}
```

## 概要

この設定はファイルシステムベースのルーティング規約を使用するルートファイルに対して特別な処理を提供します:

- **ルートファイルの命名**: ルートファイルに対する厳密なファイル名規則を無効化します
- **柔軟なパターン**: Remix のルート命名パターン（ドット、ドル記号、アンダースコア）をサポートします
- **フレームワーク非依存**: Remix、TanStack Router、React Router、類似のシステムで動作します

## ファイルパターンのオーバーライド

この設定は次のパターンのルートファイルに対してオーバーライドを適用します:

- `**/routes/**/*.{tsx,ts}` - Remix v1 routes
- `**/app/routes/**/*.{tsx,ts}` - Remix v2 routes

### ルートファイルで無効化されるルール

| ルール | 設定 | 説明 |
|------|---------|-------------|
| `useFilenamingConvention` | `off` | ルートファイルに対する kebab-case ファイル名強制を無効にします。特殊なルート命名パターンを許可します。 |

## この設定が存在する理由

ファイルベースのルーティングシステムは、ファイル名内の特殊文字をルーティングパターンの表現に使用します:

- `.`（ドット） - パス区切りを表します（例: `posts.new.tsx` → `/posts/new`）
- `$`（ドル） - 動的セグメントを表します（例: `posts.$id.tsx` → `/posts/:id`）
- `_`（アンダースコア） - レイアウトルートまたはパスレスルートを表します

これらの命名規則は Ultracite のデフォルトの kebab-case ファイル名要件と衝突するため、この設定はルートファイルに対してそのルールを無効化します。
