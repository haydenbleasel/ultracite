Ultracite のコア設定は、フレームワークに依存しない包括的なリンティングルールとフォーマット設定を提供します。これは他のすべてのフレームワーク固有の設定が拡張する基盤です。

## インストール

コア設定は次のコマンドを実行すると自動的に含まれます:

```bash
npx ultracite init
```

または手動で `biome.jsonc` に追加します:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core"]
}
```

## 概要

コア設定には、数百に及ぶ厳格なリンティングルールが含まれており、次の分野を網羅しています:

- **アクセシビリティ (a11y)**: ARIA 属性、セマンティック HTML、キーボード操作を強制
- **複雑性**: 認知的複雑さを削減しベストプラクティスを強制
- **正確性**: 一般的な誤りを防ぎ型安全性を強制
- **パフォーマンス**: 実行時パフォーマンスの最適化
- **セキュリティ**: セキュリティの脆弱性を防止
- **スタイル**: 一貫したコードスタイルを強制
- **疑わしいパターン**: バグを示唆する疑わしいパターンを検出

## フォーマッタ設定

- **インデント**: 2 スペース
- **行幅**: 80 文字
- **行末**: LF（Unix 形式）
- **セミコロン**: 常に必須
- **クォート**: JavaScript/TypeScript と JSON ではダブルクォート
- **末尾カンマ**: ES5 スタイル（複数行の配列/オブジェクト）
- **アロー関数の括弧**: 常に括弧を含める

## 主要な特徴

### TypeScript の厳格さ

コア設定は TypeScript の厳格チェックを有効にします:

- `any` の使用を抑制 (`noExplicitAny: error`)
- `null/undefined` の取り扱いを要求
- 特定の状況では明示的な型を推奨
- コードベース全体で型安全性を強制

### ESLint 推奨相当ルール

一般的なベストプラクティスのルールを含みます:

- 未使用変数の禁止 (`noUnusedVariables: error`)
- 未使用インポートの禁止 (`noUnusedImports: error`)
- `eval` の明示的使用禁止 (`noGlobalEval: error`)
- プロトタイプ汚染の防止 (`noPrototypeBuiltins: error`)

### デフォルトでのアクセシビリティ

アクセシビリティ基準に相当するルール:

- ARIA 属性は有効であること
- セマンティックな HTML 要素の優先
- キーボード操作のサポートを要求
- 画像に対する代替テキストを要求

### Node.js サポート

環境固有の取り扱い:

- Node.js のグローバル（`module`, `process`）を認識
- 組み込みモジュールに対して `node:` プロトコルを強制
- CommonJS と ESM の両方をサポート

## 注意

一部のルールは Scanner で実行するには遅すぎるため無効にされています。[Biome のドキュメント](https://biomejs.dev/guides/investigate-slowness/#first-things-first) にも記載されているように、プロジェクトで有効化されたルールは、より高度な解析の代償としてパフォーマンスに影響を与えることが知られています。

我々のテストでは、これらのルールが有効かどうかで 300ms と 13s の差が生じることが確認されています。
