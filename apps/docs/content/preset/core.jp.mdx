Ultracite のコア設定は、フレームワークに依存しない包括的なリンティングルールとフォーマット設定を提供します。これは他のフレームワーク固有の設定が拡張する基盤です。

## インストール

コア設定は次を実行すると自動的に含まれます:

```bash
npx ultracite init
```

または手動で `biome.jsonc` に追加します:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core"]
}
```

## 概要

コア設定には数百に及ぶ厳格なリンティングルールが含まれ、次の分野を網羅します:

- **アクセシビリティ（a11y）**: ARIA 属性、意味のある HTML、キーボードナビゲーションを強制
- **複雑性**: 認知的複雑さを減らしベストプラクティスを強制
- **正確性**: 一般的なエラーを防ぎ型安全性を強制
- **パフォーマンス**: 実行時パフォーマンスの最適化
- **セキュリティ**: セキュリティ脆弱性の防止
- **スタイル**: 一貫したコードスタイルを強制
- **疑わしいパターン**: バグを示唆する可能性のある疑わしいパターンを検出

## フォーマッタ設定

- **インデント**: 2 スペース
- **行幅**: 80 文字
- **行末**: LF（Unix スタイル）
- **セミコロン**: 常に必要
- **引用符**: JavaScript/TypeScript と JSON にはダブルクォート
- **末尾カンマ**: ES5 スタイル（複数行の配列／オブジェクト）
- **アロー関数の括弧**: 常に括弧を含める

## 主要な機能

### TypeScript の厳格性

コア設定は TypeScript の厳格チェックを有効にします:

- `any` の使用を推奨しません（`noExplicitAny: error`）
- `null/undefined` の扱いを要求
- 特定の状況では明示的な型を優先
- コードベース全体で型安全性を強制

### ESLint 推奨ルールの同等規則

一般的なベストプラクティスルールが含まれます:

- 未使用変数の禁止（`noUnusedVariables: error`）
- 未使用インポートの禁止（`noUnusedImports: error`）
- 明示的な eval の禁止（`noGlobalEval: error`）
- プロトタイプ汚染の禁止（`noPrototypeBuiltins: error`）

### デフォルトでのアクセシビリティ

アクセシビリティ基準に相当するルール:

- ARIA 属性は有効でなければならない
- 意味のある HTML 要素を推奨
- キーボードナビゲーションのサポートを要求
- 画像には代替テキストを要求

### Node.js サポート

環境固有の扱い:

- Node.js のグローバル（`module`, `process`）を認識
- 組み込みモジュールには `node:` プロトコルを強制
- CommonJS と ESM の両方をサポート

### Nursery（実験的）

Nursery ルールは一般的に Ultracite では有効にされていません。これらは実験的であり将来の Biome バージョンで変更または削除される可能性があります。しかし、以下のルールは人気のために有効になっています:

| ルール | 設定 | 説明 |
|------|---------|-------------|
| `useSortedClasses` | `error` (safe fix override) | CSS ユーティリティクラスのソートを強制します。 |

## 注意事項

一部のルールは Scanner と併用すると実行が遅くなるため無効化されています。[Biome ドキュメント](https://biomejs.dev/guides/investigate-slowness/#first-things-first) にあるように、有効にしたプロジェクトルールはより高度な解析と引き換えにパフォーマンスオーバーヘッドを引き起こすことが知られています。

私たちのテストでは、これらのルールがリンティング時間を約 300ms と 13s の差にする可能性があることがわかっています。
