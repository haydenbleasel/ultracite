La configuration Vue contient des règles de linting spécifiques à Vue.js pour la structure des composants, les options de données, les propriétés calculées et les mots réservés. Elle inclut également des remplacements pour éviter les faux positifs de linting dans les fichiers `.vue`.

## Installation

Ajoutez la configuration Vue à votre `biome.jsonc`:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/vue"]
}
```

## Aperçu

Cette configuration ajoute des règles spécifiques au développement Vue.js :

- **Prise en charge HTML**: Active la prise en charge HTML complète expérimentale avec indentation des balises `<script>` et `<style>`
- **Props spécifiques à React**: Interdit les props spécifiques à React dans les composants Vue
- **Remplacements pour fichiers Vue**: Règles assouplies pour les fichiers `.vue` afin de gérer la structure des composants mono-fichier

## Configuration du formatteur HTML

Le preset Vue active la prise en charge HTML expérimentale avec un formatage spécial pour les scripts et styles intégrés :

```jsonc
{
  "html": {
    "experimentalFullSupportEnabled": true,
    "formatter": {
      "indentScriptAndStyle": true
    }
  }
}
```

Cette configuration :
- **Prise en charge complète expérimentale**: Active les capacités expérimentales de Biome pour l'analyse et le formatage HTML des fichiers `.vue`
- **Indentation des script et style**: Indente correctement le code à l'intérieur des balises `<script>` et `<style>` dans les composants mono-fichier Vue

## Règles spécifiques à Vue

### Suspicieux

| Règle | Paramètre | Description |
|------|---------|-------------|
| `noReactSpecificProps` | `error` | Interdit les props spécifiques à React comme `className` et `htmlFor` dans Vue. Utilisez `class` et `for` à la place. |

## Remplacements pour les fichiers Vue

Les règles suivantes sont désactivées pour les fichiers `.vue` afin d'éviter des faux positifs dus au support partiel de Biome pour les composants mono-fichier Vue :

### Correction

| Règle | Paramètre | Description |
|------|---------|-------------|
| `noUnusedVariables` | `off` | Autorise les variables non utilisées dans les fichiers `.vue`. Les variables définies dans la section `<script>` peuvent n'être utilisées que dans la section `<template>`. |
| `noUnusedImports` | `off` | Autorise les imports non utilisés dans les fichiers `.vue`. Les imports peuvent n'être utilisés que dans la section `<template>`. |

### Style

| Règle | Paramètre | Description |
|------|---------|-------------|
| `useConst` | `off` | Autorise les déclarations `let` et `var` dans les fichiers `.vue` pour les modèles de données réactives. |
| `useImportType` | `off` | Désactive l'obligation des types d'import explicites pour une meilleure compatibilité avec le système de build de Vue. |
