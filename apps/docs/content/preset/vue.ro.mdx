Configurația Vue include reguli de lint specifice Vue.js pentru structura componentelor, opțiunile `data`, proprietățile `computed` și cuvintele rezervate. Include, de asemenea, suprascrieri pentru a preveni erori false pozitive de lint în fișierele `.vue`.

## Instalare

Adaugă configurația Vue în `biome.jsonc`:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/vue"]
}
```

## Prezentare generală

Această configurație adaugă reguli specifice dezvoltării Vue.js:

- **Suport HTML**: Activează suportul HTML complet experimental cu indentare pentru script și style
- **Props specifice React**: Interzice props specifice React în componentele Vue
- **Suprascrieri pentru fișiere Vue**: Reguli relaxate pentru fișiere `.vue` pentru a gestiona structura Single File Component

## Configurarea formatatorului HTML

Presetul Vue activează suport HTML experimental cu formatare specială pentru scripturi și style încorporate:

```jsonc
{
  "html": {
    "experimentalFullSupportEnabled": true,
    "formatter": {
      "indentScriptAndStyle": true
    }
  }
}
```

Această configurație:
- **Suport complet experimental**: Activează capacitățile experimentale de parsare și formatare HTML ale Biome pentru fișiere `.vue`
- **Indentare script și style**: Indentează corect codul din interiorul etichetelor `<script>` și `<style>` în Vue Single File Components

## Reguli specifice Vue

### Suspecte

| Regulă | Setare | Descriere |
|--------|--------|-----------|
| `noReactSpecificProps` | `error` | Interzice props specifice React precum `className` și `htmlFor` în Vue. Folosește `class` și `for` în schimb. |

## Suprascrieri pentru fișiere Vue

Următoarele reguli sunt dezactivate pentru fișierele `.vue` pentru a preveni rezultate false pozitive din cauza suportului parțial al Biome pentru Vue Single File Components:

### Corectitudine

| Regulă | Setare | Descriere |
|--------|--------|-----------|
| `noUnusedVariables` | `off` | Permite variabile neutilizate în fișierele `.vue`. Variabilele din secțiunea script pot fi folosite doar în template. |
| `noUnusedImports` | `off` | Permite importuri neutilizate în fișierele `.vue`. Importurile pot fi folosite doar în secțiunea template. |

### Stil

| Regulă | Setare | Descriere |
|--------|--------|-----------|
| `useConst` | `off` | Permite declarații `let` și `var` în fișierele `.vue` pentru pattern-uri de date reactive. |
| `useImportType` | `off` | Dezactivează cerințele explicite de import type pentru o compatibilitate mai bună cu sistemul de build al Vue. |
