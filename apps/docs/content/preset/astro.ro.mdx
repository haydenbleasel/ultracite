Configurarea pentru Astro conține ajustări ale regulilor de lint specifice Astro pentru a gestiona structura unică a componentelor Astro, în special în jurul scripturilor din frontmatter și al sintaxei componentelor.

## Instalare

Adaugă configurarea pentru Astro în `biome.jsonc`:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/astro"]
}
```

## Prezentare generală

Această configurare ajustează regulile pentru dezvoltarea cu Astro:

- **Variabile/Importuri neutilizate**: Permite variabilele și importurile neutilizate în fișierele Astro din cauza constrângerilor frontmatter, unde variabilele pot fi folosite doar în secțiunea de șablon
- **Declarații de variabile**: Relaxează aplicarea `const` deoarece frontmatter-ul Astro poate necesita variabile care pot fi reatribuite
- **Tipuri de import**: Dezactivează cerințele explicite privind tipurile de import pentru o compatibilitate mai bună
- **Props specifice React**: Semnalează props specifice React, cum ar fi `className`, deoarece Astro folosește atribute HTML standard

## Reguli specifice Astro

### Corectitudine

| Rule | Setting | Description |
|------|---------|-------------|
| `noUnusedVariables` | `off` | Permite variabilele neutilizate în fișierele Astro. Variabilele declarate în frontmatter pot fi folosite doar în secțiunea de șablon. |
| `noUnusedImports` | `off` | Permite importurile neutilizate în fișierele Astro. Importurile din frontmatter pot fi folosite doar în secțiunea de șablon. |

### Stil

| Rule | Setting | Description |
|------|---------|-------------|
| `useConst` | `off` | Permite declarațiile `let` și `var`. Frontmatter-ul componentelor Astro poate necesita variabile reatribuite. |
| `useImportType` | `off` | Dezactivează cerințele explicite privind tipul de import pentru o compatibilitate mai bună cu sistemul de build al Astro. |

### Suspecte

| Rule | Setting | Description |
|------|---------|-------------|
| `noReactSpecificProps` | `error` | Interzice props specifice React precum `className` și `htmlFor` în Astro. Folosește `class` și `for` în schimb. |
