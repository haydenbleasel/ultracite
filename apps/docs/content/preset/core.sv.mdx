Kärnkonfigurationen för Ultracite tillhandahåller en omfattande uppsättning ramverksoberoende lint-regler och formateringsinställningar. Detta är grunden som alla andra ramverksspecifika konfigurationer bygger på.

## Installation

Kärnkonfigurationen inkluderas automatiskt när du kör:

```bash
npx ultracite init
```

Eller lägg till den manuellt i din `biome.jsonc`:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core"]
}
```

## Översikt

Kärnkonfigurationen inkluderar hundratals strikta lint-regler som täcker:

- **Åtkomlighet (a11y)**: Tvinga ARIA-attribut, semantisk HTML och tangentbordsnavigering
- **Komplexitet**: Minska kognitiv komplexitet och säkerställ bästa praxis
- **Korrekthet**: Förhindra vanliga fel och upprätthålla typesäkerhet
- **Prestanda**: Optimera kod för bättre körningstidsprestanda
- **Säkerhet**: Förhindra säkerhetssårbarheter
- **Stil**: Säkerställ en konsekvent kodstil
- **Misstänkt**: Fånga misstänkta mönster som kan tyda på buggar

## Formatteringsinställningar

- **Indentering**: 2 mellanslag
- **Radbredd**: 80 tecken
- **Radslut**: LF (Unix-stil)
- **Semikolon**: Alltid obligatoriskt
- **Citattecken**: Dubbla citationstecken för JavaScript/TypeScript och JSON
- **Efterföljande kommatecken**: ES5-stil (flerradiga arrayer/objekt)
- **Parenteser i arrow-funktioner**: Inkludera alltid parenteser

## Nyckelfunktioner

### TypeScript-strikthet

Kärnkonfigurationen aktiverar TypeScripts strikta kontroller:

- Avråder från att använda `any` (`noExplicitAny: error`)
- Kräver hantering av `null/undefined`
- Föredrar explicita typer i vissa situationer
- Upprätthåller typesäkerhet över hela kodbasen

### ESLint rekommenderade ekvivalenter

Vanliga bästa-praktik-regler ingår:

- Inga oanvända variabler (`noUnusedVariables: error`)
- Inga oanvända importeringar (`noUnusedImports: error`)
- Ingen explicit eval (`noGlobalEval: error`)
- Ingen prototype-förorening (`noPrototypeBuiltins: error`)

### Åtkomlighet som standard

Regler motsvarande åtkomststandarder:

- ARIA-attribut måste vara giltiga
- Semantiska HTML-element föredras
- Stöd för tangentbordsnavigering krävs
- Alt-text för bilder krävs

### Node.js-stöd

Miljöspecifik hantering:

- Känner igen Node.js-globals (`module`, `process`)
- Kräver `node:`-protokollet för inbyggda moduler
- Stöder både CommonJS och ESM

## Noteringar

Vissa regler är inaktiverade eftersom de är för långsamma att köra med Scanner. Som nämnt i [Biome-dokumentationen](https://biomejs.dev/guides/investigate-slowness/#first-things-first), är det känt att aktiverade projektsregler kan orsaka prestandaöverhead i utbyte mot mer avancerad analys.

I våra tester har vi funnit att dessa regler kan göra skillnaden mellan en lint som tar 300 ms och en som tar 13 s.
