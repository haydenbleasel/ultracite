Core-konfigurationen för Ultracite tillhandahåller en omfattande uppsättning ramverksoberoende lint-regler och formatteringsinställningar. Detta är grunden som alla andra ramverksspecifika konfigurationer bygger vidare på.

## Installation

Kärnkonfigurationen inkluderas automatiskt när du kör:

```bash
npx ultracite init
```

Eller lägg till den manuellt i din `biome.jsonc`:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core"]
}
```

## Översikt

Kärnkonfigurationen inkluderar hundratals strikta lint-regler som täcker:

- **Tillgänglighet (a11y)**: Tvinga ARIA‑attribut, semantisk HTML och tangentbordsnavigering
- **Komplexitet**: Minska kognitiv komplexitet och upprätthåll bästa praxis
- **Korrekthet**: Förhindra vanliga fel och säkerställ typsäkerhet
- **Prestanda**: Optimera kod för bättre körningstidsprestanda
- **Säkerhet**: Förhindra säkerhetssårbarheter
- **Stil**: Upprätthåll konsekvent kodstil
- **Misstänkt**: Fånga misstänkta mönster som kan indikera buggar

## Formatteringsinställningar

- **Indragning**: 2 mellanslag
- **Radbredd**: 80 tecken
- **Radslut**: LF (Unix‑stil)
- **Semikolon**: Alltid krav
- **Citationstecken**: Dubbelcitat för JavaScript/TypeScript och JSON
- **Avslutande kommatecken**: ES5‑stil (fler‑radiga arrayer/objekt)
- **Parenteser i pilfunktioner**: Inkludera alltid parenteser

## Huvudfunktioner

### TypeScript‑strikthet

Kärnkonfigurationen aktiverar TypeScripts strikta kontroller:

- Avråder från användning av `any` (`noExplicitAny: error`)
- Kräver hantering av `null/undefined`
- Föredrar explicita typer i vissa situationer
- Upprätthåller typsäkerhet över hela kodbasen

### ESLint‑rekommenderade ekvivalenter

Vanliga bästa‑praxis‑regler ingår:

- Inga oanvända variabler (`noUnusedVariables: error`)
- Inga oanvända imports (`noUnusedImports: error`)
- Ingen explicit eval (`noGlobalEval: error`)
- Ingen prototype‑förorening (`noPrototypeBuiltins: error`)

### Tillgänglighet som standard

Regler motsvarande tillgänglighetsstandarder:

- ARIA‑attribut måste vara giltiga
- Semantiska HTML‑element föredras
- Tangentbordsnavigering måste stödjas
- Alt‑text för bilder krävs

### Node.js‑stöd

Miljö‑specifik hantering:

- Känner igen Node.js‑globala (`module`, `process`)
- Kräver `node:`‑protokoll för inbyggda moduler
- Stöder både CommonJS och ESM

## Anteckningar

Vissa regler är avaktiverade eftersom de är för långsamma att köra med Scanner. Som nämnts i [Biome docs](https://biomejs.dev/guides/investigate-slowness/#first-things-first) är vissa projektregler kända för att orsaka prestandaöverhäng i utbyte mot mer avancerad analys.

I våra tester har vi funnit att dessa regler kan göra skillnaden mellan 300 ms och 13 s i lintning.
