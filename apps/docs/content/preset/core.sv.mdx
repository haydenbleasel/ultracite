Core-konfigurationen för Ultracite tillhandahåller ett omfattande set med ramverksoberoende lintregler och formatteringsinställningar. Detta är grunden som alla andra ramverksspecifika konfigurationer bygger på.

## Installation

Kärnkonfigurationen inkluderas automatiskt när du kör:

```bash
npx ultracite init
```

Eller lägg till den manuellt i din `biome.jsonc`:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core"]
}
```

## Översikt

Kärnkonfigurationen inkluderar hundratals strikta lintregler som täcker:

- **Tillgänglighet (a11y)**: Säkerställ ARIA-attribut, semantisk HTML och tangentbordsnavigering
- **Komplexitet**: Minska kognitiv komplexitet och använd bästa praxis
- **Korrekthet**: Förhindra vanliga fel och säkerställ typ-säkerhet
- **Prestanda**: Optimera kod för bättre körtidsprestanda
- **Säkerhet**: Förhindra säkerhetssårbarheter
- **Stil**: Genomdriv konsekvent kodstil
- **Misstänkta**: Fånga misstänkta mönster som kan indikera buggar

## Formatteringsinställningar

- **Indentering**: 2 blanksteg
- **Radbredd**: 80 tecken
- **Radavslut**: LF (Unix-stil)
- **Semikolon**: Alltid obligatoriskt
- **Citat**: Dubbelcitat för JavaScript/TypeScript och JSON
- **Efterkommatecken**: ES5-stil (fler-radiga arrayer/objekt)
- **Parenteser för arrow-funktioner**: Inkludera alltid parenteser

## Huvudfunktioner

### Strikthet för TypeScript

Kärnkonfigurationen aktiverar TypeScripts strikta kontroller:

- Avråder från användning av `any` (`noExplicitAny: error`)
- Kräver hantering av `null/undefined`
- Föredrar explicita typer i vissa situationer
- Genomdriver typ-säkerhet i hela kodbasen

### ESLint rekommenderade ekvivalenter

Vanliga bästa-praktik-regler som ingår:

- Ingen oanvänd variabler (`noUnusedVariables: error`)
- Inga oanvända imports (`noUnusedImports: error`)
- Ingen explicit eval (`noGlobalEval: error`)
- Ingen prototype-pollution (`noPrototypeBuiltins: error`)

### Tillgänglighet som standard

Regler som motsvarar tillgänglighetsstandarder:

- ARIA-attribut måste vara giltiga
- Semantiska HTML-element föredras
- Stöd för tangentbordsnavigering krävs
- Alt-text för bilder krävs

### Node.js-stöd

Miljöspecifik hantering:

- Känner igen Node.js-globals (`module`, `process`)
- Genomdriver `node:`-protokollet för inbyggda moduler
- Stöder både CommonJS och ESM

## Anmärkningar

Vissa regler är inaktiverade eftersom de är för långsamma att köra med Scanner. Som nämnts i [Biome-dokumentation](https://biomejs.dev/guides/investigate-slowness/#first-things-first) orsakar vissa aktiverade projektregler prestandaöverhuvud i utbyte mot mer avancerad analys.

I våra tester har vi funnit att dessa regler kan göra skillnaden mellan en lint på 300ms och en på 13s.
