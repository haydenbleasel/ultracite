Die React-Konfiguration enthält React-spezifische Linting-Regeln für JSX, Hooks und Komponentenmuster.

## Installation

Fügen Sie die React-Konfiguration zu Ihrer `biome.jsonc` hinzu:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/react"]
}
```

## Übersicht

Diese Konfiguration fügt Regeln hinzu, die speziell für die React-Entwicklung gelten:

- **JSX-Regeln**: Durchsetzung korrekter JSX-Syntax und -Muster
- **Hooks-Regeln**: Sicherstellen, dass React-Hooks korrekt verwendet werden
- **Komponentenregeln**: Durchsetzung der besten Vorgehensweisen für React-Komponenten
- **Umgang mit children**: Verhindern unsachgemäßen Gebrauchs der children-Prop

## React-spezifische Regeln

### Korrektheit

| Regel | Einstellung | Beschreibung |
|------|---------|-------------|
| `noChildrenProp` | `error` | Verhindert, dass children als Prop übergeben werden. children sollten zwischen dem öffnenden und dem schließenden Tag platziert werden. |
| `noNestedComponentDefinitions` | `error` | Verhindert Komponenten-Definitionen innerhalb anderer Komponenten. Dies führt zu unnötigen Re-Renders. |
| `noReactPropAssignments` | `error` | Verhindert das Neuzuweisen von Props in React-Komponenten. Props sollten als unveränderlich behandelt werden. |
| `noRenderReturnValue` | `error` | Verhindert die Nutzung des Rückgabewerts von `ReactDOM.render()`. |
| `noVoidElementsWithChildren` | `error` | Verhindert, dass Void-Elemente (wie `<img>`, `<br>`) children haben. |
| `useExhaustiveDependencies` | `error` | Stellt sicher, dass alle Abhängigkeiten in React-Hooks (useEffect, useCallback, useMemo) korrekt angegeben sind. |
| `useHookAtTopLevel` | `error` | Stellt sicher, dass alle React-Hooks auf der obersten Ebene von Komponentenfunktionen aufgerufen werden und nicht in Schleifen, Bedingungen oder verschachtelten Funktionen. |
| `useJsxKeyInIterable` | `error` | Stellt sicher, dass Elemente in Iterables eine `key`-Prop für die Reconciliation von React haben. |

### Stil

| Regel | Einstellung | Beschreibung |
|------|---------|-------------|
| `useReactFunctionComponents` | `error` | Erzwingt die Verwendung von Funktionskomponenten statt Klassenkomponenten in React. |

### Verdächtig

| Regel | Einstellung | Beschreibung |
|------|---------|-------------|
| `noArrayIndexKey` | `error` | Verhindert die Verwendung von Array-Indizes als `key`. Array-Indizes sind keine stabilen Identifikatoren und können Probleme mit dem Komponentenstatus verursachen. |
| `noDuplicateJsxProps` | `error` | Verhindert doppelte Eigenschaften in JSX. |
| `noSuspiciousSemicolonInJsx` | `error` | Verhindert Semikolons, die die Semantik von JSX-Elementen verändern. |
| `noReactSpecificProps` | `off` | Erlaubt React-spezifische Props wie `className` und `htmlFor`. |
