A configuração do Qwik possui regras de linting específicas para serialização, tarefas visíveis, escopo léxico e uso de métodos.

## Instalação

Adicione a configuração do Qwik ao seu `biome.jsonc`:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/qwik"]
}
```

## Visão geral

Esta configuração adiciona regras específicas para desenvolvimento com Qwik:

- **Serialização**: Aplicar o uso adequado do modelo de retomabilidade do Qwik
- **Tarefas Visíveis**: Evitar o uso inadequado de `useVisibleTask$`
- **Props Específicas do React**: Proibir o uso de props específicas do React em componentes Qwik

## Regras específicas do Qwik

### Correção

| Regra | Configuração | Descrição |
|------|--------------|----------|
| `noQwikUseVisibleTask` | `error` | Evitar o uso de `useVisibleTask$`. Este hook executa código de forma imediata no cliente, comprometendo a retomabilidade do Qwik. Use `useTask$` ou `useResource$` quando possível. |
| `useQwikClasslist` | `error` | Exigir o uso da sintaxe de objeto `class` em vez de concatenação de strings para classes dinâmicas no Qwik. |

### Suspeitas

| Regra | Configuração | Descrição |
|------|--------------|----------|
| `noReactSpecificProps` | `error` | Proibir props específicas do React como `className` e `htmlFor` no Qwik. Use `class` e `for` em vez disso. |
