Qwik-konfigurationen indeholder Qwik-specifikke linting-regler for serialisering, synlige opgaver, leksikalsk omfang og metodebrug.

## Installation

Tilføj Qwik-konfigurationen til din `biome.jsonc`:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/qwik"]
}
```

## Oversigt

Denne konfiguration tilføjer regler specifikke for Qwik-udvikling:

- **Serialisering**: Håndhæv korrekt brug af Qwiks resumabilitetsmodel
- **Synlige opgaver**: Forhindre uhensigtsmæssig brug af `useVisibleTask$`
- **React-specifikke props**: Forbyd React-specifikke props i Qwik-komponenter

## Qwik-specifikke regler

### Korrekthed

| Regel | Indstilling | Beskrivelse |
|------|---------|-------------|
| `noQwikUseVisibleTask` | `error` | Forhindre brug af `useVisibleTask$`. Denne hook kører kode øjeblikkeligt på klienten og undergraver Qwiks resumabilitetsmodel. Brug `useTask$` eller `useResource$` i stedet, når det er muligt. |
| `useQwikClasslist` | `error` | Håndhæv brug af `class`-objektsyntaks i stedet for strengsammenkædning til dynamiske klasser i Qwik. |

### Mistænkelige

| Regel | Indstilling | Beskrivelse |
|------|---------|-------------|
| `noReactSpecificProps` | `error` | Forbyd React-specifikke props som `className` og `htmlFor` i Qwik. Brug `class` og `for` i stedet. |
