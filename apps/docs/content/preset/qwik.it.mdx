La configurazione Qwik include regole di linting specifiche per Qwik relative a serializzazione, task visibili, scope lessicale e uso dei metodi.

## Installazione

Aggiungi la configurazione Qwik al tuo `biome.jsonc`:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/qwik"]
}
```

## Panoramica

Questa configurazione aggiunge regole specifiche per lo sviluppo con Qwik:

- **Serializzazione**: Imporre l'uso corretto del modello di resumabilità di Qwik
- **Task visibili**: Impedire l'uso inappropriato di `useVisibleTask$`
- **Prop specifiche di React**: Vietare le prop specifiche di React nei componenti Qwik

## Regole specifiche per Qwik

### Correttezza

| Regola | Impostazione | Descrizione |
|--------|--------------|-------------|
| `noQwikUseVisibleTask` | `error` | Impedire l'uso di `useVisibleTask$`. Questo hook esegue il codice in modo immediato sul client, compromettendo la resumabilità di Qwik. Usare `useTask$` o `useResource$` quando possibile. |
| `useQwikClasslist` | `error` | Imporre l'uso della sintassi oggetto `class` invece della concatenazione di stringhe per le classi dinamiche in Qwik. |

### Sospetti

| Regola | Impostazione | Descrizione |
|--------|--------------|-------------|
| `noReactSpecificProps` | `error` | Vietare le prop specifiche di React come `className` e `htmlFor` in Qwik. Usare `class` e `for` al loro posto. |
