La configuration Qwik contient des règles de linting spécifiques à Qwik pour la sérialisation, les tâches visibles, la portée lexicale et l'utilisation des méthodes.

## Installation

Ajoutez la configuration Qwik à votre `biome.jsonc` :

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/qwik"]
}
```

## Aperçu

Cette configuration ajoute des règles spécifiques au développement Qwik :

- **Sérialisation** : Faire respecter l'utilisation appropriée du modèle de résumabilité de Qwik
- **Tâches visibles** : Empêcher l'utilisation inappropriée de `useVisibleTask$`
- **Props spécifiques à React** : Interdire les props spécifiques à React dans les composants Qwik

## Règles spécifiques à Qwik

### Correction

| Règle | Paramètre | Description |
|------|---------|-------------|
| `noQwikUseVisibleTask` | `error` | Empêcher l'utilisation de `useVisibleTask$`. Ce hook exécute le code immédiatement côté client, compromettant la résumabilité de Qwik. Utilisez `useTask$` ou `useResource$` lorsque c'est possible. |
| `useQwikClasslist` | `error` | Imposer l'utilisation de la syntaxe d'objet `class` plutôt que la concaténation de chaînes pour les classes dynamiques dans Qwik. |

### Suspectes

| Règle | Paramètre | Description |
|------|---------|-------------|
| `noReactSpecificProps` | `error` | Interdire les props spécifiques à React comme `className` et `htmlFor` dans Qwik. Utilisez `class` et `for` à la place. |
