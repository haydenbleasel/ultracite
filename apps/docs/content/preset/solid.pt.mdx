A configuração para Solid inclui regras de lint específicas do Solid.js para reatividade, padrões de componentes e tratamento de JSX.

## Instalação

Adicione a configuração para Solid ao seu `biome.jsonc`:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/solid"]
}
```

## Visão geral

Esta configuração adiciona regras específicas para desenvolvimento com Solid.js:

- **Regras de Reatividade**: Garantir o uso correto dos primitivos reativos do Solid
- **Padrões de Componentes**: Garantir que componentes Solid sigam as melhores práticas
- **Manipulação de Props**: Prevenir a desestruturação de props (quebra a reatividade)
- **Props Específicos do React**: Proibir props específicos do React em componentes Solid

## Regras Específicas do Solid

### Correção

| Regra | Configuração | Descrição |
|------|---------|-------------|
| `noSolidDestructuredProps` | `error` | Prevenir a desestruturação de props em componentes Solid. A desestruturação quebra o sistema de reatividade do Solid. Acesse as props diretamente. |

### Desempenho

| Regra | Configuração | Descrição |
|------|---------|-------------|
| `useSolidForComponent` | `error` | Fazer cumprir o uso do componente `<For>` para iterar sobre arrays no Solid. O componente `<For>` é otimizado para a reatividade do Solid. |

### Suspeitas

| Regra | Configuração | Descrição |
|------|---------|-------------|
| `noReactSpecificProps` | `error` | Proibir props específicos do React como `className` e `htmlFor` no Solid. Use `class` e `for` em vez disso. |
