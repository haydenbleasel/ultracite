Core Ultracite -konfiguraatio tarjoaa kattavan joukon kehysriippumattomia lint-sääntöjä ja muotoiluasetuksia. Tämä on perusta, jota kaikki muut kehyskohtaiset konfiguraatiot laajentavat.

## Asennus

Core-konfiguraatio sisältyy automaattisesti, kun suoritat:

```bash
npx ultracite init
```

Tai lisää se manuaalisesti `biome.jsonc`-tiedostoosi:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core"]
}
```

## Yleiskatsaus

Core-konfiguraatio sisältää satoja tiukkoja lint-sääntöjä, jotka kattaa:

- **Esteettömyys (a11y)**: Pakota ARIA-attribuutit, semanttinen HTML ja näppäimistönavigointi
- **Monimutkaisuus**: Vähennä kognitiivista monimutkaisuutta ja pakota parhaat käytännöt
- **Oikeellisuus**: Estä yleiset virheet ja pakota tyyppiturvallisuus
- **Suorituskyky**: Optimoi koodi parempaan suoritusaikaan
- **Turvallisuus**: Estä tietoturva-aukkoja
- **Tyyli**: Pakota yhtenäinen koodityyli
- **Epäilyttävät**: Tunnista epäilyttäviä kaavoja, jotka voivat viitata virheisiin

## Muotoilijan asetukset

- **Sisennys**: 2 välilyöntiä
- **Rivileveys**: 80 merkkiä
- **Rivin lopetus**: LF (Unix-tyyli)
- **Puolipisteet**: Vaaditaan aina
- **Lainausmerkit**: Kaksoislainausmerkit JavaScript/TypeScript- ja JSON-tiedostoille
- **Perään lisättävät pilkut**: ES5-tyyli (moniriviset taulukot/oliot)
- **Nuolifunktioiden sulkeet**: Sulje aina mukana

## Keskeiset ominaisuudet

### TypeScriptin tiukkuus

Core-konfiguraatio ottaa käyttöön TypeScriptin tiukat tarkistukset:

- Vastustaa `any`-tyypin käyttöä (`noExplicitAny: error`)
- Edellyttää `null`/`undefined`-tilojen käsittelyä
- Suosii eksplisiittisiä tyyppejä tietyissä tilanteissa
- Pakottaa tyyppiturvallisuuden koko koodikannassa

### ESLintin suositellut vastineet

Yleiset parhaat käytännöt sisällytetään:

- Ei käyttämättömiä muuttujia (`noUnusedVariables: error`)
- Ei käyttämättömiä importteja (`noUnusedImports: error`)
- Ei `eval`-käyttöä (`noGlobalEval: error`)
- Ei prototype-pollutioa (`noPrototypeBuiltins: error`)

### Esteettömyys oletuksena

Säännöt, jotka vastaavat esteettömyysstandardeja:

- ARIA-attribuuttien on oltava kelvollisia
- Semanttisia HTML-elementtejä suositellaan
- Näppäimistönavigoinnin tuki vaaditaan
- Kuville vaaditaan vaihtoehtoinen teksti

### Node.js-tuki

Ympäristökohtainen käsittely:

- Tunnistaa Node.js:n globaaliavaimet (`module`, `process`)
- Pakottaa `node:`-protokollan sisäänrakennetuille moduuleille
- Tukee sekä CommonJS:ää että ESM:ää

## Huomautukset

Tietyt säännöt on poistettu käytöstä, koska ne ovat liian hitaita Scannerin kanssa suoritettaviksi. Kuten [Biome-dokumentaatiossa](https://biomejs.dev/guides/investigate-slowness/#first-things-first) mainitaan, projektissa käytössä olevat säännöt tiedetään aiheuttavan suorituskykykustannuksia vastineeksi edistyneemmästä analyysista.

Testauksessamme olemme havainneet, että nämä säännöt voivat tehdä eron 300 ms ja 13 s lint-suorituksen välillä.
