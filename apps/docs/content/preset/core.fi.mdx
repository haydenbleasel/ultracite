Ultraciten ydinkonfiguraatio tarjoaa kattavan joukon kehysriippumattomia lint-sääntöjä ja muotoiluasetuksia. Tämä on perusta, jota kaikki muut kehyspesifiset määritykset laajentavat.

## Asennus

Ydinkonfiguraatio sisältyy automaattisesti, kun suoritat:

```bash
npx ultracite init
```

Tai lisää se manuaalisesti `biome.jsonc`-tiedostoosi:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core"]
}
```

## Yleiskatsaus

Ydinkonfiguraatio sisältää satoja tiukkoja lint-sääntöjä, jotka kattavat:

- **Saavutettavuus (a11y)**: Pakota ARIA-attribuutit, semanttinen HTML ja näppäimistönavigointi
- **Monimutkaisuus**: Vähennä kognitiivista monimutkaisuutta ja pakota parhaat käytännöt
- **Oikeellisuus**: Estä yleiset virheet ja pakota tyyppiturvallisuus
- **Suorituskyky**: Optimoi koodi parempaa ajoaikaa varten
- **Turvallisuus**: Estä turvallisuushaavoittuvuudet
- **Tyyli**: Pakota yhtenäinen koodityyli
- **Epäilyttävät**: Tunnista epäilyttävät kaavat, jotka saattavat viitata bugeihin

## Muotoilijan asetukset

- **Sisennys**: 2 välilyöntiä
- **Rivileveys**: 80 merkkiä
- **Rivin loppu**: LF (Unix-tyyli)
- **Puolipisteet**: Aina vaadittu
- **Lainausmerkit**: Kaksoislainausmerkit JavaScript/TypeScript- ja JSON-tiedostoissa
- **Jälkikäteen lisättävät pilkut**: ES5-tyyli (moniriviset taulukot/oliot)
- **Nuolifunktioiden sulkumerkit**: Sisällytä aina sulkumerkit

## Keskeiset ominaisuudet

### TypeScriptin tiukkuus

Ydinkonfiguraatio ottaa käyttöön TypeScriptin tiukat tarkistukset:

- Ei kannusta `any`-tyypin käyttöön (`noExplicitAny: error`)
- Edellyttää `null/undefined`-arvojen käsittelyä
- Suosii eksplisiittisiä tyyppejä tietyissä tilanteissa
- Pakottaa tyyppiturvallisuuden koko koodikannassa

### ESLintin suositellut vastineet

Yleisimmät parhaat käytäntöjä koskevat säännöt sisältyvät:

- Ei käyttämättömiä muuttujia (`noUnusedVariables: error`)
- Ei käyttämättömiä importteja (`noUnusedImports: error`)
- Ei eksplisiittistä evalia (`noGlobalEval: error`)
- Ei prototype-pollutointia (`noPrototypeBuiltins: error`)

### Saavutettavuus oletuksena

Säännöt, jotka vastaavat saavutettavuusstandardeja:

- ARIA-attribuuttien tulee olla valideja
- Semanttiset HTML-elementit ovat suositeltavia
- Näppäimistönavigoinnin tuki vaaditaan
- Kuville vaaditaan alt-teksti

### Node.js-tuki

Ympäristökohtainen käsittely:

- Tunnistaa Node.js:n globaalit (`module`, `process`)
- Pakottaa `node:`-protokollan sisäänrakennettuihin moduuleihin
- Tukee sekä CommonJS:ää että ESM:ää

## Huomautukset

Tietyt säännöt on otettu pois käytöstä, koska ne ovat liian hitaita Scannerin kanssa suoritettaviksi. Kuten [Biome-dokumentaatiossa](https://biomejs.dev/guides/investigate-slowness/#first-things-first) todetaan, projektikohtaiset säännöt voivat aiheuttaa suorituskykykustannuksia vastineena edistyneemmälle analyysille.

Testauksessamme olemme havainneet, että näiden sääntöjen vaikutus voi olla ero noin 300 ms ja 13 s lintin välillä.
