Core-konfiguraatio tarjoaa kattavan joukon kehyksettömiä lint-sääntöjä ja muotoiluasetuksia. Tämä on perusta, jota kaikki muut kehyskohtaiset konfiguraatiot laajentavat.

## Asennus

Core-konfiguraatio sisällytetään automaattisesti, kun suoritat:

```bash
npx ultracite init
```

Tai lisää se manuaalisesti `biome.jsonc`-tiedostoosi:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core"]
}
```

## Yleiskatsaus

Core-konfiguraatio sisältää satoja tiukkoja lint-sääntöjä, jotka kattavat:

- **Esteettömyys (a11y)**: Varmistaa ARIA-attribuuttien, semanttisen HTML:n ja näppäimistönavigoinnin noudattamisen
- **Monimutkaisuus**: Vähentää kognitiivista monimutkaisuutta ja edistää parhaita käytäntöjä
- **Oikeellisuus**: Estää yleisiä virheitä ja valvoo tyyppiturvallisuutta
- **Suorituskyky**: Optimoi koodia parempaa ajonaikaista suorituskykyä varten
- **Tietoturva**: Estää tietoturva-aukkoja
- **Tyyli**: Varmistaa yhtenäisen koodityylin
- **Epäilyttävät**: Tunnistaa epäilyttäviä kuvioita, jotka voivat viitata virheisiin

## Muotoilijan asetukset

- **Sisennys**: 2 välilyöntiä
- **Rivipituus**: 80 merkkiä
- **Rivin loppu**: LF (Unix-tyyli)
- **Puolipisteet**: Aina vaaditaan
- **Lainausmerkit**: Kaksoislainausmerkit JavaScript/TypeScript- ja JSON-tiedostoissa
- **Viimeiset pilkut**: ES5-tyyli (moniriviset taulukot/oliot)
- **Nuolifunktioiden sulkeet**: Sisällytä sulkeet aina

## Keskeiset ominaisuudet

### TypeScriptin tiukkuus

Core-konfiguraatio ottaa käyttöön TypeScriptin tiukat tarkistukset:

- Ei kannusta käyttämään `any` (`noExplicitAny: error`)
- Edellyttää `null`/`undefined`-arvojen käsittelyä
- Suosisaa eksplisiittisiä tyyppejä tietyissä tilanteissa
- Varmistaa tyyppiturvallisuuden koko koodikannassa

### ESLintin suositellut vastaavat

Yleisiä parhaiden käytäntöjen sääntöjä sisältyy:

- Ei käyttämättömiä muuttujia (`noUnusedVariables: error`)
- Ei käyttämättömiä importteja (`noUnusedImports: error`)
- Ei suoraa eval-kutsua (`noGlobalEval: error`)
- Ei prototype-pollutointia (`noPrototypeBuiltins: error`)

### Esteettömyys oletuksena

Säännöt, jotka vastaavat esteettömyysstandardeja:

- ARIA-attribuuttien tulee olla kelvollisia
- Semanttisia HTML-elementtejä suositellaan
- Näppäimistönavigoinnin tuki vaaditaan
- Kuville vaaditaan alt-teksti

### Node.js-tuki

Ympäristökohtainen käsittely:

- Tunnistaa Node.js:n globaalit (`module`, `process`)
- Pakottaa `node:`-protokollan sisäänrakennettujen moduulien tuonnissa
- Tukee sekä CommonJS:ää että ESM:ää

## Huomautukset

Tietyt säännöt on poistettu käytöstä, koska ne ovat liian hitaita Scannerin kanssa suoritettavaksi. Kuten [Biome-dokumentaatiossa](https://biomejs.dev/guides/investigate-slowness/#first-things-first) todetaan, käytössä olevat projektisäännöt voivat aiheuttaa suorituskykykustannuksia vastineena edistyneemmästä analyysistä.

Testauksessamme olemme havainneet, että nämä säännöt voivat tehdä eron 300 ms:n ja 13 s:n lint-kierroksen välillä.
