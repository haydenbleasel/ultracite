A configuração Vue contém regras de lint específicas do Vue.js para a estrutura de componentes, opções de data, propriedades computadas e palavras-chave reservadas. Inclui também anulações para evitar falsos positivos de lint em ficheiros `.vue`.

## Instalação

Adicione a configuração Vue ao seu `biome.jsonc`:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/vue"]
}
```

## Visão geral

Esta configuração adiciona regras específicas ao desenvolvimento com Vue.js:

- **Suporte HTML**: Ativa suporte HTML experimental completo com indentação de script e style
- **Props Específicas do React**: Proíbe props específicas do React em componentes Vue
- **Anulações para Ficheiros Vue**: Regras relaxadas para ficheiros `.vue` para lidar com a estrutura de Single File Component

## Configuração do Formatador HTML

O preset Vue ativa suporte HTML experimental com formatação especial para scripts e estilos embutidos:

```jsonc
{
  "html": {
    "experimentalFullSupportEnabled": true,
    "formatter": {
      "indentScriptAndStyle": true
    }
  }
}
```

Esta configuração:
- **Suporte Experimental Completo**: Ativa as capacidades experimentais de parsing e formatação HTML do Biome para ficheiros `.vue`
- **Indentar Script e Style**: Indenta corretamente o código dentro das tags `<script>` e `<style>` em Vue Single File Components

## Regras específicas do Vue

### Suspeitas

| Regra | Configuração | Descrição |
|------|---------|-------------|
| `noReactSpecificProps` | `error` | Proíbe props específicas do React como `className` e `htmlFor` no Vue. Use `class` e `for` em vez disso. |

## Anulações para ficheiros Vue

As seguintes regras estão desativadas para ficheiros `.vue` para evitar falsos positivos devido ao suporte parcial do Biome para Vue Single File Components:

### Correção

| Regra | Configuração | Descrição |
|------|---------|-------------|
| `noUnusedVariables` | `off` | Permite variáveis não utilizadas em ficheiros `.vue`. Variáveis na secção script podem ser usadas apenas no template. |
| `noUnusedImports` | `off` | Permite importações não utilizadas em ficheiros `.vue`. Importações podem ser usadas apenas na secção do template. |

### Estilo

| Regra | Configuração | Descrição |
|------|---------|-------------|
| `useConst` | `off` | Permite declarações `let` e `var` em ficheiros `.vue` para padrões de dados reativos. |
| `useImportType` | `off` | Desativa requisitos explícitos de tipo de importação para melhor compatibilidade com o sistema de build do Vue. |
