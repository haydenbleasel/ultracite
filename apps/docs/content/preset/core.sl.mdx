Osnovna konfiguracija Ultracite zagotavlja obsežen nabor pravil za lintanje in nastavitev oblikovanja, neodvisnih od konkretnih ogrodij. To je temelj, od katerega izhajajo vse druge konfiguracije, specifične za posamezna ogrodja.

## Namestitev

Osnovna konfiguracija je samodejno vključena, ko izvedete:

```bash
npx ultracite init
```

Ali jo dodajte ročno v vašo `biome.jsonc`:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core"]
}
```

## Pregled

Osnovna konfiguracija vključuje na stotine strogih pravil za lintanje, ki zajemajo:

- **Dostopnost (a11y)**: Uveljavljajte ARIA atribute, semantični HTML in navigacijo s tipkovnico
- **Kompleksnost**: Zmanjševanje kognitivne kompleksnosti in uveljavljanje najboljših praks
- **Pravilnost**: Preprečevanje pogostih napak in zagotavljanje varnosti tipov
- **Zmogljivost**: Optimizacija kode za boljšo izvedbo v času zagona
- **Varnost**: Preprečevanje varnostnih ranljivosti
- **Slog**: Uveljavljanje doslednega sloga kode
- **Sumljivo**: Zajem sumljivih vzorcev, ki lahko nakazujejo napake

## Nastavitve oblikovalnika

- **Zamik**: 2 presledka
- **Širina vrstice**: 80 znakov
- **Končnica vrstice**: LF (Unix-stil)
- **Podpičja**: Vedno zahtevana
- **Navednice**: Dvokrate za JavaScript/TypeScript in JSON
- **Zamikajoče vejice**: ES5 slog (večvrstične polja/objekti)
- **Oklepaji pri puščičnih funkcijah**: Vedno vključeni

## Ključne funkcije

### Strogost TypeScript-a

Osnovna konfiguracija omogoča stroge kontrole TypeScript-a:

- Odvrača uporabo `any` (`noExplicitAny: error`)
- Zahteva obravnavo `null/undefined`
- Prednost daje eksplicitnim tipom v določenih situacijah
- Uveljavlja varnost tipov po celotni bazi kode

### Priporočeni ekvivalenti ESLint

Vključena so pogosta pravila za najboljše prakse:

- Brez neuporabljenih spremenljivk (`noUnusedVariables: error`)
- Brez neuporabljenih uvozov (`noUnusedImports: error`)
- Brez eksplicitnega eval (`noGlobalEval: error`)
- Brez onesnaževanja prototipa (`noPrototypeBuiltins: error`)

### Dostopnost privzeto

Pravila, ekvivalentna standardom dostopnosti:

- ARIA atributi morajo biti veljavni
- Prednost imajo semantični HTML elementi
- Podpora za navigacijo s tipkovnico je zahtevana
- Alt besedilo za slike je zahtevano

### Podpora za Node.js

Ravnanje specifično za okolje:

- Prepozna Node.js globalne spremenljivke (`module`, `process`)
- Uveljavlja `node:` protokol za vgrajene module
- Podpira tako CommonJS kot ESM

## Opombe

Nekatera pravila so onemogočena, ker so za izvajanje s Scannerjem prepočasna. Kot je omenjeno v [Dokumentacija Biome](https://biomejs.dev/guides/investigate-slowness/#first-things-first), so pravila, omogočena v projektih, znana po tem, da povzročajo performančni overhead v zameno za bolj napredno analizo.

V naših testiranjih smo ugotovili, da ta pravila lahko pomenijo razliko med 300 ms in 13 s pri lintanju.
