Osnovna konfiguracija Ultracite zagotavlja obsežen nabor okvirno-neodvisnih pravil za lintanje in nastavitev formatiranja. To je temelj, na katerem gradijo vse ostale konfiguracije, specifične za posamezne ogrodja.

## Namestitev

Jedrna konfiguracija je samodejno vključena, ko zaženete:

```bash
npx ultracite init
```

Ali jo dodajte ročno v vaš `biome.jsonc`:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core"]
}
```

## Pregled

Jedrna konfiguracija vključuje na stotine strogih pravil za lintanje, ki pokrivajo:

- **Dostopnost (a11y)**: Uveljavi ARIA atribute, semantični HTML in navigacijo s tipkovnico
- **Kompleksnost**: Zmanjšajte kognitivno kompleksnost in uveljavite najboljše prakse
- **Pravilnost**: Preprečite pogoste napake in uveljavite varnost tipov
- **Zmogljivost**: Optimizirajte kodo za boljšo izvedbo med izvajanjem
- **Varnost**: Preprečite varnostne ranljivosti
- **Slog**: Uveljavite dosleden slog kode
- **Sumljivo**: Zaznajte sumljive vzorce, ki lahko kažejo na napake

## Nastavitve formatiranja

- **Vpičkanje**: 2 presledka
- **Širina vrstice**: 80 znakov
- **Konec vrstice**: LF (Unix-stil)
- **Podpičja**: Vedno zahtevana
- **Narekovaji**: Dvokraki narekovaji za JavaScript/TypeScript in JSON
- **Končne vejice**: ES5 slog (večvrstični polja/objekti)
- **Oklepaji pri arrow funkcijah**: Vedno vključeni

## Ključne funkcije

### Strogost TypeScripta

Jedrna konfiguracija omogoča stroge TypeScript preverbe:

- Odsvetuje uporabo `any` (`noExplicitAny: error`)
- Zahteva ravnanje z `null/undefined`
- Prednost daje eksplicitnim tipom v določenih situacijah
- Uveljavlja varnost tipov po celotni kodi

### Priporočeni ekvivalenti ESLint

Vključena so običajna pravila najboljših praks:

- Ni neuporabljenih spremenljivk (`noUnusedVariables: error`)
- Ni neuporabljenih uvozov (`noUnusedImports: error`)
- Ni eksplicitnega evala (`noGlobalEval: error`)
- Brez prototype pollution (`noPrototypeBuiltins: error`)

### Dostopnost privzeto

Pravila, enakovredna standardom dostopnosti:

- ARIA atributi morajo biti veljavni
- Prednost imajo semantični HTML elementi
- Podpora navigaciji s tipkovnico je zahtevana
- Za slike mora obstajati alt besedilo

### Podpora Node.js

Ravnanje, specifično za okolje:

- Prepozna Node.js globalne spremenljivke (`module`, `process`)
- Uveljavlja `node:` protokol za vgrajene module
- Podpira tako CommonJS kot ESM

## Opombe

Nekatera pravila so onemogočena, ker so prepočasi za zagon s Scanner. Kot je navedeno v [Biome dokumentacija](https://biomejs.dev/guides/investigate-slowness/#first-things-first), so v projektu omogočena pravila znana po tem, da povzročajo uspešnostne obremenitve v zameno za bolj napredno analizo.

Pri naših testih smo ugotovili, da ta pravila lahko pomenijo razliko med 300 ms in 13 s pri zagonu linterja.
