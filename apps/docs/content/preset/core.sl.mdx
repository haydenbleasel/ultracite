Jedrna konfiguracija Ultracite zagotavlja obsežen nabor pravil za lintanje in nastavitev oblikovanja, neodvisnih od ogrodja. To je temelj, od katerega izhajajo vse druge konfiguracije, specifične za posamezna ogrodja.

## Namestitev

Jedrna konfiguracija se samodejno vključi, ko zaženete:

```bash
npx ultracite init
```

Ali jo ročno dodajte v vaš `biome.jsonc`:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core"]
}
```

## Pregled

Jedrna konfiguracija vključuje na stotine strogih pravil za lintanje, ki zajemajo:

- **Dostopnost (a11y)**: Uveljavljanje ARIA atributov, semantičnega HTML in navigacije s tipkovnico
- **Kompleksnost**: Zmanjševanje kognitivne kompleksnosti in uveljavljanje najboljših praks
- **Pravilnost**: Preprečevanje pogostih napak in zagotavljanje varnosti tipov
- **Zmogljivost**: Optimizacija kode za boljšo časovno učinkovitost med izvajanjem
- **Varnost**: Preprečevanje varnostnih ranljivosti
- **Slog**: Uveljavljanje konsistentnega sloga kode
- **Sumljivo**: Zaznavanje sumljivih vzorcev, ki lahko nakazujejo napake

## Nastavitve formatiranja

- **Odmik**: 2 presledka
- **Širina vrstice**: 80 znakov
- **Končnica vrstice**: LF (Unix-stil)
- **Podpičja**: Vedno zahtevana
- **Navednice**: Dvojne navednice za JavaScript/TypeScript in JSON
- **Končna vejica**: ES5 slog (večvrstične tabele/objekti)
- **Oklepaji pri arrow funkcijah**: Vedno vključeni

## Ključne funkcije

### Strogost TypeScript-a

Jedrna konfiguracija omogoča stroge TypeScript preverjanja:

- Odsvetuje uporabo `any` (`noExplicitAny: error`)
- Zahteva obravnavo `null/undefined`
- Prednost daje eksplicitnim tipom v določenih situacijah
- Uveljavlja varnost tipov po celotni bazi kode

### Priporočeni ekvivalenti ESLint-a

Vključena so pogosta pravila najboljših praks:

- Ni neuporabljenih spremenljivk (`noUnusedVariables: error`)
- Ni neuporabljenih uvozov (`noUnusedImports: error`)
- Ni eksplicitnega eval (`noGlobalEval: error`)
- Ni prototype pollution (`noPrototypeBuiltins: error`)

### Dostopnost privzeto

Pravila, ekvivalentna standardom dostopnosti:

- ARIA atributi morajo biti veljavni
- Prednost imajo semantični HTML elementi
- Zahtevana podpora navigacije s tipkovnico
- Zahtevano besedilo alt za slike

### Podpora Node.js

Ravnanje, specifično za okolje:

- Prepozna Node.js globalne spremenljivke (`module`, `process`)
- Zahteva `node:` protokol za vgrajene module
- Podpira tako CommonJS kot ESM

## Opombe

Nekatera pravila so onemogočena, ker so prepočasna za zagon s Scannerjem. Kot je omenjeno v [Biome dokumentacija](https://biomejs.dev/guides/investigate-slowness/#first-things-first), so pravila omogočena v projektu znana po tem, da povzročajo dodatno časovno obremenitev v zameno za naprednejšo analizo.

Pri naših testih smo ugotovili, da lahko ta pravila povzročijo razliko med 300 ms in 13 s pri lintanju.
