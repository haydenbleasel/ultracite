Core yapılandırması, çerçeveye bağımsız kapsamlı bir lint kuralları ve biçimlendirme ayarları seti sağlar. Bu, diğer tüm çerçeveye özel yapılandırmaların genişlettiği temel yapılandırmadır.

## Kurulum

Core yapılandırması aşağıdaki komutu çalıştırdığınızda otomatik olarak dahil edilir:

```bash
npx ultracite init
```

Veya elle `biome.jsonc` dosyanıza ekleyin:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core"]
}
```

## Genel Bakış

Core yapılandırması yüzlerce sıkı lint kuralı kapsar:

- **Erişilebilirlik (a11y)**: ARIA özniteliklerini, semantik HTML'i ve klavye gezintisini zorunlu kılar
- **Karmaşıklık**: Bilişsel karmaşıklığı azaltır ve en iyi uygulamaları zorunlu kılar
- **Doğruluk**: Yaygın hataları önler ve tip güvenliğini zorunlu kılar
- **Performans**: Kodun çalışma zamanı performansını optimize eder
- **Güvenlik**: Güvenlik açıklarını önler
- **Stil**: Tutarlı kod stilini zorunlu kılar
- **Şüpheli**: Hatalara işaret edebilecek şüpheli kalıpları yakalar

## Biçimlendirici Ayarları

- **Girintileme**: 2 boşluk
- **Satır Genişliği**: 80 karakter
- **Satır Sonu**: LF (Unix stili)
- **Noktalı Virgül**: Her zaman gerekli
- **Tırnak İşaretleri**: JavaScript/TypeScript ve JSON için çift tırnak
- **Sondaki Virgüller**: ES5 stili (çok satırlı dizi/nesneler)
- **Ok Parantezleri**: Parantezler her zaman dahil edilir

## Temel Özellikler

### TypeScript Sıkı Denetimleri

Core yapılandırması TypeScript'in sıkı kontrollerini etkinleştirir:

- `any` kullanımını caydırır (`noExplicitAny: error`)
- `null/undefined` durumlarının ele alınmasını gerektirir
- Belirli durumlarda açık tipleri tercih eder
- Kod tabanı genelinde tip güvenliğini zorunlu kılar

### ESLint Önerilen Eşdeğerleri

Yaygın en iyi uygulama kuralları dahildir:

- Kullanılmayan değişken yok (`noUnusedVariables: error`)
- Kullanılmayan import yok (`noUnusedImports: error`)
- Açık eval yok (`noGlobalEval: error`)
- Prototype kirliliği yok (`noPrototypeBuiltins: error`)

### Varsayılan Olarak Erişilebilirlik

Erişilebilirlik standartlarına karşılık gelen kurallar:

- ARIA öznitelikleri geçerli olmalıdır
- Semantik HTML öğeleri tercih edilir
- Klavye gezinmesi desteği gereklidir
- Görseller için alt metin gereklidir

### Node.js Desteği

Ortam-a özgü işleme:

- Node.js global değişkenlerini tanır (`module`, `process`)
- Yerleşik modüller için `node:` protokolünü zorunlu kılar
- Hem CommonJS hem de ESM'i destekler
