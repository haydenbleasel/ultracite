The Svelte configuration (`ultracite/svelte`) extends the base Ultracite preset with Svelte-specific rules. Currently, this is a minimal preset as Biome doesn't yet have dedicated Svelte-specific linting rules.

## Installation

Add the Svelte configuration to your `biome.jsonc`:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/svelte"]
}
```

## Overview

This configuration ensures that React-specific props like `className` and `htmlFor` are flagged in Svelte components, as Svelte uses standard HTML attributes (`class` and `for`).

## Current Rules

### Suspicious

| Rule | Setting | Description |
|------|---------|-------------|
| `noReactSpecificProps` | `error` | Disallow React-specific props like `className` and `htmlFor` in Svelte. Use `class` and `for` instead. |

## Usage Examples

### Standard HTML Attributes

```svelte
<!-- Bad: Using React-specific props -->
<script>
  let email = '';
</script>

<div className="container">
  <label htmlFor="email">Email</label>
  <input id="email" type="email" bind:value={email} />
</div>

<!-- Good: Using standard HTML attributes -->
<script>
  let email = '';
</script>

<div class="container">
  <label for="email">Email</label>
  <input id="email" type="email" bind:value={email} />
</div>
```

## Combining with Other Configurations

The Svelte configuration works well with SvelteKit and routing configurations:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/svelte"]
}
```

## Customization

To override or extend the Svelte configuration:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/svelte"],
  "linter": {
    "rules": {
      "suspicious": {
        "noReactSpecificProps": "warn"
      }
    }
  }
}
```
