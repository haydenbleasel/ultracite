A Next.js konfiguráció Next.js-specifikus lintelési szabályokat tartalmaz a képoptimalizálásra, a dokumentumszerkezetre és az App Router mintákra vonatkozóan.

## Telepítés

Adja hozzá a Next.js konfigurációt a `biome.jsonc` fájlhoz:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/next"]
}
```

## Áttekintés

Ez a konfiguráció Next.js-fejlesztésre specifikus szabályokat ad hozzá:

- **Képoptimalizálás**: Előírja a Next.js Image komponens használatát
- **Dokumentumszerkezet**: Megakadályozza a Next.js speciális komponensek nem megfelelő használatát
- **Konfigurációs fájlok**: Speciális kezelés a `next.config.*` fájlokhoz

## Next.js-specifikus szabályok

### Teljesítmény

| Szabály | Beállítás | Leírás |
|------|---------|-------------|
| `noImgElement` | `error` | Nem engedélyezi az `<img>` HTML elem használatát. Az automatikus képoptimalizálás érdekében használja a Next.js `<Image>` komponenst. |

### Stílus

| Szabály | Beállítás | Leírás |
|------|---------|-------------|
| `noHeadElement` | `error` | Nem engedélyezi a `<head>` HTML elem használatát. Használja helyette a Next.js `next/head`-et vagy az App Router metadata API-ját. |

### Gyanús

| Szabály | Beállítás | Leírás |
|------|---------|-------------|
| `noDocumentImportInPage` | `error` | Megakadályozza a `next/document` importálását oldal fájlokban. A `_document.tsx` csak a `pages/_document.tsx`-ben létezhet. |
| `noHeadImportInDocument` | `error` | Megakadályozza a `next/head` importálását `_document.tsx`-ben. Használja helyette a `<Head>`-et a `next/document`-ből. |

## Konfigurációs fájlok felülírása

A Next.js preset speciális kezelést tartalmaz a konfigurációs fájlok és az App Router komponensek számára:

### Next.js konfigurációs fájlok

```jsonc
{
  "overrides": [
    {
      "includes": ["next.config.*"],
      "linter": {
        "rules": {
          "suspicious": {
            "useAwait": "off"
          }
        }
      }
    }
  ]
}
```

Ez lehetővé teszi olyan Next.js konfigurációs mintákat, amelyek tipikusan nem felelnének meg a lintelési szabályoknak.

### App Router oldalak és elrendezések

```jsonc
{
  "overrides": [
    {
      "includes": ["**/page.{ts,tsx,js,jsx}", "**/layout.{ts,tsx,js,jsx}"],
      "linter": {
        "rules": {
          "suspicious": {
            "useAwait": "off"
          }
        }
      }
    }
  ]
}
```

Az App Router oldal- és elrendezés komponensek esetében megengedett, hogy async-ek legyenek await nélkül, mivel a Next.js ezeket külön kezeli szerveroldali renderelés és adatlekérés céljából.
