Konfiguracija Next.js vsebuje specifična pravila za lintanje za optimizacijo slik, strukturo dokumenta in vzorce App Routerja.

## Namestitev

Dodajte Next.js konfiguracijo v vaš `biome.jsonc`:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/next"]
}
```

## Pregled

Ta konfiguracija doda pravila, specifična za razvoj Next.js:

- **Image Optimization**: Zahteva uporabo Next.js Image komponente
- **Document Structure**: Preprečuje nepravilno uporabo posebnih komponent Next.js
- **Configuration Files**: Posebno ravnanje za datoteke `next.config.*`

## Next.js-specifična pravila

### Zmogljivost

| Pravilo | Nastavitev | Opis |
|------|---------|-------------|
| `noImgElement` | `error` | Prepoved uporabe HTML elementa `<img>`. Namesto tega uporabite Next.js `<Image>` komponento za samodejno optimizacijo slik. |

### Slog

| Pravilo | Nastavitev | Opis |
|------|---------|-------------|
| `noHeadElement` | `error` | Prepoved uporabe HTML elementa `<head>`. Namesto tega uporabite Next.js `next/head` ali App Router metadata API. |

### Sumljivo

| Pravilo | Nastavitev | Opis |
|------|---------|-------------|
| `noDocumentImportInPage` | `error` | Preprečite uvoz `next/document` v datotekah strani. `_document.tsx` bi moral obstajati le v `pages/_document.tsx`. |
| `noHeadImportInDocument` | `error` | Preprečite uvoz `next/head` v `_document.tsx`. Namesto tega uporabite `<Head>` iz `next/document`. |

## Prekrite nastavitve konfiguracijskih datotek

Next.js prednastavitev vključuje posebno obravnavo konfiguracijskih datotek in komponent App Routerja:

### Konfiguracijske datoteke Next.js

```jsonc
{
  "overrides": [
    {
      "includes": ["next.config.*"],
      "linter": {
        "rules": {
          "suspicious": {
            "useAwait": "off"
          }
        }
      }
    }
  ]
}
```

To omogoča vzorce konfiguracije Next.js, ki morda običajno ne bi prestali pravil za lintanje.

### Strani in postavitve App Routerja

```jsonc
{
  "overrides": [
    {
      "includes": ["**/page.{ts,tsx,js,jsx}", "**/layout.{ts,tsx,js,jsx}"],
      "linter": {
        "rules": {
          "suspicious": {
            "useAwait": "off"
          }
        }
      }
    }
  ]
}
```

Komponentam strani in postavitvam App Routerja je dovoljeno, da so async brez obveznega `await`, saj Next.js to posebej obravnava za upodabljanje na strežniku in pridobivanje podatkov.
