The Qwik configuration has Qwik-specific linting rules for serialization, visible tasks, lexical scope, and method usage.

## Asennus

Lisää Qwik-konfiguraatio tiedostoon `biome.jsonc`:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/qwik"]
}
```

## Yleiskatsaus

Tämä konfiguraatio lisää Qwik-kehitykseen liittyviä sääntöjä:

- **Serialisointi**: Pakottaa Qwikin resumability-mallin oikean käytön
- **Näkyvät tehtävät**: Estää `useVisibleTask$`-hookin epäasianmukaisen käytön
- **React-spesifiset propit**: Estää React-spesifisten propien käytön Qwik-komponenteissa

## Qwik-kohtaiset säännöt

### Oikeellisuus

| Sääntö | Asetus | Kuvaus |
|-------|--------|--------|
| `noQwikUseVisibleTask` | `error` | Estä `useVisibleTask$`-käyttö. Tämä hook suorittaa koodin välittömästi asiakaspuolella, mikä rikkoo Qwikin resumability-mallin. Käytä tarvittaessa mieluummin `useTask$`- tai `useResource$`-hookia. |
| `useQwikClasslist` | `error` | Pakottaa käyttämään `class`-objektisyntaksia merkkijonojen yhdistämisen sijaan dynaamisissa luokissa Qwikissä. |

### Epäilyttävät

| Sääntö | Asetus | Kuvaus |
|-------|--------|--------|
| `noReactSpecificProps` | `error` | Estää React-spesifisten propien, kuten `className` ja `htmlFor`, käytön Qwikissä. Käytä sen sijaan `class`- ja `for`-ominaisuuksia. |
