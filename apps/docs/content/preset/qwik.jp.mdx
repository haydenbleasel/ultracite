このQwik設定は、シリアライズ、可視タスク、レキシカルスコープ、およびメソッド使用に関するQwik固有のリンティングルールを提供します。

## インストール

Qwik設定をあなたの `biome.jsonc` に追加します:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/qwik"]
}
```

## 概要

この設定はQwik開発に特化したルールを追加します:

- **シリアライズ**: Qwikの再開可能性モデルの適切な使用を強制する
- **可視タスク**: `useVisibleTask$` の不適切な使用を防止する
- **React固有のProps**: Qwikコンポーネント内でReact固有のpropsを禁止する

## Qwik固有のルール

### 正確性

| ルール | 設定 | 説明 |
|------|---------|-------------|
| `noQwikUseVisibleTask` | `error` | `useVisibleTask$` の使用を防止します。 このフックはクライアント上でコードを積極的に実行し、Qwikの再開可能性を損ないます。可能であれば `useTask$` や `useResource$` を代わりに使用してください。 |
| `useQwikClasslist` | `error` | Qwikで動的クラスに対して文字列連結ではなく `class` オブジェクト構文を使用することを強制する。 |

### 疑わしい

| ルール | 設定 | 説明 |
|------|---------|-------------|
| `noReactSpecificProps` | `error` | Qwikで`className`や`htmlFor`のようなReact固有のpropsを禁止します。代わりに `class` や `for` を使用してください。 |
