La configurazione per React include regole di linting specifiche per JSX, per gli hook e per i pattern dei componenti.

## Installazione

Aggiungi la configurazione React a `biome.jsonc`:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/react"]
}
```

## Panoramica

Questa configurazione aggiunge regole specifiche per lo sviluppo con React:

- **Regole JSX**: Far rispettare la sintassi e i pattern JSX corretti
- **Regole per gli Hook**: Assicurare che gli hook di React siano usati correttamente
- **Regole per i componenti**: Applicare le migliori pratiche per i componenti React
- **Gestione dei children**: Evitare un uso improprio della prop children

## Regole specifiche per React

### Correttezza

| Regola | Impostazione | Descrizione |
|------|---------|-------------|
| `noChildrenProp` | `error` | Impedire il passaggio di children come props. I children dovrebbero essere nidificati tra i tag di apertura e chiusura. |
| `noNestedComponentDefinitions` | `error` | Impedire definizioni di componenti all'interno di altri componenti. Questo causa ri-render non necessari. |
| `noReactPropAssignments` | `error` | Impedire la riassegnazione delle props nei componenti React. Le props devono essere trattate come immutabili. |
| `noRenderReturnValue` | `error` | Impedire l'uso del valore di ritorno di `ReactDOM.render()`. |
| `noVoidElementsWithChildren` | `error` | Impedire che elementi void (come `<img>`, `<br>`) abbiano children. |
| `useExhaustiveDependencies` | `error` | Forzare che tutte le dipendenze siano specificate correttamente negli hook React (useEffect, useCallback, useMemo). |
| `useHookAtTopLevel` | `error` | Forzare che tutti gli hook di React siano chiamati dal livello superiore delle funzioni componente, non all'interno di loop, condizioni o funzioni nidificate. |
| `useJsxKeyInIterable` | `error` | Forzare che gli elementi negli iterabili abbiano una prop `key` per la riconciliazione di React. |

### Stile

| Regola | Impostazione | Descrizione |
|------|---------|-------------|
| `useReactFunctionComponents` | `error` | Forzare l'uso di componenti funzionali invece dei componenti di classe in React. |
