Vue の設定には、コンポーネント構造、data オプション、computed プロパティ、予約語に関する Vue.js 固有のリンティングルールが含まれます。 また、`.vue` ファイルで誤検出されるリンティングエラーを防ぐためのオーバーライドも含まれます。

## インストール

Vue 設定をあなたの `biome.jsonc` に追加してください:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/vue"]
}
```

## 概要

この設定は Vue.js 開発に特有のルールを追加します:

- **HTML サポート**: 実験的な完全な HTML サポートを有効にし、script と style のインデントを行います
- **React 固有の Props**: Vue コンポーネント内で React 固有の props を禁止します
- **Vue ファイルのオーバーライド**: 単一ファイルコンポーネント構造を扱うために `.vue` ファイルに対してルールを緩和します

## HTML フォーマッタ設定

Vue プリセットは、埋め込まれた script と style のための特別なフォーマットを伴う実験的な HTML サポートを有効にします:

```jsonc
{
  "html": {
    "experimentalFullSupportEnabled": true,
    "formatter": {
      "indentScriptAndStyle": true
    }
  }
}
```

この設定:
- **実験的な完全サポート**: Biome の実験的な HTML パースおよびフォーマット機能を `.vue` ファイルで有効にします
- **script と style のインデント**: Vue Single File Component の `<script>` と `<style>` タグ内のコードを適切にインデントします

## Vue 固有のルール

### 疑わしい

| ルール | 設定 | 説明 |
|------|---------|-------------|
| `noReactSpecificProps` | `error` | Vue で `className` や `htmlFor` のような React 固有の props を禁止します。代わりに `class` と `for` を使用してください。 |

## Vue ファイルのオーバーライド

Biome の Vue Single File Component に対する部分的なサポートにより誤検出が発生するのを防ぐため、次のルールは `.vue` ファイルでは無効化されています:

### 正確性

| ルール | 設定 | 説明 |
|------|---------|-------------|
| `noUnusedVariables` | `off` | `.vue` ファイルで未使用の変数を許可します。script セクション内の変数は template 内でのみ使用される場合があります。 |
| `noUnusedImports` | `off` | `.vue` ファイルで未使用のインポートを許可します。インポートは template セクションでのみ使用される場合があります。 |

### スタイル

| ルール | 設定 | 説明 |
|------|---------|-------------|
| `useConst` | `off` | リアクティブなデータパターンのために、`.vue` ファイルで `let` と `var` 宣言を許可します。 |
| `useImportType` | `off` | Vue のビルドシステムとの互換性を高めるために、明示的な import 型要件を無効にします。 |
