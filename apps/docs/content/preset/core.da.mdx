Den grundlæggende Ultracite-konfiguration leverer et omfattende sæt ramme-agnostiske lint-regler og formatteringsindstillinger. Dette er fundamentet, som alle andre ramme-specifikke konfigurationer udvider.

## Installation

Kernekonfigurationen inkluderes automatisk, når du kører:

```bash
npx ultracite init
```

Eller tilføj den manuelt til din `biome.jsonc`:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core"]
}
```

## Oversigt

Kernekonfigurationen inkluderer hundreder af strenge lint-regler, der dækker:

- **Tilgængelighed (a11y)**: Håndhæv ARIA-attributter, semantisk HTML og tastaturnavigation
- **Kompleksitet**: Reducer kognitiv kompleksitet og håndhæv bedste praksis
- **Korrekthed**: Forhindre almindelige fejl og håndhæv typesikkerhed
- **Ydeevne**: Optimér kode for bedre runtime-ydeevne
- **Sikkerhed**: Forhindre sikkerhedssårbarheder
- **Stil**: Håndhæv konsekvent kodestil
- **Mistænkeligt**: Fang mistænkelige mønstre, der kan indikere fejl

## Formatteringsindstillinger

- **Indrykning**: 2 mellemrum
- **Linjebredde**: 80 tegn
- **Linjeafslutning**: LF (Unix-stil)
- **Semikolon**: Kræves altid
- **Anførselstegn**: Dobbelte anførselstegn for JavaScript/TypeScript og JSON
- **Sluttende kommaer**: ES5-stil (flere-linje arrays/objekter)
- **Parenteser i arrow-funktioner**: Medtag altid parenteser

## Nøglefunktioner

### TypeScript-strenghed

Kernekonfigurationen aktiverer TypeScripts strenge kontroller:

- Fraråder brug af `any` (`noExplicitAny: error`)
- Kræver håndtering af `null/undefined`
- Foretrækker eksplicitte typer i visse situationer
- Håndhæver typesikkerhed på tværs af kodebasen

### ESLint anbefalede ækvivalenter

Almindelige bedste-praksis-regler er inkluderet:

- Ingen ubrugte variabler (`noUnusedVariables: error`)
- Ingen ubrugte imports (`noUnusedImports: error`)
- Ingen eksplicit eval (`noGlobalEval: error`)
- Ingen prototype-pollution (`noPrototypeBuiltins: error`)

### Tilgængelighed som standard

Regler ækvivalente med tilgængelighedsstandarder:

- ARIA-attributter skal være gyldige
- Semantiske HTML-elementer foretrækkes
- Tastaturnavigation skal understøttes
- Alternativ tekst for billeder kræves

### Node.js-understøttelse

Miljøspecifik håndtering:

- Genkender Node.js-globals (`module`, `process`)
- Håndhæver `node:`-protokol for indbyggede moduler
- Understøtter både CommonJS og ESM

## Bemærkninger

Visse regler er deaktiverede, fordi de er for langsomme at køre med Scanner. Som nævnt i [Biome-dokumentationen](https://biomejs.dev/guides/investigate-slowness/#first-things-first), er aktiverede projektregler kendt for at medføre performance-overhead i bytte for mere avanceret analyse.

I vores test har vi fundet, at disse regler kan gøre forskellen mellem en lint på 300 ms og 13 s.
