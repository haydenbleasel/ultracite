Core Ultracite-konfigurationen leverer et omfattende sæt ramme-uafhængige lint-regler og formatteringsindstillinger. Dette er fundamentet, som alle andre ramme-specifikke konfigurationer bygger videre på.

## Installation

Kernekonfigurationen inkluderes automatisk, når du kører:

```bash
npx ultracite init
```

Eller tilføj den manuelt til din `biome.jsonc`:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core"]
}
```

## Oversigt

Kernekonfigurationen inkluderer hundredvis af strenge lint-regler, der dækker:

- **Tilgængelighed (a11y)**: Håndhæv ARIA-attributter, semantisk HTML og tastaturnavigation
- **Kompleksitet**: Reducer kognitiv kompleksitet og håndhæv bedste praksis
- **Korrekthed**: Forhindre almindelige fejl og håndhæv typesikkerhed
- **Performance**: Optimér kode for bedre kørselstid
- **Sikkerhed**: Forhindre sikkerhedssårbarheder
- **Stil**: Håndhæv konsekvent kodestil
- **Mistænkeligt**: Fang mistænkelige mønstre, der kan indikere bugs

## Formatteringsindstillinger

- **Indrykning**: 2 mellemrum
- **Linjebredde**: 80 tegn
- **Linjeafslutning**: LF (Unix-stil)
- **Semikoloner**: Altid påkrævet
- **Anførselstegn**: Dobbelt anførselstegn for JavaScript/TypeScript og JSON
- **Afsluttende kommaer**: ES5-stil (flere linjer arrays/objekter)
- **Parenteser ved arrow-funktioner**: Medtag altid parenteser

## Nøglefunktioner

### TypeScript-striktheder

Kernekonfigurationen aktiverer TypeScripts strenge kontrol:

- Fraråder brug af `any` (`noExplicitAny: error`)
- Påkræver håndtering af `null/undefined`
- Foretrækker eksplicitte typer i visse situationer
- Håndhæver typesikkerhed på tværs af kodebasen

### ESLint-anbefalede ækvivalenter

Almindelige bedste-praksis regler er inkluderet:

- Ingen ubrugte variabler (`noUnusedVariables: error`)
- Ingen ubrugte imports (`noUnusedImports: error`)
- Ingen eksplicit eval (`noGlobalEval: error`)
- Ingen prototype-pollution (`noPrototypeBuiltins: error`)

### Tilgængelighed som standard

Regler svarende til tilgængelighedsstandarder:

- ARIA-attributter skal være gyldige
- Semantiske HTML-elementer foretrækkes
- Tastaturnavigationsstøtte kræves
- Alt-tekst til billeder kræves

### Node.js-support

Miljøspecifik håndtering:

- Genkender Node.js-globale (`module`, `process`)
- Håndhæver `node:`-protokol for indbyggede moduler
- Understøtter både CommonJS og ESM

## Noter

Visse regler er deaktiveret fordi de er for langsomme at køre med Scanner. Som nævnt i [Biome-dokumentationen](https://biomejs.dev/guides/investigate-slowness/#first-things-first), er projektregler der aktiveres kendt for at forårsage performance-overhead til gengæld for mere avanceret analyse.

I vores tests har vi fundet, at disse regler kan udgøre forskellen mellem 300 ms og 13 s i lint-tid.
