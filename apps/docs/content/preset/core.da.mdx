Kernekonfigurationen for Ultracite leverer et omfattende sæt af ramme-agnostiske lint-regler og formatteringsindstillinger. Dette er fundamentet, som alle andre rammespecifikke konfigurationer bygger videre på.

## Installation

Kernekonfigurationen inkluderes automatisk, når du kører:

```bash
npx ultracite init
```

Eller tilføj den manuelt til din `biome.jsonc`:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core"]
}
```

## Oversigt

Kernekonfigurationen indeholder hundreder af strenge lint-regler, der dækker:

- **Tilgængelighed (a11y)**: Håndhæv ARIA-attributter, semantisk HTML og tastaturnavigation
- **Complexity**: Reducer kognitiv kompleksitet og håndhæv bedste praksis
- **Correctness**: Forhindre almindelige fejl og håndhæv typesikkerhed
- **Performance**: Optimér kode for bedre runtime-performance
- **Security**: Forhindre sikkerheds-sårbarheder
- **Style**: Håndhæv konsekvent kodestil
- **Suspicious**: Fang mistænkelige mønstre, der kan indikere fejl

## Formatteringsindstillinger

- **Indrykning**: 2 mellemrum
- **Linjebredde**: 80 tegn
- **Linjeafslutning**: LF (Unix-stil)
- **Semikoloner**: Kræves altid
- **Anførselstegn**: Dobbelte anførselstegn for JavaScript/TypeScript og JSON
- **Hængende kommaer**: ES5-stil (flere-linjer arrays/objekter)
- **Parenteser i arrow-funktioner**: Inkludér altid parenteser

## Nøglefunktioner

### TypeScript-strenghed

Kernekonfigurationen aktiverer TypeScripts strenge checks:

- Fraråder brug af `any` (`noExplicitAny: error`)
- Kræver håndtering af `null/undefined`
- Foretrækker eksplicitte typer i bestemte situationer
- Håndhæver typesikkerhed på tværs af kodebasen

### ESLint-anbefalede ækvivalenter

Almindelige bedste-praksis-regler medtages:

- Ingen ubrugte variabler (`noUnusedVariables: error`)
- Ingen ubrugte imports (`noUnusedImports: error`)
- Intet eksplicit eval (`noGlobalEval: error`)
- Ingen prototype pollution (`noPrototypeBuiltins: error`)

### Tilgængelighed som standard

Regler svarende til tilgængelighedsstandarder:

- ARIA-attributter skal være gyldige
- Semantiske HTML-elementer foretrækkes
- Tastaturnavigationsstøtte kræves
- Alt-tekst for billeder kræves

### Node.js-support

Miljøspecifik håndtering:

- Genkender Node.js globals (`module`, `process`)
- Håndhæver `node:`-protokol for indbyggede moduler
- Understøtter både CommonJS og ESM

## Noter

Visse regler er deaktiveret, fordi de er for langsomme at køre med Scanner. Som nævnt i [Biome-dokumentationen](https://biomejs.dev/guides/investigate-slowness/#first-things-first), er projektregler, der er aktiveret, kendt for at medføre performance-overhead i bytte for mere avanceret analyse.

I vores test har vi fundet, at disse regler kan betyde forskellen mellem 300ms og 13s lint.
