Az Ultracite alapkonfigurációja átfogó, keretfüggetlen lintelési szabályokat és formázási beállításokat biztosít. Ez az az alap, amelyre minden egyéb, keret-specifikus konfiguráció épül.

## Telepítés

A core konfiguráció automatikusan beillesztődik, amikor futtatod:

```bash
npx ultracite init
```

Vagy add hozzá kézzel a `biome.jsonc`-dhoz:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core"]
}
```

## Áttekintés

A core konfiguráció több száz szigorú lintelési szabályt tartalmaz, amelyek lefedik:

- **Akadálymentesség (a11y)**: Követeli az ARIA attribútumokat, a szemantikus HTML-t és a billentyűzetes navigációt
- **Bonyolultság**: Csökkenti a kognitív komplexitást és érvényesíti a legjobb gyakorlatokat
- **Helyesség**: Megelőzi a gyakori hibákat és biztosítja a típusbiztonságot
- **Teljesítmény**: Optimalizálja a kódot a jobb futásidejű teljesítmény érdekében
- **Biztonság**: Megelőzi a biztonsági sebezhetőségeket
- **Stílus**: Következetes kódstílust ír elő
- **Gyanús**: Elkapja azokat a gyanús mintázatokat, amelyek hibára utalhatnak

## Formázó beállítások

- **Behúzás**: 2 szóköz
- **Sorszélesség**: 80 karakter
- **Sorszakvége**: LF (Unix-stílus)
- **Pontosvesszők**: Mindig kötelezők
- **Idézőjelek**: Dupla idézőjelek JavaScript/TypeScript és JSON esetén
- **Végző vesszők**: ES5 stílus (többsoros tömbök/objektumok)
- **Nyílfüggvény zárójelek**: Mindig tartalmazza a zárójeleket

## Főbb jellemzők

### TypeScript szigorúsága

A core konfiguráció engedélyezi a TypeScript szigorú ellenőrzéseit:

- Nem javasolja az `any` használatát (`noExplicitAny: error`)
- Követeli a `null/undefined` kezelést
- Bizonyos helyzetekben előnyben részesíti a kötelező típusmegadást
- Érvényesíti a típusbiztonságot a teljes kódbázison át

### ESLint javasolt egyenértékek

A gyakori legjobb gyakorlat szabályok benne vannak:

- Nincsenek nem használt változók (`noUnusedVariables: error`)
- Nincsenek nem használt importok (`noUnusedImports: error`)
- Nincs explicit eval (`noGlobalEval: error`)
- Nincs prototype pollution (`noPrototypeBuiltins: error`)

### Alapértelmezett akadálymentesség

A szabályok megfelelnek az akadálymentességi szabványoknak:

- Az ARIA attribútumoknak érvényesnek kell lenniük
- Előnyben részesülnek a szemantikus HTML elemek
- Kötelező a billentyűzetes navigáció támogatása
- Kötelező a képek alternatív szövege

### Node.js támogatás

Környezet-specifikus kezelés:

- Felismeri a Node.js globális változókat (`module`, `process`)
- Követeli a `node:` protokollt beépített modulokhoz
- Támogatja a CommonJS-t és az ESM-et is
