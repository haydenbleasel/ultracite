A core Ultracite konfiguráció átfogó, keretrendszer-független lintelési szabályok és formázási beállítások készletét biztosítja. Ez az alap, amire minden egyéb, keretrendszer-specifikus konfiguráció épül.

## Telepítés

A core konfiguráció automatikusan tartalmazva lesz, amikor futtatod:

```bash
npx ultracite init
```

Vagy add hozzá kézzel a `biome.jsonc` fájlodhoz:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core"]
}
```

## Áttekintés

A core konfiguráció több száz szigorú lintelési szabályt tartalmaz, amelyek lefedik:

- **Hozzáférhetőség (a11y)**: ARIA attribútumok, szemantikus HTML és billentyűzetes navigáció kikényszerítése
- **Komplexitás**: Kognitív komplexitás csökkentése és legjobb gyakorlatok előírása
- **Helyesség**: Gyakori hibák megelőzése és típusbiztonság kikényszerítése
- **Teljesítmény**: Kód optimalizálása a jobb futási teljesítményért
- **Biztonság**: Biztonsági sebezhetőségek megakadályozása
- **Stílus**: Egységes kódstílus előírása
- **Gyanús minták**: Olyan minták felismerése, amelyek hibára utalhatnak

## Formázó beállítások

- **Behúzás**: 2 szóköz
- **Sorhossz**: 80 karakter
- **Soremelés**: LF (Unix-stílus)
- **Pontvesszők**: Mindig kötelezők
- **Idézőjelek**: Dupla idézőjelek JavaScript/TypeScript és JSON esetén
- **Vesszők a sor végén**: ES5 stílus (többsoros tömbök/objektumok)
- **Nyílfüggvény zárójelei**: A zárójelek mindig szerepeljenek

## Főbb jellemzők

### TypeScript szigorúsága

A core konfiguráció engedélyezi a TypeScript szigorú ellenőrzéseit:

- Nem ösztönzi az `any` használatát (`noExplicitAny: error`)
- Megköveteli a `null/undefined` kezelést
- Bizonyos helyzetekben előnyben részesíti a explicit típusokat
- A típusbiztonságot az egész kódbázison kikényszeríti

### ESLint javasolt megfelelői

Gyakori legjobb gyakorlat szabályok szerepelnek:

- Nincsenek használaton kívüli változók (`noUnusedVariables: error`)
- Nincsenek használaton kívüli importok (`noUnusedImports: error`)
- Nincs explicit eval (`noGlobalEval: error`)
- Nincs prototype töltés (`noPrototypeBuiltins: error`)

### Alapértelmezett hozzáférhetőség

A hozzáférhetőségi szabványoknak megfelelő szabályok:

- ARIA attribútumoknak érvényesnek kell lenniük
- Szemantikus HTML elemek előnyben részesítendők
- Kell legyen támogatás a billentyűzetes navigációhoz
- Képekhez alternatív szöveg (alt) szükséges

### Node.js támogatás

Környezetspecifikus kezelés:

- Felismeri a Node.js globálisokat (`module`, `process`)
- Megköveteli a `node:` protokollt beépített moduloknál
- Támogatja mind a CommonJS-t, mind az ESM-et
