A core Ultracite konfiguráció átfogó, keretfüggetlen lintelési szabályok és formázási beállítások készletét biztosítja. Ez az alap, amire minden egyéb, keret-specifikus konfiguráció épül.

## Telepítés

A core konfiguráció automatikusan hozzáadódik, amikor futtatod:

```bash
npx ultracite init
```

Vagy manuálisan add hozzá a `biome.jsonc` fájlodhoz:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core"]
}
```

## Áttekintés

A core konfiguráció számos szigorú lintelési szabályt foglal magában, többek között:

- **Hozzáférhetőség (a11y)**: Kötelező ARIA attribútumok, szemantikus HTML és billentyűzet alapú navigáció
- **Bonyolultság**: Csökkenti a kognitív komplexitást és érvényesíti a bevált gyakorlatokat
- **Helyesség**: Megakadályozza a gyakori hibákat és érvényesíti a típusbiztonságot
- **Teljesítmény**: Optimalizálja a kódot a jobb futásidejű teljesítmény érdekében
- **Biztonság**: Megakadályozza a biztonsági sérülékenységeket
- **Stílus**: Következetes kódstílust érvényesít
- **Gyanús**: Felismeri a hibára utaló gyanús mintákat

## Formázó beállítások

- **Behúzás**: 2 szóköz
- **Sorszélesség**: 80 karakter
- **Sorvég**: LF (Unix-stílus)
- **Pontokként**: Mindig kötelező
- **Idézőjelek**: Dupla idézőjelek JavaScript/TypeScript és JSON esetén
- **Vessző a végén**: ES5 stílus (többsoros tömbök/objektumok)
- **Nyílfüggvény zárójelek**: Mindig tartalmazza a zárójeleket

## Főbb jellemzők

### TypeScript szigorúsága

A core konfiguráció engedélyezi a TypeScript szigorú ellenőrzéseit:

- Kerüli az `any` használatát (`noExplicitAny: error`)
- Kötelező a `null`/`undefined` kezelése
- Bizonyos esetekben előnyben részesíti a explicit típusokat
- Érvényesíti a típusbiztonságot az egész kódbázison

### ESLint ajánlott megfelelői

Gyakori, bevált gyakorlatokra vonatkozó szabályok:

- Nem használt változók tilalma (`noUnusedVariables: error`)
- Nem használt importok tilalma (`noUnusedImports: error`)
- Explicit eval használatának tilalma (`noGlobalEval: error`)
- Prototype szennyezés elleni védelem (`noPrototypeBuiltins: error`)

### Alapértelmezett hozzáférhetőség

A hozzáférhetőségre vonatkozó standardoknak megfelelő szabályok:

- Az ARIA attribútumoknak érvényesnek kell lenniük
- Szemantikus HTML elemek előnyben részesítése
- Billentyűzetes navigáció támogatása kötelező
- Képekhez kötelező az alt szöveg

### Node.js támogatás

Környezetspecifikus kezelés:

- Felismeri a Node.js globális változókat (`module`, `process`)
- Kötelezővé teszi a `node:` protokoll használatát beépített modulokhoz
- Támogatja a CommonJS-t és az ESM-et is
