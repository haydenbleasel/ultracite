A core Ultracite konfiguráció átfogó, keretrendszertől független lintelési szabályokat és formázási beállításokat biztosít. Ez az alap, amire az összes egyéb, konkrét keretrendszerre szabott konfiguráció épül.

## Telepítés

A core konfiguráció automatikusan felkerül, amikor futtatod:

```bash
npx ultracite init
```

Vagy add hozzá kézzel a `biome.jsonc` fájlodhoz:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core"]
}
```

## Áttekintés

A core konfiguráció több száz szigorú lintelési szabályt tartalmaz, amelyek lefedik:

- **Hozzáférhetőség (a11y)**: Kötelező érvényű ARIA attribútumok, szemantikus HTML és billentyűzetes navigáció
- **Komplexitás**: A kognitív komplexitás csökkentése és a legjobb gyakorlatok érvényesítése
- **Helyesség**: Gyakori hibák megelőzése és típusbiztonság érvényesítése
- **Teljesítmény**: A futási teljesítmény optimalizálása
- **Biztonság**: Biztonsági sebezhetőségek megelőzése
- **Stílus**: Összhangban lévő kódstílus érvényesítése
- **Gyanús**: Gyanús minták észlelése, amelyek hibára utalhatnak

## Formázó beállítások

- **Behúzás**: 2 szóköz
- **Sorszélesség**: 80 karakter
- **Sorszakasz vége**: LF (Unix-stílus)
- **Pontosvesszők**: Mindig kötelező
- **Idézőjelek**: Dupla idézőjelek JavaScript/TypeScript és JSON esetén
- **Vessző a sor végén**: ES5-stílus (többsoros tömbök/objektumok)
- **Nyílfüggvény zárójelei**: Mindig tartalmazza a zárójeleket

## Főbb jellemzők

### TypeScript szigorúsága

A core konfiguráció engedélyezi a TypeScript szigorú ellenőrzéseit:

- Kerüli az `any` használatát (`noExplicitAny: error`)
- Kívánja a `null/undefined` kezelését
- Bizonyos helyzetekben előnyben részesíti a explicitt típusokat
- Típusbiztonságot érvényesít az egész kódbázison

### ESLint ajánlott egyenértékei

Gyakori, bevett gyakorlaton alapuló szabályok találhatók benne:

- Nincs fel nem használt változó (`noUnusedVariables: error`)
- Nincsenek fel nem használt importok (`noUnusedImports: error`)
- Nincs explict eval (`noGlobalEval: error`)
- Nincs prototype pollution (`noPrototypeBuiltins: error`)

### Alapértelmezett hozzáférhetőség

A hozzáférhetőségi szabványoknak megfelelő szabályok:

- ARIA attribútumoknak érvényesnek kell lenniük
- Szemantikus HTML elemek használata javasolt
- Billentyűzetes navigáció támogatása kötelező
- Képeknél alt szöveg kötelező

### Node.js támogatás

Környezetspecifikus kezelés:

- Felismeri a Node.js globális változókat (`module`, `process`)
- Kötelező a `node:` protokoll beépített modulokhoz
- Támogatja mind a CommonJS-t, mind az ESM-et
