Kjernekonfigurasjonen for Ultracite gir et omfattende sett med rammeverksuavhengige lint-regler og formatteringsinnstillinger. Dette er fundamentet som alle andre rammeverks-spesifikke konfigurasjoner bygger videre på.

## Installasjon

Kjernekonfigurasjonen inkluderes automatisk når du kjører:

```bash
npx ultracite init
```

Eller legg den til manuelt i din `biome.jsonc`:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core"]
}
```

## Oversikt

Kjernekonfigurasjonen inkluderer hundrevis av strenge lint-regler som dekker:

- **Tilgjengelighet (a11y)**: Håndhev ARIA-attributter, semantisk HTML og tastaturnavigasjon
- **Kompleksitet**: Reduser kognitiv kompleksitet og håndhev beste praksis
- **Korrekthet**: Forhindre vanlige feil og håndhev typesikkerhet
- **Ytelse**: Optimaliser kode for bedre kjøretidsytelse
- **Sikkerhet**: Forhindre sikkerhetssårbarheter
- **Stil**: Håndhev konsekvent kodestil
- **Mistenkelig**: Fang mistenkelige mønstre som kan indikere feil

## Formatteringsinnstillinger

- **Innrykk**: 2 mellomrom
- **Linjebredde**: 80 tegn
- **Linjeslutt**: LF (Unix-stil)
- **Semikolon**: Alltid påkrevd
- **Sitattegn**: Doble anførselstegn for JavaScript/TypeScript og JSON
- **Etterfølgende komma**: ES5-stil (flere-linjers arrays/objekter)
- **Parentese rundt piler**: Inkluder alltid parenteser

## Hovedtrekk

### TypeScript-strenghet

Kjernekonfigurasjonen aktiverer TypeScripts strenge sjekker:

- Fraråder bruk av `any` (`noExplicitAny: error`)
- Krever håndtering av `null/undefined`
- Foretrekker eksplisitte typer i visse situasjoner
- Håndhever typesikkerhet på tvers av kodebasen

### ESLint-anbefalte ekvivalenter

Vanlige beste-praksis-regler er inkludert:

- Ingen ubrukte variabler (`noUnusedVariables: error`)
- Ingen ubrukte imports (`noUnusedImports: error`)
- Ingen eksplisitt eval (`noGlobalEval: error`)
- Ingen prototype-pollution (`noPrototypeBuiltins: error`)

### Tilgjengelighet som standard

Regler tilsvarende tilgjengelighetsstandarder:

- ARIA-attributter må være gyldige
- Semantiske HTML-elementer foretrekkes
- Støtte for tastaturnavigasjon kreves
- Alternativ tekst for bilder kreves

### Node.js-støtte

Miljøspesifikk håndtering:

- Gjenkjenner Node.js-globals (`module`, `process`)
- Håndhever `node:`-protokoll for innebygde moduler
- Støtter både CommonJS og ESM

## Notater

Noen regler er deaktivert fordi de er for trege å kjøre med Scanner. Som nevnt i [Biome-dokumentasjonen](https://biomejs.dev/guides/investigate-slowness/#first-things-first), er prosjektregler som er aktivert kjent for å gi ytelseskostnader i bytte mot mer avansert analyse.

I våre tester har vi funnet at disse reglene kan utgjøre forskjellen mellom 300 ms og 13 s lint-tid.
