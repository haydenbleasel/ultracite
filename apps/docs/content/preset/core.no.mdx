Kjernekonfigurasjonen for Ultracite gir et omfattende sett med rammeverksuavhengige lint-regler og formatteringsinnstillinger. Dette er grunnlaget som alle andre rammeverksspesifikke konfigurasjoner bygger på.

## Installasjon

Kjernekonfigurasjonen inkluderes automatisk når du kjører:

```bash
npx ultracite init
```

Eller legg den manuelt til i din `biome.jsonc`:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core"]
}
```

## Oversikt

Kjernekonfigurasjonen inneholder hundrevis av strenge lint-regler som dekker:

- **Tilgjengelighet (a11y)**: Påse ARIA-attributter, semantisk HTML og tastaturnavigasjon
- **Kompleksitet**: Reduser kognitiv kompleksitet og håndhev beste praksis
- **Korrekthet**: Forhindre vanlige feil og håndhev typesikkerhet
- **Ytelse**: Optimaliser kode for bedre kjøretid
- **Sikkerhet**: Forhindre sikkerhetssårbarheter
- **Stil**: Håndhev konsekvent kodestil
- **Mistenkelige**: Fang mistenkelige mønstre som kan indikere feil

## Formatteringsinnstillinger

- **Innrykk**: 2 mellomrom
- **Linjebredde**: 80 tegn
- **Linjeavslutning**: LF (Unix‑stil)
- **Semikolon**: Alltid påkrevd
- **Anførselstegn**: Dobbeltsitat for JavaScript/TypeScript og JSON
- **Hengende komma**: ES5‑stil (flere linjer i arrays/objekter)
- **Parenteser for arrow-funksjoner**: Inkluder alltid parenteser

## Nøkkelfunksjoner

### TypeScript-strenghet

Kjernekonfigurasjonen aktiverer TypeScripts strenge sjekker:

- Fraråder bruk av `any` (`noExplicitAny: error`)
- Krever håndtering av `null/undefined`
- Foretrekker eksplisitte typer i visse situasjoner
- Håndhever typesikkerhet på tvers av kodebasen

### ESLint anbefalte ekvivalenter

Vanlige beste-praksis regler er inkludert:

- Ingen ubrukte variabler (`noUnusedVariables: error`)
- Ingen ubrukte imports (`noUnusedImports: error`)
- Ingen eksplisitt eval (`noGlobalEval: error`)
- Ingen prototype-pollution (`noPrototypeBuiltins: error`)

### Tilgjengelighet som standard

Regler ekvivalente med tilgjengelighetsstandarder:

- ARIA‑attributter må være gyldige
- Semantiske HTML‑elementer foretrukket
- Tastaturnavigasjon må støttes
- Alternativ tekst for bilder er påkrevd

### Node.js‑støtte

Miljøspesifikk håndtering:

- Gjenkjenner Node.js‑globale (`module`, `process`)
- Håndhever `node:`‑protokoll for innebygde moduler
- Støtter både CommonJS og ESM

## Notater

Visse regler er deaktivert fordi de er for trege å kjøre med Scanner. Som omtalt i [Biome‑dokumentasjonen](https://biomejs.dev/guides/investigate-slowness/#first-things-first), er det kjent at prosjektregler som er aktivert kan medføre ytelseskostnad i bytte for mer avansert analyse.

I våre tester har vi funnet at disse reglene kan utgjøre forskjellen mellom 300 ms og 13 s lint.
