Kjernekonfigurasjonen for Ultracite gir et omfattende sett med rammeverksuavhengige linteregler og formatteringsinnstillinger. Dette er grunnlaget som alle andre rammeverks-spesifikke konfigurasjoner bygger videre på.

## Installasjon

Kjernekonfigurasjonen inkluderes automatisk når du kjører:

```bash
npx ultracite init
```

Eller legg den til manuelt i din `biome.jsonc`:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core"]
}
```

## Oversikt

Kjernekonfigurasjonen inneholder hundrevis av strenge linteregler som dekker:

- **Tilgjengelighet (a11y)**: Håndhev ARIA-attributter, semantisk HTML og tastaturnavigasjon
- **Kompleksitet**: Reduser kognitiv kompleksitet og håndhev beste praksis
- **Korrekthet**: Forebygg vanlige feil og håndhev typesikkerhet
- **Ytelse**: Optimaliser kode for bedre kjøretidsytelse
- **Sikkerhet**: Forhindre sikkerhetssårbarheter
- **Stil**: Håndhev konsekvent kodestil
- **Mistenkelig**: Fang mistenkelige mønstre som kan indikere feil

## Formatteringsinnstillinger

- **Innrykk**: 2 mellomrom
- **Linjebredde**: 80 tegn
- **Linjeavslutning**: LF (Unix-stil)
- **Semikoloner**: Alltid påkrevd
- **Anførselstegn**: Dobbeltsitater for JavaScript/TypeScript og JSON
- **Etterslepende kommaer**: ES5-stil (flere-linjers arrays/objekter)
- **Parenteser ved piler**: Alltid inkluder parenteser

## Viktige funksjoner

### TypeScript-strenghet

Kjernekonfigurasjonen aktiverer TypeScripts strenge kontroller:

- Fraråder bruk av `any` (`noExplicitAny: error`)
- Krever håndtering av `null/undefined`
- Foretrekker eksplisitte typer i visse situasjoner
- Håndhever typesikkerhet på tvers av kodebasen

### ESLint-anbefalte ekvivalenter

Vanlige praksisregler er inkludert:

- Ingen ubrukte variabler (`noUnusedVariables: error`)
- Ingen ubrukte imports (`noUnusedImports: error`)
- Ingen eksplisitt eval (`noGlobalEval: error`)
- Ingen prototype-pollution (`noPrototypeBuiltins: error`)

### Tilgjengelighet som standard

Regler tilsvarende tilgjengelighetsstandarder:

- ARIA-attributter må være gyldige
- Semantiske HTML-elementer foretrukket
- Støtte for tastaturnavigasjon kreves
- Alternativ tekst for bilder kreves

### Node.js-støtte

Miljøspesifikk håndtering:

- Gjenkjenner Node.js-globals (`module`, `process`)
- Håndhever `node:`-protokoll for innebygde moduler
- Støtter både CommonJS og ESM

## Notater

Visse regler er deaktivert fordi de er for trege å kjøre med Scanner. Som nevnt i [Biome-dokumentasjonen](https://biomejs.dev/guides/investigate-slowness/#first-things-first), er det kjent at prosjektregler som er aktivert kan forårsake ytelsesoverhead i bytte mot mer avansert analyse.

I våre tester har vi funnet at disse reglene kan utgjøre forskjellen mellom 300 ms og 13 s i linting.
