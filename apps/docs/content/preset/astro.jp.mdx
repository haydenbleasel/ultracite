Astro の設定には、特にフロントマターのスクリプトやコンポーネント構文周りの独自の構造を扱うための、Astro 固有のリンティングルールの調整が含まれます。

## インストール

Astro の設定をあなたの `biome.jsonc` に追加します:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/astro"]
}
```

## 概要

この設定は Astro 開発のためにルールを調整します:

- **未使用の変数/インポート**: フロントマターの制約により、変数がテンプレート内でのみ使用される場合があるため、Astro ファイルでの未使用の変数やインポートを許可します
- **変数宣言**: Astro のフロントマターでは再代入可能な変数が必要になる場合があるため、`const` の強制を緩和します
- **インポートの型指定**: 互換性向上のため、明示的な import type の要件を無効にします
- **React 固有の Props**: Astro は標準の HTML 属性を使用するため、`className` のような React 固有のプロパティをフラグにします

## Astro 固有のルール

### 正確性

| ルール | 設定 | 説明 |
|------|---------|-------------|
| `noUnusedVariables` | `off` | Astro ファイルでの未使用の変数を許可します。フロントマターで宣言された変数はテンプレートセクションでのみ使用される場合があります。 |
| `noUnusedImports` | `off` | Astro ファイルでの未使用のインポートを許可します。フロントマターのインポートはテンプレートセクションでのみ使用される場合があります。 |

### スタイル

| ルール | 設定 | 説明 |
|------|---------|-------------|
| `useConst` | `off` | `let` および `var` 宣言を許可します。Astro コンポーネントのフロントマターでは再代入可能な変数が必要になる場合があります。 |
| `useImportType` | `off` | Astro のビルドシステムとの互換性を高めるために、明示的な import type の要件を無効にします。 |

### 疑わしい

| ルール | 設定 | 説明 |
|------|---------|-------------|
| `noReactSpecificProps` | `error` | Astro では `className` や `htmlFor` のような React 固有のプロパティを禁止します。代わりに `class` や `for` を使用してください。 |
