Astro-konfigurasjonen har Astro-spesifikke justeringer av lint-regler for å håndtere den unike strukturen til Astro-komponenter, spesielt rundt frontmatter-skript og komponent-syntaks.

## Installasjon

Legg til Astro-konfigurasjonen i din `biome.jsonc`:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/astro"]
}
```

## Oversikt

Denne konfigurasjonen justerer regler for Astro-utvikling:

- **Ubrukte variabler/importer**: Tillater ubrukte variabler og importer i Astro-filer på grunn av begrensninger i frontmatter hvor variabler kanskje bare brukes i malseksjonen
- **Variabeldeklarasjoner**: Leter på `const`-håndhevelsen siden Astro-frontmatter kan kreve variabler som kan tildeles på nytt
- **Importtyper**: Deaktiverer eksplisitte krav til import-typer for bedre kompatibilitet
- **React-spesifikke props**: Marker React-spesifikke props som `className` siden Astro bruker standard HTML-attributter

## Astro-spesifikke regler

### Korrekthet

| Regel | Innstilling | Beskrivelse |
|------|---------|-------------|
| `noUnusedVariables` | `off` | Tillat ubrukte variabler i Astro-filer. Variabler deklarert i frontmatter kan kun brukes i malseksjonen. |
| `noUnusedImports` | `off` | Tillat ubrukte importer i Astro-filer. Importer i frontmatter kan kun brukes i malseksjonen. |

### Stil

| Regel | Innstilling | Beskrivelse |
|------|---------|-------------|
| `useConst` | `off` | Tillat `let` og `var` deklarasjoner. Astro-komponentfrontmatter kan kreve variabler som kan tildeles på nytt. |
| `useImportType` | `off` | Deaktiver eksplisitte krav til import-typer for bedre kompatibilitet med Astros byggesystem. |

### Mistenkelige

| Regel | Innstilling | Beskrivelse |
|------|---------|-------------|
| `noReactSpecificProps` | `error` | Ikke tillat React-spesifikke props som `className` og `htmlFor` i Astro. Bruk `class` og `for` i stedet. |
