La configuración de React incluye reglas de linting específicas para JSX, hooks y patrones de componentes.

## Instalación

Agrega la configuración de React a tu `biome.jsonc`:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/react"]
}
```

## Descripción general

Esta configuración añade reglas específicas para el desarrollo con React:

- **Reglas de JSX**: Hacer cumplir la sintaxis y los patrones correctos de JSX
- **Reglas de Hooks**: Asegurar que los hooks de React se usen correctamente
- **Reglas de Componentes**: Hacer cumplir las mejores prácticas para componentes de React
- **Manejo de children**: Evitar el uso incorrecto de la prop `children`

## Reglas específicas de React

### Corrección

| Regla | Configuración | Descripción |
|------|---------|-------------|
| `noChildrenProp` | `error` | Evitar pasar la prop `children`. Los `children` deben anidarse entre las etiquetas de apertura y cierre. |
| `noNestedComponentDefinitions` | `error` | Evitar definiciones de componentes dentro de otros componentes. Esto causa re-renderizaciones innecesarias. |
| `noReactPropAssignments` | `error` | Evitar reasignar `props` en componentes React. Las `props` deben tratarse como inmutables. |
| `noRenderReturnValue` | `error` | Evitar el uso del valor retornado por `ReactDOM.render()`. |
| `noVoidElementsWithChildren` | `error` | Evitar que elementos void (como `<img>`, `<br>`) tengan `children`. |
| `useExhaustiveDependencies` | `error` | Exigir que todas las dependencias estén especificadas correctamente en los hooks de React (useEffect, useCallback, useMemo). |
| `useHookAtTopLevel` | `error` | Exigir que todos los hooks de React se llamen desde el nivel superior de las funciones de los componentes, no dentro de bucles, condicionales o funciones anidadas. |
| `useJsxKeyInIterable` | `error` | Exigir que los elementos en iterables tengan una prop `key` para la reconciliación de React. |

### Estilo

| Regla | Configuración | Descripción |
|------|---------|-------------|
| `useReactFunctionComponents` | `error` | Exigir el uso de componentes funcionales en lugar de componentes de clase en React. |

### Sospechosas

| Regla | Configuración | Descripción |
|------|---------|-------------|
| `noArrayIndexKey` | `error` | Evitar usar índices de array como `key`. Los índices no son identificadores estables y pueden causar problemas con el estado del componente. |
| `noDuplicateJsxProps` | `error` | Evitar propiedades duplicadas en JSX. |
| `noSuspiciousSemicolonInJsx` | `error` | Evitar puntos y comas que cambien la semántica de elementos JSX. |
| `noReactSpecificProps` | `off` | Permitir props específicas de React como `className` y `htmlFor`. |
