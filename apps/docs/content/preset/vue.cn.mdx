Vue 配置包含针对组件结构、data 选项、computed 属性和保留关键字的 Vue.js 特定 lint 规则。它还包括针对 `.vue` 文件的覆盖以防止误报的 lint 错误。

## 安装

将 Vue 配置添加到你的 `biome.jsonc` 中：

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/vue"]
}
```

## 概览

此配置添加了针对 Vue.js 开发的特定规则：

- **HTML 支持**: 启用实验性的完整 HTML 支持，并对 script 和 style 缩进进行处理
- **React 特定 Props**: 禁止在 Vue 组件中使用 React 特定的 props
- **Vue 文件覆盖**: 对 `.vue` 文件放宽规则以处理单文件组件结构

## HTML 格式化器配置

Vue 预设启用了实验性的 HTML 支持，并对嵌入的脚本和样式使用特殊格式化：

```jsonc
{
  "html": {
    "experimentalFullSupportEnabled": true,
    "formatter": {
      "indentScriptAndStyle": true
    }
  }
}
```

此配置：
- **实验性完整支持**: 为 `.vue` 文件启用 Biome 的实验性 HTML 解析和格式化功能
- **缩进脚本和样式**: 正确缩进 Vue 单文件组件中 `<script>` 和 `<style>` 标签内的代码

## Vue 特定规则

### 可疑

| 规则 | 设置 | 描述 |
|------|---------|-------------|
| `noReactSpecificProps` | `error` | 禁止在 Vue 中使用像 `className` 和 `htmlFor` 这样的 React 特定 props。请改用 `class` 和 `for`。 |

## Vue 文件覆盖

以下规则在 `.vue` 文件中被禁用，以防止由于 Biome 对 Vue 单文件组件的部分支持而导致的误报：

### 正确性

| 规则 | 设置 | 描述 |
|------|---------|-------------|
| `noUnusedVariables` | `off` | 允许在 `.vue` 文件中存在未使用的变量。脚本部分中的变量可能仅在模板中使用。 |
| `noUnusedImports` | `off` | 允许在 `.vue` 文件中存在未使用的导入。导入可能仅在模板部分使用。 |

### 样式

| 规则 | 设置 | 描述 |
|------|---------|-------------|
| `useConst` | `off` | 允许在 `.vue` 文件中使用 `let` 和 `var` 声明以支持响应式数据模式。 |
| `useImportType` | `off` | 为了更好地兼容 Vue 的构建系统，禁用对显式 import type 要求的检查。 |
