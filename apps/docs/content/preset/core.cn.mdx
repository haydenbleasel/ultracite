Ultracite 的核心配置提供了一套全面的、与框架无关的 linting 规则和格式化设置。这是所有其他特定框架配置扩展的基础。

## 安装

当您运行以下命令时，核心配置会自动包含在内：

```bash
npx ultracite init
```

或者手动将其添加到您的 `biome.jsonc` 中：

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core"]
}
```

## 概览

核心配置包含数百条严格的 lint 规则，覆盖：

- **可访问性 (a11y)**：强制使用 ARIA 属性、语义化 HTML 和键盘导航
- **复杂度**：降低认知复杂度并强制执行最佳实践
- **正确性**：防止常见错误并强制类型安全
- **性能**：优化代码以获得更好的运行时性能
- **安全**：防止安全漏洞
- **风格**：强制一致的代码风格
- **可疑**：捕获可能表明存在 bug 的可疑模式

## 格式化器设置

- **缩进**：2 个空格
- **最大行宽**：80 个字符
- **行结束符**：LF（Unix 风格）
- **分号**：始终需要
- **引号**：JavaScript/TypeScript 与 JSON 使用双引号
- **尾随逗号**：ES5 风格（多行数组/对象）
- **箭头函数括号**：始终包含括号

## 主要特性

### TypeScript 严格性

核心配置启用 TypeScript 的严格检查：

- 不鼓励使用 `any`（`noExplicitAny: error`）
- 要求处理 `null/undefined`
- 在某些情况下偏好显式类型
- 在整个代码库中强制类型安全

### 等价的 ESLint 推荐规则

包含常见的最佳实践规则：

- 禁止未使用的变量（`noUnusedVariables: error`）
- 禁止未使用的导入（`noUnusedImports: error`）
- 禁止显式 eval（`noGlobalEval: error`）
- 禁止原型污染（`noPrototypeBuiltins: error`）

### 默认可访问性

等同于可访问性标准的规则：

- ARIA 属性必须有效
- 优先使用语义化 HTML 元素
- 需要支持键盘导航
- 图片必须包含替代文本

### Node.js 支持

环境特定的处理：

- 识别 Node.js 全局变量（`module`, `process`）
- 强制对内置模块使用 `node:` 协议
- 支持 CommonJS 和 ESM 两种模块系统

## 备注

某些规则已被禁用，因为它们对于 Scanner 来说运行速度太慢。正如在 [Biome 文档](https://biomejs.dev/guides/investigate-slowness/#first-things-first) 中所述，已启用的项目规则已知会带来性能开销，以换取更高级的分析能力。

在我们的测试中，我们发现这些规则可能会导致 lint 时间在 300ms 和 13s 之间产生差异。
