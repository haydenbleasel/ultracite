核心 Ultracite 配置提供了一套全面的与框架无关的 lint 规则和格式化设置。这是所有其他特定框架配置扩展的基础。

## 安装

核心配置会在你运行时自动包含：

```bash
npx ultracite init
```

或者手动将其添加到你的 `biome.jsonc` 中：

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core"]
}
```

## 概览

核心配置包括数百条严格的 lint 规则，涵盖：

- **无障碍（a11y）**: 强制 ARIA 属性、语义化 HTML 和键盘导航
- **复杂度**: 降低认知复杂性并强制最佳实践
- **正确性**: 防止常见错误并强制类型安全
- **性能**: 优化代码以获得更好的运行时性能
- **安全**: 防止安全漏洞
- **风格**: 强制一致的代码风格
- **可疑模式**: 捕获可能表明缺陷的可疑模式

## 格式化设置

- **缩进**: 2 个空格
- **行宽**: 80 个字符
- **行结束符**: LF（Unix 风格）
- **分号**: 始终要求
- **引号**: JavaScript/TypeScript 和 JSON 使用双引号
- **尾随逗号**: ES5 风格（多行数组/对象）
- **箭头函数括号**: 始终包含括号

## 主要特性

### TypeScript 严格性

核心配置启用了 TypeScript 的严格检查：

- 不鼓励使用 `any`（`noExplicitAny: error`）
- 要求处理 `null/undefined`
- 在某些情况下偏好显式类型
- 在整个代码库中强制类型安全

### ESLint 推荐等价项

包含常见的最佳实践规则：

- 不允许未使用的变量（`noUnusedVariables: error`）
- 不允许未使用的导入（`noUnusedImports: error`）
- 不允许显式 eval（`noGlobalEval: error`）
- 不允许原型污染（`noPrototypeBuiltins: error`）

### 默认可访问性

相当于可访问性标准的规则：

- 必须使 ARIA 属性有效
- 优先使用语义化 HTML 元素
- 要求支持键盘导航
- 要求图像的替代文本

### Node.js 支持

与环境相关的处理：

- 识别 Node.js 全局变量（`module`、`process`）
- 强制对内置模块使用 `node:` 协议
- 同时支持 CommonJS 和 ESM

## 注意事项

某些规则因与 Scanner 一起运行过慢而被禁用。正如 [Biome 文档](https://biomejs.dev/guides/investigate-slowness/#first-things-first) 所述，启用的项目规则在换取更高级分析的同时会带来性能开销。

在我们的测试中，我们发现这些规则可能会使 lint 的运行时间从 300ms 增加到 13s。
