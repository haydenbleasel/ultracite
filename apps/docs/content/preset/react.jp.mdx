この React 設定は、JSX、hooks、およびコンポーネントパターンに対する React 固有のリンティングルールを提供します。

## インストール

React 設定をあなたの `biome.jsonc` に追加します:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/react"]
}
```

## 概要

この設定は React 開発に特有のルールを追加します:

- **JSX ルール**: 適切な JSX 構文とパターンを強制します
- **Hooks ルール**: React の hooks が正しく使用されることを保証します
- **コンポーネント ルール**: React コンポーネントのベストプラクティスを強制します
- **children の取り扱い**: 不適切な children prop の使用を防ぎます

## React 固有のルール

### 正確性

| ルール | 設定 | 説明 |
|------|---------|-------------|
| `noChildrenProp` | `error` | props として children を渡すことを防ぎます。children は開閉タグの間にネストされるべきです。 |
| `noNestedComponentDefinitions` | `error` | コンポーネント内で別のコンポーネント定義を行うことを防ぎます。これは不要な再レンダーを引き起こします。 |
| `noReactPropAssignments` | `error` | React コンポーネント内で props を再代入することを防ぎます。props は不変として扱うべきです。 |
| `noRenderReturnValue` | `error` | `ReactDOM.render()` の戻り値の使用を防ぎます。 |
| `noVoidElementsWithChildren` | `error` | void 要素（例: `<img>`, `<br>`）が children を持つことを防ぎます。 |
| `useExhaustiveDependencies` | `error` | React の hooks 内で全ての依存関係が正しく指定されていることを強制します（useEffect、useCallback、useMemo）。 |
| `useHookAtTopLevel` | `error` | 全ての React hooks がコンポーネント関数のトップレベルから呼び出され、ループや条件分岐、入れ子関数の内部から呼び出されないことを強制します。 |
| `useJsxKeyInIterable` | `error` | イテラブル内の要素が React の再調整のために `key` prop を持つことを強制します。 |

### スタイル

| ルール | 設定 | 説明 |
|------|---------|-------------|
| `useReactFunctionComponents` | `error` | React でクラスコンポーネントより関数コンポーネントを使用することを強制します。 |

### 疑わしい

| ルール | 設定 | 説明 |
|------|---------|-------------|
| `noArrayIndexKey` | `error` | 配列のインデックスをキーとして使用することを防ぎます。配列のインデックスは安定した識別子ではなく、コンポーネントの状態に問題を引き起こす可能性があります。 |
| `noDuplicateJsxProps` | `error` | JSX 内で重複したプロパティが存在することを防ぎます。 |
| `noSuspiciousSemicolonInJsx` | `error` | JSX 要素の意味を変えてしまう疑わしいセミコロンを防ぎます。 |
| `noReactSpecificProps` | `off` | `className` や `htmlFor` のような React 固有のプロパティを許可します。 |
