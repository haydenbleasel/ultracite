Die Next.js-Konfiguration enthält Next.js-spezifische Linting-Regeln für Bildoptimierung, Dokumentstruktur und App Router-Muster.

## Installation

Fügen Sie die Next.js-Konfiguration zu Ihrer `biome.jsonc` hinzu:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/next"]
}
```

## Übersicht

Diese Konfiguration fügt Regeln hinzu, die speziell für die Next.js-Entwicklung gelten:

- **Bildoptimierung**: Durchsetzung der Verwendung der Next.js Image-Komponente
- **Dokumentstruktur**: Verhindern der unsachgemäßen Verwendung spezieller Next.js-Komponenten
- **Konfigurationsdateien**: Spezielle Behandlung von `next.config.*`-Dateien

## Next.js-spezifische Regeln

### Leistung

| Regel | Einstellung | Beschreibung |
|------|---------|-------------|
| `noImgElement` | `error` | Die Verwendung des HTML-Elements `<img>` ist nicht erlaubt. Verwenden Sie stattdessen die Next.js-Komponente `<Image>` für automatische Bildoptimierung. |

### Stil

| Regel | Einstellung | Beschreibung |
|------|---------|-------------|
| `noHeadElement` | `error` | Die Verwendung des HTML-Elements `<head>` ist nicht erlaubt. Verwenden Sie stattdessen `next/head` oder die Metadata-API des App Router. |

### Verdächtig

| Regel | Einstellung | Beschreibung |
|------|---------|-------------|
| `noDocumentImportInPage` | `error` | Verhindern, dass `next/document` in Seiten-Dateien importiert wird. `_document.tsx` darf nur in `pages/_document.tsx` existieren. |
| `noHeadImportInDocument` | `error` | Verhindern, dass `next/head` in `_document.tsx` importiert wird. Verwenden Sie stattdessen `<Head>` aus `next/document`. |

## Überschreibungen für Konfigurationsdateien

Das Next.js-Preset enthält spezielle Behandlung für Konfigurationsdateien und App Router-Komponenten:

### Next.js-Konfigurationsdateien

```jsonc
{
  "overrides": [
    {
      "includes": ["next.config.*"],
      "linter": {
        "rules": {
          "suspicious": {
            "useAwait": "off"
          }
        }
      }
    }
  ]
}
```

Dies ermöglicht Next.js-Konfigurationsmuster, die normalerweise nicht die Linting-Regeln bestehen würden.

### App Router-Seiten und Layouts

```jsonc
{
  "overrides": [
    {
      "includes": ["**/page.{ts,tsx,js,jsx}", "**/layout.{ts,tsx,js,jsx}"],
      "linter": {
        "rules": {
          "suspicious": {
            "useAwait": "off"
          }
        }
      }
    }
  ]
}
```

App Router-Seiten- und Layout-Komponenten dürfen asynchron sein, ohne dass `await` erforderlich ist, da Next.js diese speziell für serverseitiges Rendering und Datenabruf behandelt.
