Ultracite fornece integração perfeita com assistentes de codificação por IA gerando automaticamente ficheiros de regras específicos para cada editor. Essas regras ajudam a orientar os assistentes de codificação por IA para escrever código melhor e mais consistente que segue as melhores práticas.

As regras do editor funcionam em conjunto com o linting e a formatação do Biome. Enquanto o Biome lida com a formatação do código e detecta erros de sintaxe, as regras do editor orientam os assistentes de codificação por IA a escrever código melhor desde o início, reduzindo a necessidade de correções posteriores.

Essa abordagem dupla garante tanto qualidade de código automatizada (através do Biome) quanto geração inteligente de código (através de regras de codificação por IA).

## Configuração

### Configuração automática

When you run `npx ultracite init`, you'll be prompted to select which editor rules you want to enable:

```bash
? Which editor rules do you want to enable (optional)?
  ◯ GitHub Copilot (VSCode)
  ◯ Cursor
  ◯ Windsurf
  # ... etcetera
```

This will create the relevant file in your project directory.

### Configuração manual

Se você quiser configurar manualmente, crie o ficheiro relevante no diretório do seu projeto (por exemplo `.cursor/rules` para o Cursor ou `.rules` para o Zed). Depois, copie e cole [este conteúdo](https://github.com/haydenbleasel/ultracite/tree/main/.cursor/rules/ultracite.mdc) no ficheiro.

## Vantagens

Usar as regras do editor do Ultracite oferece várias vantagens:

- **Consistência**: Todo o código gerado por IA segue as mesmas convenções
- **Qualidade**: As regras reforçam melhores práticas e previnem erros comuns
- **Acessibilidade**: Diretrizes incorporadas de acessibilidade garantem código inclusivo
- **Desempenho**: As regras promovem padrões eficientes e evitam anti-padrões
- **Segurança**: As diretrizes ajudam a prevenir vulnerabilidades de segurança comuns

## Personalização

Embora as regras padrão sejam abrangentes, você pode personalizá-las para as suas necessidades específicas:

1. **Modificar regras existentes**: Edite os ficheiros de regras diretamente
2. **Adicionar regras específicas do projeto**: Acrescente diretrizes adicionais
3. **Desativar regras**: Remova ou comente as regras que não se aplicam ao seu projeto
