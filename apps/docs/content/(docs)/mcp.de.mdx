Das **Modell‑Kontext‑Protokoll (MCP)** ist ein offener Standard, der es KI‑Assistenten wie Claude, Cursor und anderen KI‑gestützten Werkzeugen ermöglicht, sich sicher mit externen Datenquellen und Systemen zu verbinden. Man kann es sich wie eine "Universalfernbedienung" vorstellen, mit der Ihre KI‑Tools auf reale Daten und Funktionalität zugreifen können.

Ultracite unterstützt MCP, um Ihren KI‑Entwicklungsworkflow zu beschleunigen.

## Installationsanleitung

### Schritt 1: Wählen Sie Ihr KI‑Tool

Stellen Sie zunächst sicher, dass Sie ein KI‑Entwicklungstool verwenden, das MCP unterstützt:

- [Claude Desktop](https://claude.ai/download) (Kostenlos - empfohlen für Anfänger)
- [Cursor](https://www.cursor.com/) (KI‑gestützter Code‑Editor)
- [Windsurf by Codeium](https://windsurf.com/) (KI‑Entwicklungsplattform)
- Andere MCP‑kompatible Tools

### Schritt 2: Suchen Sie Ihre Konfigurationsdatei

Je nach Ihrem KI‑Tool müssen Sie eine dieser Dateien bearbeiten:

- **Claude Desktop**: `~/Library/Application\ Support/Claude/claude_desktop_config.json` für macOS und `%APPDATA%\Claude\claude_desktop_config.json` für Windows
- **Cursor**: `.cursor/mcp.json`
- **Windsurf**: `.codeium/windsurf/mcp_config.json`
- **Andere Tools**: Prüfen Sie die MCP‑Dokumentation Ihres Tools

### Schritt 3: Ultracite‑Konfiguration hinzufügen

Kopieren Sie diese Konfiguration in Ihre MCP‑Konfigurationsdatei:

```json
{
  "mcpServers": {
    "ultracite": {
      "command": "npx",
      "args": [
        "-y",
        "mcp-remote",
        "https://www.ultracite.ai/api/mcp/mcp"
      ]
    }
  }
}
```

### Schritt 4: Starten Sie Ihre KI‑Anwendung neu

Schließen und öffnen Sie Ihre KI‑Anwendung neu, damit die Änderungen wirksam werden.

### Schritt 5: Überprüfen Sie die Verbindung

Testen Sie die Integration, indem Sie Ihren KI‑Assistenten fragen:

> Welche Ultracite‑Regeln sind verfügbar?

Wenn erfolgreich, sollte Ihr KI‑Assistent in der Lage sein, Ultracite‑Regeln aufzulisten und zu erklären!

## Mehrere MCP‑Server

Sie können Ultracite zusammen mit anderen MCP‑Servern verwenden:

```json
{
  "mcpServers": {
    "ultracite": {
      "command": "npx",
      "args": ["-y", "mcp-remote", "https://www.ultracite.ai/api/mcp/mcp"]
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"]
    },
    "filesystem": {
      "command": "npx", 
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/project"]
    }
  }
}
```

## Anwendungsbeispiele

### Verfügbare Regeln erkunden

Fragen Sie Ihren KI‑Assistenten:

> Welche Regeln hat Ultracite für React‑Hooks?

Die erwartete Antwort enthält relevante Linting‑Regeln für React‑Hooks und deren Beschreibungen.

### Regelkategorien verstehen

> Zeige mir alle Barrierefreiheitsregeln in Ultracite

Ihre KI kann die in Ultracite konfigurierten a11y‑Regeln auflisten und erläutern.

### Presets vergleichen

> Was ist der Unterschied zwischen ultracite/core und ultracite/react?

Erhalten Sie detaillierte Informationen zu verschiedenen Preset‑Konfigurationen und Hinweise, wann welches Preset zu verwenden ist.

## Sicherheit und Datenschutz

### Datenverarbeitung

- Der Ultracite‑MCP‑Server stellt nur öffentliche Dokumentation zu Linting‑Regeln und Konfigurationsinformationen bereit
- Es werden keine personenbezogenen Daten oder Code an unsere Server übertragen
- Alle Kommunikation erfolgt über die Sicherheitsschicht Ihres gewählten KI‑Tools

### Authentifizierung

- Für den Zugriff auf öffentliche Dokumentation ist keine Authentifizierung erforderlich
- Zukünftige Premium‑Funktionen können API‑Schlüssel erfordern
- Verwenden Sie stets die offiziellen Ultracite MCP‑Endpunkte
