Ultracite kan worden geïnstalleerd met alle grote package managers. Je hebt Node.js v14.18+ nodig (Node 18+ aanbevolen) en een project met een `package.json`-bestand.

## Installatie

De eenvoudigste manier om te beginnen is het uitvoeren van het `init`-script van Ultracite, dat de benodigde pakketten installeert en het configbestand voor je aanmaakt. In je projectdirectory, voer uit:

```package-install
npx ultracite init
```

Je kunt optioneel de volgende flags meegeven aan het `init`-commando:

| Flag                | Beschrijving                                     | Opties                                                                  |
|---------------------|--------------------------------------------------|-------------------------------------------------------------------------|
| `--pm`              | De te gebruiken pakketbeheerder.                 | `pnpm`, `bun`, `yarn`, `npm`                                            |
| `--frameworks`      | De frameworks die geconfigureerd moeten worden.  | Komma-gescheiden: `react`, `next`, `solid`, `vue`, `qwik`, `angular`, `remix`, `svelte` |
| `--editors`         | De editors die geconfigureerd moeten worden.     | Komma-gescheiden: `vscode`, `zed`                                       |
| `--agents`          | De AI-agents die geconfigureerd moeten worden.   | Komma-gescheiden: `vscode-copilot`, `cursor`, `windsurf`, `zed`, `claude`, `codex`, `kiro`, `cline`, `amp`, `aider`, `firebase-studio`, `open-hands`, `gemini-cli`, `junie`, `augmentcode`, `kilo-code`, `goose`, `roo-code`, `warp`, `droid` |
| `--hooks`           | De agent-hooks die geconfigureerd moeten worden. | Komma-gescheiden: `cursor`, `claude`                                    |
| `--integrations`    | Aanvullende integraties om in te stellen.        | Komma-gescheiden: `husky`, `lefthook`, `lint-staged`                    |
| `--migrate`         | Migratietools die verwijderd moeten worden.      | Komma-gescheiden: `eslint`, `prettier` (verwijdert afhankelijkheden, configuratiebestanden en editorinstellingen) |
| `--skip-install`    | Het overslaan van het installeren van afhankelijkheden. | (geen waarde nodig)                                                       |
| `--quiet`           | Interactieve prompts en visuele output onderdrukken. | (geen waarde nodig, automatisch ingeschakeld in CI)                          |

### Programmematig gebruik

Voor tools die Ultracite programmeerbaar moeten aanroepen (zoals scaffolding-CLIs), kun je de `--quiet`-vlag gebruiken om alle interactieve prompts en visuele output te onderdrukken:

```bash title="Terminal"
npx ultracite init --quiet --pm npm
```

De `--quiet`-vlag wordt automatisch ingeschakeld wanneer de omgevingsvariabele `CI` is ingesteld op `true` of `1`, waardoor het geschikt is voor CI/CD-pijplijnen:

```bash title="Terminal"
CI=true npx ultracite init --pm npm
```

In de stille modus zal Ultracite:
- Alle interactieve prompts overslaan
- Verstandige standaardwaarden gebruiken voor alle opties
- ASCII-art, spinners en logberichten onderdrukken
- Alleen de kernconfiguratie voor Biome aanmaken
- Netjes afsluiten met code 0 bij succes of 1 bij falen

## Extensies

Als je Visual Studio Code gebruikt, installeer dan de officiële [Biome](https://marketplace.visualstudio.com/items?itemName=biomejs.biome) extensie, die formatteren en correcties bij het opslaan mogelijk maakt:

```bash title="Terminal"
code --install-extension biomejs.biome
```

Daarnaast, als je Tailwind CSS gebruikt, wordt aanbevolen de [Tailwind CSS IntelliSense](https://marketplace.visualstudio.com/items?itemName=bradlc.vscode-tailwindcss) extensie te installeren om classes te sorteren en hints/suggesties te krijgen:

```bash title="Terminal"
code --install-extension bradlc.vscode-tailwindcss
```
