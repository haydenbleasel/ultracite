Ultracite erbjuder sömlös integration med AI-kodassistenter genom att automatiskt generera redigerarspecifika regelfiler. Dessa regler hjälper till att vägleda AI-kodassistenter att skriva bättre, mer konsekvent kod som följer bästa praxis.

Redigeringsreglerna fungerar tillsammans med Biome:s linting och formattering. Medan Biome hanterar kodformatering och fångar syntaxfel, vägleder redigeringsreglerna AI-kodassistenter att skriva bättre kod från början, vilket minskar behovet av korrigeringar i efterhand.

Denna dubbla strategi säkerställer både automatiserad kodkvalitet (genom Biome) och intelligent kodgenerering (genom AI-redigeringsregler).

## Konfiguration

### Automatisk konfiguration

När du kör `npx ultracite init` blir du tillfrågad att välja vilka redigeringsregler du vill aktivera:

```bash
? Which editor rules do you want to enable (optional)?
  ◯ GitHub Copilot (VSCode)
  ◯ Cursor
  ◯ Windsurf
  # ... etcetera
```

Detta skapar den relevanta filen i din projektkatalog.

### Manuell konfiguration

Om du vill ställa in saker manuellt, skapa den relevanta filen i din projektkatalog (till exempel `.cursor/rules` för Cursor eller `.rules` för Zed). Sedan kopierar och klistrar du in [detta innehåll](https://github.com/haydenbleasel/ultracite/tree/main/.cursor/rules/ultracite.mdc) i filen.

## Fördelar

Att använda Ultracite redigeringsregler ger flera fördelar:

- **Konsekvens**: All AI-genererad kod följer samma konventioner
- **Kvalitet**: Reglerna upprätthåller bästa praxis och förhindrar vanliga misstag
- **Tillgänglighet**: Inbyggda riktlinjer för tillgänglighet säkerställer inkluderande kod
- **Prestanda**: Regler främjar effektiva mönster och undviker anti-mönster
- **Säkerhet**: Riktlinjer hjälper till att förebygga vanliga säkerhetsbrister

## Anpassning

Även om standardreglerna är omfattande kan du anpassa dem för dina specifika behov:

1. **Ändra befintliga regler**: Redigera regelfilerna direkt
2. **Lägg till projektspecifika regler**: Lägg till ytterligare riktlinjer
3. **Inaktivera regler**: Ta bort eller kommentera ut regler som inte gäller för ditt projekt
