Ultracite erbjuder sömlös integration med AI-kodassistenter genom att automatiskt generera editorregelfiler. Dessa regler hjälper till att vägleda AI-kodassistenter att skriva bättre, mer konsekvent kod som följer bästa praxis.

Editorreglerna fungerar tillsammans med Biomes lintning och formatering. Medan Biome hanterar kodformatering och fångar syntaxfel, leder editorreglerna AI-kodassistenter att skriva bättre kod från början, vilket minskar behovet av rättningar senare.

Denna dubbla strategi säkerställer både automatisk kodkvalitet (genom Biome) och intelligent kodgenerering (genom AI-editorregler).

## Konfiguration

### Automatisk installation

När du kör `npx ultracite init` kommer du att uppmanas att välja vilka editorregler du vill aktivera:

```bash
? Which editor rules do you want to enable (optional)?
  ◯ GitHub Copilot (VSCode)
  ◯ Cursor
  ◯ Windsurf
  # ... etcetera
```

Detta kommer att skapa den relevanta filen i din projektkatalog.

### Manuell installation

Om du vill ställa in det manuellt, skapa den relevanta filen i din projektkatalog (till exempel `.cursor/rules` för Cursor eller `.rules` för Zed). Kopiera sedan och klistra in [detta innehåll](https://github.com/haydenbleasel/ultracite/tree/main/.cursor/rules/ultracite.mdc) i filen.

## Fördelar

Att använda Ultracite editorregler ger flera fördelar:

- **Konsekvens**: All AI-genererad kod följer samma konventioner
- **Kvalitet**: Regler upprätthåller bästa praxis och förhindrar vanliga misstag
- **Tillgänglighet**: Inbyggda riktlinjer för tillgänglighet säkerställer inkluderande kod
- **Prestanda**: Regler främjar effektiva mönster och undviker anti-mönster
- **Säkerhet**: Riktlinjer hjälper till att förhindra vanliga säkerhetssårbarheter

## Anpassning

Även om standardreglerna är omfattande kan du anpassa dem efter dina specifika behov:

1. **Ändra befintliga regler**: Redigera regelfilerna direkt
2. **Lägg till projektspecifika regler**: Lägg till ytterligare riktlinjer
3. **Inaktivera regler**: Ta bort eller kommentera ut regler som inte gäller för ditt projekt
