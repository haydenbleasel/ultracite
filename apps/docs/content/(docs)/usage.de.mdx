## IDE-Integration

Ultracite funktioniert am besten, wenn es in Ihren Editor integriert ist, sodass Sie sofortiges Feedback beim Schreiben von Code erhalten. Es ist so konzipiert, dass es automatisch beim Speichern ausgeführt wird. Um es auszuprobieren, öffnen Sie ein Projekt in Ihrer IDE. Während Sie Dateien bearbeiten, sollten Sie sehen, dass beim Speichern die Formatierung angewendet wird. Versuchen Sie, einen kleinen Stilfehler im Code einzuführen (wie ein zusätzliches Semikolon oder eine falsch eingerückte Zeile) und klicken Sie auf **Speichern** – Ultracite (Biome) formatiert die Datei sofort neu.

Wenn Sie ein häufiges Lint-Problem einführen (wie eine unbenutzte Variable), sehen Sie möglicherweise eine wellenförmige Unterstreichung oder eine Warnung im Problems-Bereich; beim Speichern versucht Ultracite, das Problem zu beheben, wenn es automatisch behebbar ist (zum Beispiel eine unbenutzte Import-Anweisung entfernen), oder lässt andernfalls eine Warnung für Sie stehen, die Sie beheben müssen.

### Sofortige Formatierung

Jedes Mal, wenn Sie eine Datei speichern, formatiert Biome (mit Ultracites Konfiguration) die Datei. Sie müssen kein separates Prettier ausführen oder sich um Formatierung kümmern – das wird übernommen.

Dank der `codeActionsOnSave`-Einstellungen wendet VS Code beim Speichern auch Biomes „fix all“-Aktion an. Das bedeutet, jede Lint-Regel, die eine automatische Korrektur hat (wie die Umwandlung von `!=` zu `!==`, das Hinzufügen fehlender Klammern, das Beheben der Import-Reihenfolge usw.), wird angewendet. Es ist so, als hätten Sie automatisch `--fix` mit ESLint ausgeführt.

Wir haben sogar `formatOnPaste` aktiviert – wenn Sie also Code von anderswo einfügen, wird er sofort formatiert, um Ihrem Stil zu entsprechen.

Der Hauptvorteil ist hier das sofortige Feedback. Sie schreiben Code in VS Code, und Ultracite hält ihn kontinuierlich sauber. Mit der Zeit werden Sie feststellen, dass Sie wenig bis keine Zeit mehr damit verbringen, Lint-Fehler zu beheben – Ultracite behebt sie entweder für Sie oder macht Sie frühzeitig darauf aufmerksam. Das führt zu einem reibungslosen Entwicklungsworkflow, in dem sich Code-Stil und kleinere Probleme nie aufstauen.

### Integration in das Problems-Panel

Alle verbleibenden Probleme (die Ihre Aufmerksamkeit erfordern) werden im Problems-Panel angezeigt. Wenn Ultracite beispielsweise eine Regel hat, die etwas verbietet, oder einen Fehler findet, den es nicht beheben kann (z. B. Sie haben eine veraltete API verwendet oder vergessen, eine Promise-Ablehnung zu behandeln), wird dies als Fehler oder Warnung aufgeführt. Sie können darauf klicken, um zur entsprechenden Stelle im Code zu springen.

### Code Lenses / Schnellkorrekturen

In vielen Fällen bietet die VS Code-Erweiterung für Biome Vorschläge für Schnellkorrekturen. Wenn Sie eine gelbe Glühbirne oder ein Vorschlags-Popup sehen, können Sie Korrekturen auch manuell anwenden. Zum Beispiel, wenn ein Import ungenutzt ist, erhalten Sie möglicherweise eine Schnellkorrektur zum Entfernen dieses Imports (falls er beim Speichern nicht bereits entfernt wurde).

## CLI-Verwendung

Ultracite wird mit einer praktischen CLI geliefert (bei Installation als `ultracite` aufrufbar). Hier sind gängige Möglichkeiten, sie zu verwenden.

### Code prüfen

Der `check`-Befehl kapselt den Biome-`check`-Befehl, der den Linter ausführt, ohne Dateien zu ändern.

```bash title="Terminal"
npx ultracite check
```

Sie können den `--diagnostic-level`-Filter übergeben, um nur Fehler oder Warnungen anzuzeigen:

```bash title="Terminal"
# Show only errors
npx ultracite check --diagnostic-level error

# Show warnings and above (errors)
npx ultracite check --diagnostic-level warn
```

### Code korrigieren

Der `fix`-Befehl kapselt den Biome-`check --write`-Befehl, der den Linter ausführt und Dateien korrigiert.

```bash title="Terminal"
npx ultracite fix
```

Sie können außerdem unsichere Korrekturen anwenden, indem Sie ausführen:

```bash title="Terminal"
npx ultracite fix --unsafe
```

### Konfiguration validieren

Der `doctor`-Befehl überprüft Ihre Konfiguration auf Probleme und gibt Empfehlungen. Dies ist nützlich, nachdem Sie Ultracite installiert haben, um sicherzustellen, dass alles korrekt konfiguriert ist.

```bash title="Terminal"
npx ultracite doctor
```
