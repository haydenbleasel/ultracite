Az Ultracite egyik legjobb tulajdonsága, hogy azonnal használható, konfiguráció nélkül – egyszerűen kiterjesztheted a `ultracite` presetet és indulhatsz. Ugyanakkor minden projekt némileg eltér, és előfordulhat, hogy bizonyos szabályokat testre szeretnél szabni, vagy az Ultracite-ot különböző keretrendszerekhez szeretnéd igazítani. Ez a rész az alapértelmezett konfigurációt, valamint annak kiterjesztését és felülírását ismerteti.

Az Ultracite keretrendszer-specifikus konfigurációkat is kínál, amelyeket a báziskonfiguráción felül kiterjeszthetsz. Ez lehetővé teszi, hogy keretrendszer-specifikus lint-szabályokat adj hozzá anélkül, hogy a báziskonfigurációt felduzzasztanád vagy irreleváns szabályokkal kellene foglalkoznod.

A rendelkezésre álló keretrendszer-specifikus konfigurációk teljes listája a [Előre beállított konfigurációk](/preset/core) szakaszban található.

### Használat

Keretrendszer-specifikus konfiguráció használatához add hozzá az `extends` tömbhöz:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/solid"]
}
```

Több konfigurációt is kombinálhatsz, ha több keretrendszert használó projekten dolgozol, vagy olyan eszközt (pl. Astro) használsz, amely több keretrendszert támogat.

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/react", "ultracite/remix"]
}
```

## Alapértelmezett konfiguráció

Ha a `biome.jsonc`-edben szerepel az `"extends": ["ultracite/core"]`, akkor az Ultracite bázispreset konfigurációját húzod be. Ez a preset keretrendszer-agnosztikus szabályokat és beállításokat tartalmaz. Az alapértelmezett konfiguráció néhány jelentősebb jellemzője:

- **TypeScript szigorúsága**: Az Ultracite engedélyezi a TypeScript szigorú ellenőrzéseit és lint-szabályait, amelyek megkövetelik a megbízható típusozást. Például kerülendő az `any` használata, megköveteli a `null/undefined` kezelést, és bizonyos helyzetekben előnyben részesíti a kifejezett típusokat az egyértelműség érdekében.
- **ESLint "Recommended" megfelelői**: Tartalmazza a gyakori legjobb gyakorlatokra vonatkozó szabályokat (például no unused vars, no explicit eval, no proto pollution stb.).
- **Akadálymentesség**: Az akadálymentességi szabványoknak megfelelő szabályok alapértelmezetten aktívak (ARIA attribútumok, szemantikus HTML, billentyűzet-navigáció stb.).
- **Node.js**: A környezet-specifikus sajátosságok kezelve vannak. Például a Node által biztosított globális változók (mint a `module`, `process`) felismerésre kerülnek, így nem lesznek tévesen ismeretlennek jelölve.
- **Formázási konvenciók**: Az Ultracite formázóbeállításai (nagy részben a Biome/Prettier alapértelmezéseiből örökölve) tartalmazzák a 2 szóközös behúzást, 80 karakteres maximális sorhosszt, pontosvesszők használatát a kifejezések végén, sorvégi vesszőket ahol érvényes (pl. több soros tömbök/objektumok esetén), dupla idézőjeleket a stringekhez stb. Ezeket a gyakori konvenciókhoz igazodva és a diffek tisztaságának biztosítása érdekében választottuk.

Ezek az alapbeállítások azt a célt szolgálják, hogy egységes stílust érvényesítsenek és megelőzzék a gyakori hibákat anélkül, hogy minden egyes szabályt kézzel kellene konfigurálnod.

Az Ultracite báziskonfigurációjába bevont összes lint-szabály részletes listájáért lásd az [Alapkonfiguráció](/preset/core) dokumentációt.

## A konfiguráció felülírása

### A `biome.jsonc` módosítása

A szabályokat úgy is letilthatod, hogy módosítod a `biome.jsonc` fájlt.

Például a `noAutofocus` szabály letiltásához a következőt adhatod hozzá:

```jsonc title="biome.jsonc"
{
  "linter": {
    "rules": {
      "a11y": {
        "noAutofocus": "off"
      }
    }
  }
}
```

### Sorvégi megjegyzések

Szabályokat soronként is letilthatsz úgy, hogy a sor végére egy megjegyzést írsz:

```tsx title="page.tsx"
// biome-ignore lint/security/noDangerouslySetInnerHtml: This is a valid use case.
<div dangerouslySetInnerHTML={{ ... }} />
```
