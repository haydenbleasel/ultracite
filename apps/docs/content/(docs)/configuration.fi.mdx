Yksi Ultraciten parhaista ominaisuuksista on, että se toimii heti ilman konfiguraatiota – voit vain laajentaa `ultracite`-esiasetusta ja aloittaa. Kuitenkin jokainen projekti on hieman erilainen, ja saatat haluta mukauttaa tiettyjä sääntöjä tai sovittaa Ultraciten eri kehystoihin. Tämä osio selittää oletuskonfiguraation ja miten sitä voi laajentaa tai ylikirjoittaa.

Ultracite tarjoaa kehyskohtaisia konfiguraatioita, joita voit laajentaa pohjakonfiguraation lisäksi. Tämä mahdollistaa kehyskohtaisen linttauslogiikan lisäämisen ilman, että pohjakonfiguraatio paisuu tai että joudut käsittelemään irrelevantteja sääntöjä.

Löydät kaikki saatavilla olevat kehyskonfiguraatiot [Esiasetukset](/preset/core) -osiosta.

### Käyttö

Käyttääksesi kehyskohtaista konfiguraatiota, lisää se `extends`-taulukkoosi:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/solid"]
}
```

Voit yhdistää useita konfiguraatioita, jos työskentelet monikehysprojektissa tai käytät jotain kuten Astroa, joka tukee useita kehyksiä.

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/react", "ultracite/remix"]
}
```

## Oletuskonfiguraatio

Kun lisäät `"extends": ["ultracite/core"]` `biome.jsonc`-tiedostoosi, otat käyttöön Ultraciten pohjaesiasetuksen. Tämä esiasetus sisältää kehysriippumattomat säännöt ja konfiguraatiot. Jotkin huomionarvoiset piirteet oletuskonfiguraatiossa:

- **TypeScriptin tiukkuus**: Ultracite ottaa käyttöön TypeScriptin tiukat tarkistukset ja linterisäännöt, jotka edellyttävät vankkaa tyypitystä. Esimerkiksi se ei suosi `any`-tyypin käyttöä, vaatii `null/undefined`-tilanteiden käsittelyä ja suosii eksplisiittisiä tyyppejä tietyissä tilanteissa epäselvyyksien välttämiseksi.
- **ESLintin "Recommended"-vastaavuudet**: Yleisiä parhaiden käytäntöjen sääntöjä (no unused vars, no explicit eval, no proto pollution, jne.) sisältyy.
- **Saavutettavuus**: Saavutettavuusstandardeja vastaavat säännöt ovat oletuksena päällä (ARIA-attribuutit, semanttinen HTML, näppäimistönavigaatio, jne.).
- **Node.js**: Ympäristökohtaiset erityispiirteet käsitellään. Esimerkiksi Nodesta tulevat globaalit muuttujat (kuten `module`, `process`) tunnistetaan, joten niitä ei merkitä virheellisesti määrittelemättömiksi.
- **Muotoilukäytännöt**: Ultraciten muotoilijan asetukset (suurelta osin periytyneet Biome/Prettier-oletuksista) sisältävät 2-välisen sisennyksen, maksimirivileveyden 80 merkkiä, puolipisteet lauseiden lopussa, perässä tulevat pilkut siellä missä validia (esim. monirivisissä taulukoissa/olioissa), lainausmerkit merkkijonoille jne. Nämä valinnat on tehty vastaamaan yleisiä käytäntöjä ja varmistamaan muutosvertailujen selkeys.

Kaikki nämä oletukset pyrkivät pakottamaan johdonmukaisen tyylin ja estämään yleisiä virheitä ilman, että sinun tarvitsee määrittää jokaista sääntöä manuaalisesti.

Kattavan viitteen kaikista Ultraciten pohjakonfiguraatiossa määritellyistä linttausrajoista löydät [Ydinkonfiguraatio](/preset/core) -dokumentaatiosta.

## Konfiguraation ohittaminen

### `biome.jsonc`-tiedoston muokkaaminen

Voit poistaa sääntöjä käytöstä muokkaamalla `biome.jsonc`-tiedostoa.

Esimerkiksi poistaaksesi käytöstä `noAutofocus`-säännön, voit lisätä seuraavaa:

```jsonc title="biome.jsonc"
{
  "linter": {
    "rules": {
      "a11y": {
        "noAutofocus": "off"
      }
    }
  }
}
```

### Rivikohtaiset kommentit

Voit myös poistaa sääntöjä käytöstä rivi kerrallaan lisäämällä kommentin rivin loppuun:

```tsx title="page.tsx"
// biome-ignore lint/security/noDangerouslySetInnerHtml: This is a valid use case.
<div dangerouslySetInnerHTML={{ ... }} />
```
