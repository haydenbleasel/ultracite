Ultracite menyediakan integrasi yang mulus dengan asisten pengkodean AI dengan secara otomatis menghasilkan file aturan khusus editor. Aturan-aturan ini membantu membimbing asisten pengkodean AI untuk menulis kode yang lebih baik, lebih konsisten, dan mengikuti praktik terbaik.

Aturan editor bekerja berdampingan dengan linting dan pemformatan Biome. Sementara Biome menangani pemformatan kode dan menangkap kesalahan sintaks, aturan editor membimbing asisten pengkodean AI untuk menulis kode yang lebih baik sejak awal, mengurangi kebutuhan perbaikan di kemudian hari.

Pendekatan ganda ini memastikan kualitas kode otomatis (melalui Biome) dan pembuatan kode cerdas (melalui aturan pengkodean AI).

## Pengaturan

### Pengaturan Otomatis

Ketika Anda menjalankan `npx ultracite init`, Anda akan diminta untuk memilih aturan editor mana yang ingin Anda aktifkan:

```bash
? Which editor rules do you want to enable (optional)?
  ◯ GitHub Copilot (VSCode)
  ◯ Cursor
  ◯ Windsurf
  # ... etcetera
```

Ini akan membuat file yang relevan di direktori proyek Anda.

### Pengaturan Manual

Jika Anda ingin mengatur secara manual, buat file yang relevan di direktori proyek Anda (misalnya `.cursor/rules` untuk Cursor atau `.rules` untuk Zed). Kemudian, salin dan tempel [konten ini](https://github.com/haydenbleasel/ultracite/tree/main/.cursor/rules/ultracite.mdc) ke dalam file tersebut.

## Manfaat

Menggunakan aturan editor Ultracite memberikan beberapa keuntungan:

- **Konsistensi**: Semua kode yang dihasilkan AI mengikuti konvensi yang sama
- **Kualitas**: Aturan menegakkan praktik terbaik dan mencegah kesalahan umum
- **Aksesibilitas**: Pedoman aksesibilitas bawaan memastikan kode yang inklusif
- **Performa**: Aturan mendorong pola yang efisien dan menghindari anti-pola
- **Keamanan**: Pedoman membantu mencegah kerentanan keamanan umum

## Kustomisasi

Meskipun aturan default sudah komprehensif, Anda dapat menyesuaikannya untuk kebutuhan spesifik Anda:

1. **Ubah aturan yang ada**: Edit file aturan secara langsung
2. **Tambah aturan spesifik proyek**: Tambahkan pedoman tambahan
3. **Nonaktifkan aturan**: Hapus atau komentari aturan yang tidak berlaku untuk proyek Anda
