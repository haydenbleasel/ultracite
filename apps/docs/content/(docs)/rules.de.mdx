Ultracite bietet eine nahtlose Integration mit KI-Coding-Assistenten, indem automatisch editor-spezifische Regeldateien erzeugt werden. Diese Regeln helfen dabei, KI-Coding-Assistenten anzuleiten, besseren und konsistenteren Code zu schreiben, der Best Practices befolgt.

Die Editor-Regeln arbeiten parallel zu Biomes Linting und Formatierung. Während Biome die Codeformatierung übernimmt und Syntaxfehler erkennt, leiten die Editor-Regeln KI-Coding-Assistenten an, von Anfang an besseren Code zu erzeugen, wodurch später weniger Korrekturen erforderlich sind.

Dieser duale Ansatz stellt sowohl automatisierte Codequalität (durch Biome) als auch intelligente Codegenerierung (durch KI-Coding-Regeln) sicher.

## Einrichtung

### Automatische Einrichtung

Wenn Sie `npx ultracite init` ausführen, werden Sie aufgefordert auszuwählen, welche Editor-Regeln Sie aktivieren möchten:

```bash
? Which editor rules do you want to enable (optional)?
  ◯ GitHub Copilot (VSCode)
  ◯ Cursor
  ◯ Windsurf
  # ... etcetera
```

Dies erstellt die relevante Datei in Ihrem Projektverzeichnis.

### Manuelle Einrichtung

Wenn Sie die Einrichtung manuell vornehmen möchten, erstellen Sie die entsprechende Datei in Ihrem Projektverzeichnis (z. B. `.cursor/rules` für Cursor oder `.rules` für Zed). Kopieren Sie dann [diesen Inhalt](https://github.com/haydenbleasel/ultracite/tree/main/.cursor/rules/ultracite.mdc) in die Datei.

## Vorteile

Die Verwendung der Ultracite-Editor-Regeln bietet mehrere Vorteile:

- **Konsistenz**: Alle von KI generierten Codes folgen denselben Konventionen
- **Qualität**: Regeln erzwingen Best Practices und verhindern häufige Fehler
- **Barrierefreiheit**: Eingebaute Richtlinien zur Barrierefreiheit sorgen für inklusiven Code
- **Performance**: Regeln fördern effiziente Muster und vermeiden Anti-Pattern
- **Sicherheit**: Richtlinien helfen, häufige Sicherheitslücken zu vermeiden

## Anpassung

Während die Standardregeln umfassend sind, können Sie sie an Ihre spezifischen Bedürfnisse anpassen:

1. **Bestehende Regeln ändern**: Bearbeiten Sie die Regeldateien direkt
2. **Projektspezifische Regeln hinzufügen**: Fügen Sie zusätzliche Richtlinien hinzu
3. **Regeln deaktivieren**: Entfernen oder kommentieren Sie Regeln aus, die nicht für Ihr Projekt gelten
