---
title: 规则
description: 通过编辑器规则将 Ultracite 与 AI 编码助理集成的方法。
---

Ultracite 通过自动生成针对特定编辑器的规则文件，实现与 AI 编码助理的无缝集成。 这些规则帮助引导 AI 编码助理编写更好、更一致且遵循最佳实践的代码。

这些编辑器规则与 Biome 的 lint 和格式化功能配合工作。虽然 Biome 负责代码格式化并捕获语法错误，编辑器规则从一开始就引导 AI 编码助理编写更好的代码，从而减少后续修复的需求。

这种双重方法既确保了通过 Biome 实现的自动化代码质量，也确保了通过 AI 编码规则实现的智能代码生成。

## 设置

### 自动设置

当你运行 `npx ultracite init` 时，你将被提示选择要启用哪些编辑器规则：

```bash
? Which editor rules do you want to enable (optional)?
  ◯ GitHub Copilot (VSCode)
  ◯ Cursor
  ◯ Windsurf
  # ... etcetera
```

这将在你的项目目录中创建相应的文件。

### 手动设置

如果你想手动设置，请在项目目录中创建相应的文件（例如 Cursor 的 `.cursor/rules` 或 Zed 的 `.rules`）。然后，将 [此内容](https://github.com/haydenbleasel/ultracite/tree/main/.cursor/rules/ultracite.mdc) 复制并粘贴到该文件中。

## 优势

使用 Ultracite 编辑器规则具有以下几个优点：

- **一致性**: 所有 AI 生成的代码遵循相同的约定
- **质量**: 规则强制实施最佳实践并防止常见错误
- **可访问性**: 内置的可访问性指南确保代码的包容性
- **性能**: 规则推动高效模式并避免反模式
- **安全性**: 指南有助于防止常见的安全漏洞

## 自定义

尽管默认规则已经很全面，但你可以根据特定需求进行自定义：

1. **修改现有规则**: 直接编辑规则文件
2. **添加项目特定规则**: 附加额外的指南
3. **禁用规则**: 删除或注释掉不适用于你项目的规则
