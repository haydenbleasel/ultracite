---

Ultraciteは、エディタ固有のルールファイルを自動生成することで、AIコーディングアシスタントとのシームレスな統合を提供します。これらのルールは、ベストプラクティスに従った、より一貫性のある良質なコードをAIが生成するように導くのに役立ちます。

エディタルールはBiomeのリンティングとフォーマット機能と並行して動作します。Biomeがコードのフォーマットを処理し構文エラーを検出する一方で、エディタルールはAIコーディングアシスタントに最初からより良いコードを書くよう指示し、後で修正が必要になる頻度を減らします。

この二重のアプローチにより、自動化されたコード品質（Biome経由）とインテリジェントなコード生成（AIコーディングルール経由）の両方が確保されます。

## セットアップ

### 自動セットアップ

`npx ultracite init` を実行すると、有効にするエディタルールの選択を求められます:

```bash
? Which editor rules do you want to enable (optional)?
  ◯ GitHub Copilot (VSCode)
  ◯ Cursor
  ◯ Windsurf
  # ... etcetera
```

これにより、プロジェクトディレクトリに該当するファイルが作成されます。

### 手動セットアップ

手動で設定したい場合は、プロジェクトディレクトリに該当するファイルを作成してください（例: Cursor用は`.cursor/rules`、Zed用は`.rules`）。その後、ファイルに[この内容](https://github.com/haydenbleasel/ultracite/tree/main/.cursor/rules/ultracite.mdc)をコピー＆ペーストしてください。

## 利点

Ultraciteのエディタルールを使用することにはいくつかの利点があります:

- **一貫性**: すべてのAI生成コードが同じ規約に従います
- **品質**: ルールはベストプラクティスを強制し、一般的なミスを防ぎます
- **アクセシビリティ**: 組み込みのアクセシビリティガイドラインにより包摂的なコードを実現します
- **パフォーマンス**: ルールは効率的なパターンを促進し、アンチパターンを回避します
- **セキュリティ**: ガイドラインは一般的なセキュリティ脆弱性の防止に役立ちます

## カスタマイズ

デフォルトのルールは包括的ですが、プロジェクト固有のニーズに合わせてカスタマイズできます:

1. **既存のルールを変更する**: ルールファイルを直接編集します
2. **プロジェクト固有のルールを追加する**: 追加のガイドラインを追記します
3. **ルールを無効化する**: プロジェクトに当てはまらないルールを削除またはコメントアウトします
