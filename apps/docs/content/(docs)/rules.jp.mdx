Ultraciteは、エディタ固有のルールファイルを自動生成することで、AIコーディングアシスタントとのシームレスな統合を提供します。これらのルールは、AIコーディングアシスタントがベストプラクティスに従い、より良く一貫性のあるコードを書くようガイドします。

エディタルールはBiomeのリンティングおよびフォーマッティングと並行して動作します。Biomeがコードのフォーマットを担当し構文エラーを検出する一方で、エディタルールはAIコーディングアシスタントに初めから良いコードを書くよう指示し、後から修正する必要を減らします。

この二重のアプローチにより、自動化されたコード品質（Biomeを通じて）とインテリジェントなコード生成（AIコーディングルールを通じて）の両方が確保されます。

## 設定

### 自動セットアップ

When you run `npx ultracite init`, you'll be prompted to select which editor rules you want to enable:

```bash
? Which editor rules do you want to enable (optional)?
  ◯ GitHub Copilot (VSCode)
  ◯ Cursor
  ◯ Windsurf
  # ... etcetera
```

This will create the relevant file in your project directory.

### 手動セットアップ

If you want to manually set things up, create the relevant file in your project directory (for example `.cursor/rules` for Cursor or `.rules` for Zed). Then, copy and paste [この内容](https://github.com/haydenbleasel/ultracite/tree/main/.cursor/rules/ultracite.mdc) into the file.

## 利点

Ultraciteのエディタルールを使用することには、いくつかの利点があります:

- **一貫性**: 生成されたAIコードがすべて同じ規約に従う
- **品質**: ルールがベストプラクティスを適用し、よくあるミスを防ぐ
- **アクセシビリティ**: 組み込みのアクセシビリティガイドラインにより包括的なコードを確保する
- **パフォーマンス**: 効率的なパターンを促進し、アンチパターンを回避する
- **セキュリティ**: ガイドラインが一般的なセキュリティ脆弱性を防ぐのに役立つ

## カスタマイズ

デフォルトのルールは包括的ですが、プロジェクト固有のニーズに合わせてカスタマイズできます:

1. **Modify existing rules**: ルールファイルを直接編集する
2. **Add project-specific rules**: 追加のガイドラインを追記する
3. **Disable rules**: プロジェクトに該当しないルールを削除またはコメントアウトする
