Model Bağlam Protokolü (**MCP**), Claude, Cursor ve diğer yapay zeka destekli araçların harici veri kaynakları ve sistemlerle güvenli bir şekilde bağlanmasını sağlayan açık bir standarttır. Bunu, yapay zeka araçlarınızın gerçek dünya verilerine ve işlevlerine erişmesine izin veren bir "evrensel kumanda" olarak düşünün.

Ultracite, AI geliştirme iş akışınızı hızlandırmak için MCP'yi destekler.

## Kurulum Kılavuzu

### Adım 1: AI Aracınızı Seçin

Öncelikle MCP'yi destekleyen bir AI geliştirme aracı kullandığınızdan emin olun:

- [Claude Masaüstü](https://claude.ai/download) (Ücretsiz - yeni başlayanlar için önerilir)
- [Cursor](https://www.cursor.com/) (AI destekli kod editörü)
- [Codeium tarafından Windsurf](https://windsurf.com/) (AI geliştirme platformu)
- Diğer MCP uyumlu araçlar

### Adım 2: Yapılandırma Dosyanızı Bulun

Kullandığınız AI aracına bağlı olarak, şu dosyalardan birini düzenlemeniz gerekecek:

- **Claude Masaüstü**: `~/Library/Application\ Support/Claude/claude_desktop_config.json` (macOS) ve `%APPDATA%\Claude\claude_desktop_config.json` (Windows)
- **Cursor**: `.cursor/mcp.json`
- **Windsurf**: `.codeium/windsurf/mcp_config.json`
- **Diğer araçlar**: Aracınızın MCP dokümantasyonuna bakın

### Adım 3: Ultracite Yapılandırmasını Ekleyin

Bu yapılandırmayı MCP yapılandırma dosyanıza kopyalayın ve yapıştırın:

```json
{
  "mcpServers": {
    "ultracite": {
      "command": "npx",
      "args": [
        "-y",
        "mcp-remote",
        "https://www.ultracite.ai/api/mcp/mcp"
      ]
    }
  }
}
```

### Adım 4: AI Aracınızı Yeniden Başlatın

Değişikliklerin geçerli olması için AI uygulamanızı kapatıp yeniden açın.

### Adım 5: Bağlantıyı Doğrulayın

Entegrasyonu test etmek için AI asistanınıza sorun:

> Ultracite için hangi kurallar mevcut?

Başarılıysa, AI asistanınız Ultracite kurallarını listeleyip açıklayabilmelidir!

## Birden Fazla MCP Sunucusu

Ultracite'yi diğer MCP sunucularıyla birlikte kullanabilirsiniz:

```json
{
  "mcpServers": {
    "ultracite": {
      "command": "npx",
      "args": ["-y", "mcp-remote", "https://www.ultracite.ai/api/mcp/mcp"]
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"]
    },
    "filesystem": {
      "command": "npx", 
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/project"]
    }
  }
}
```

## Kullanım Örnekleri

### Mevcut Kuralları Keşfetme

AI asistanınıza sorun:

> Ultracite, React hook'ları için hangi kurallara sahip?

Beklenen yanıt, ilgili React hook lint kurallarını ve açıklamalarını içerecektir.

### Kural Kategorilerini Anlama

> Ultracite'teki tüm erişilebilirlik kurallarını gösterir misin?

AI asistanınız, Ultracite'de yapılandırılmış erişilebilirlik (a11y) kurallarını listeleyip açıklayabilir.

### Önayarları Karşılaştırma

> ultracite/core ile ultracite/react arasındaki fark nedir?

Farklı önayar yapılandırmaları hakkında ayrıntılı bilgi alabilir ve ne zaman kullanılmaları gerektiğini öğrenebilirsiniz.

## Güvenlik ve Gizlilik

### Veri İşleme

- Ultracite MCP sunucusu yalnızca genel lint kuralı dokümantasyonu ve yapılandırma bilgisi sağlar
- Kişisel veriler veya kod sunucularımıza iletilmez
- Tüm iletişim, seçtiğiniz AI aracının güvenlik katmanı üzerinden gerçekleşir

### Kimlik Doğrulama

- Genel dokümantasyona erişim için kimlik doğrulama gerekmez
- Gelecekteki premium özellikler API anahtarları gerektirebilir
- Her zaman resmi Ultracite MCP uç noktalarını kullanın
