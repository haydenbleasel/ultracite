Ultracite tarjoaa saumattoman integraation tekoälykoodiapuohjelmien kanssa generoimalla automaattisesti editorikohtaisia sääntötiedostoja. Nämä säännöt auttavat ohjaamaan tekoälykoodiapuohjelmia kirjoittamaan parempaa, johdonmukaista koodia, joka noudattaa parhaita käytäntöjä.

Editorisäännöt toimivat rinnakkain Biomen linttauksen ja muotoilun kanssa. Kun Biome huolehtii koodin muotoilusta ja havaitsee syntaksivirheitä, editorisäännöt ohjaavat tekoälykoodiapuohjelmia kirjoittamaan parempaa koodia alusta alkaen, mikä vähentää myöhempien korjausten tarvetta.

Tämä kaksijakoinen lähestymistapa varmistaa sekä automaattisen koodin laadun (Biome:n kautta) että älykkään koodin generoinnin (tekoälysääntöjen kautta).

## Määritys

### Automaattinen määritys

When you run `npx ultracite init`, you'll be prompted to select which editor rules you want to enable:

```bash
? Which editor rules do you want to enable (optional)?
  ◯ GitHub Copilot (VSCode)
  ◯ Cursor
  ◯ Windsurf
  # ... etcetera
```

This will create the relevant file in your project directory.

### Manuaalinen määritys

If you want to manually set things up, create the relevant file in your project directory (for example `.cursor/rules` for Cursor or `.rules` for Zed). Then, copy and paste [this content](https://github.com/haydenbleasel/ultracite/tree/main/.cursor/rules/ultracite.mdc) into the file.

## Hyödyt

Using Ultracite editor rules provides several advantages:

- **Johdonmukaisuus**: Kaikki tekoälyn generoima koodi noudattaa samoja käytäntöjä
- **Laatu**: Säännöt edistävät parhaita käytäntöjä ja estävät yleisiä virheitä
- **Saavutettavuus**: Sisäänrakennetut saavutettavuusohjeet varmistavat inklusiivisen koodin
- **Suorituskyky**: Säännöt suosivat tehokkaita malleja ja välttävät anti-malleja
- **Tietoturva**: Ohjeet auttavat estämään yleisiä haavoittuvuuksia

## Mukauttaminen

Vaikka oletussäännöt ovat kattavia, voit mukauttaa niitä tarpeidesi mukaan:

1. **Muokkaa olemassa olevia sääntöjä**: Muokkaa sääntötiedostoja suoraan
2. **Lisää projekti-/projektikohtaisia sääntöjä**: Lisää lisäohjeita
3. **Poista säännöt käytöstä**: Poista tai kommentoi pois säännöt, jotka eivät koske projektiasi
