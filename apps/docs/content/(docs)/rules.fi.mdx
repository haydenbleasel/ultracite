Ultracite tarjoaa saumattoman integraation tekoälyohjelmointiapulaisten kanssa luomalla automaattisesti editorikohtaisia sääntötiedostoja. Nämä säännöt ohjaavat tekoälyapulaisia kirjoittamaan parempaa, johdonmukaisempaa koodia, joka noudattaa parhaita käytäntöjä.

Editorisäännöt toimivat rinnakkain Biomen linttauksen ja muotoilun kanssa. Kun Biome huolehtii koodin muotoilusta ja havaitsee syntaksivirheitä, editorisäännöt ohjaavat tekoälyapulaisia kirjoittamaan parempaa koodia alusta lähtien, mikä vähentää myöhempien korjausten tarvetta.

Tämä kaksijakoinen lähestymistapa varmistaa sekä automatisoidun koodin laadun (Biomen kautta) että älykkään koodin generoinnin (tekoälysääntöjen kautta).

## Asetus

### Automaattinen määritys

When you run `npx ultracite init`, you'll be prompted to select which editor rules you want to enable:

```bash
? Which editor rules do you want to enable (optional)?
  ◯ GitHub Copilot (VSCode)
  ◯ Cursor
  ◯ Windsurf
  # ... etcetera
```

This will create the relevant file in your project directory.

### Manuaalinen määritys

If you want to manually set things up, create the relevant file in your project directory (for example `.cursor/rules` for Cursor or `.rules` for Zed). Then, copy and paste [tämä sisältö](https://github.com/haydenbleasel/ultracite/tree/main/.cursor/rules/ultracite.mdc) into the file.

## Hyödyt

Using Ultracite editor rules provides several advantages:

- **Johdonmukaisuus**: Kaikki tekoälyn generoima koodi noudattaa samoja käytäntöjä
- **Laatu**: Säännöt pakottavat parhaiden käytäntöjen noudattamisen ja estävät yleisiä virheitä
- **Saavutettavuus**: Sisäänrakennetut saavutettavuusohjeet varmistavat esteettömyyden huomioivan koodin
- **Suorituskyky**: Säännöt edistävät tehokkaita malleja ja estävät huonoja käytäntöjä
- **Turvallisuus**: Ohjeet auttavat estämään yleisiä tietoturva-aukkoja

## Mukauttaminen

While the default rules are comprehensive, you can customize them for your specific needs:

1. **Muokkaa olemassa olevia sääntöjä**: Muokkaa sääntötiedostoja suoraan
2. **Lisää projektikohtaisia sääntöjä**: Lisää tiedostoihin lisäohjeita
3. **Poista säännöt käytöstä**: Poista tai kommentoi pois säännöt, jotka eivät koske projektiasi
