title: Hookit

---

Ultracite voi automaattisesti muotoilla ja korjata koodia sen jälkeen, kun tekoälypohjaiset koodiapulaiset tekevät muokkauksia editori-hookkien kautta. Tämä varmistaa, että kaikki tekoälyn tuottama koodi noudattaa projektisi käytäntöjä ilman manuaalista puuttumista.

Hookit toimivat rinnakkain Biomen linttauksen ja muotoilun kanssa. Sen jälkeen kun tekoälyapuri muokkaa tiedoston, hook suorittaa automaattisesti `npx ultracite fix` muotoillakseen ja korjatakseen automaattisesti korjattavissa olevat linttausongelmat. Tämä kaksiosainen lähestymistapa varmistaa sekä älykkään koodin generoinnin (tekoälysääntöjen avulla) että automatisoidun koodin laadun (hookkien avulla).

> **Huomautus:** Hookit ovat erillisiä tekoälykoodisäännöistä. `--agents`-valitsin määrittää tekoälykoodisäännöt, kun taas `--hooks`-valitsin ottaa käyttöön automaattisen muotoilun tiedostomuokkausten jälkeen.

## Asetus

### Automaattinen määritys

Kun suoritat `npx ultracite init`, sinua pyydetään valitsemaan, mitkä editori-hookit haluat ottaa käyttöön:

```bash
? Which agent hooks do you want to enable (optional)?
  ◯ Cursor
  ◯ Claude Code
```

Tämä luo asianmukaisen hook-konfiguraatiotiedoston projektihakemistoon.

### Manuaalinen määritys

Jos haluat määrittää asiat manuaalisesti tai lisätä hookkeja alustamisen jälkeen, suorita:

```bash
npx ultracite init --hooks cursor,claude
```

Vaihtoehtoisesti voit luoda tai muokata hook-konfiguraatiotiedostoja projektihakemistossasi ja lisätä sopivan hook-komennon manuaalisesti.

## Tuetut editorit

| Editor      | Hook File Location          | Dokumentaatio                                                           |
| ----------- | --------------------------- | ----------------------------------------------------------------------- |
| Cursor      | `.cursor/hooks.json`        | [Cursor-hookit](https://cursor.com/docs/agent/hooks)                    |
| Claude Code | `.claude/settings.json`     | [Claude Code -hookit](https://docs.claude.com/en/docs/claude-code/hooks) |

## Miten se toimii

1. **Automaattinen suoritus**: Sen jälkeen kun tekoälyagentti muokkaa tiedoston, hook suorittaa automaattisesti `npx ultracite fix`
2. **Koodin muotoilu**: Ultracite muotoilee muokatun koodin Biome-määrityksesi mukaisesti
3. **Linttauksen korjaukset**: Automaattisesti korjattavissa olevat linttausongelmat ratkaistaan automaattisesti
4. **Saumaton integraatio**: Prosessi tapahtuu läpinäkyvästi häiritsemättä työskentelyäsi

## Hyödyt

Ultraciten editori-hookkien käyttö tarjoaa useita etuja:

- **Johdonmukaisuus**: Kaikki tekoälyn generoima koodi muotoillaan automaattisesti tyylisi mukaiseksi
- **Laatu**: Linttausongelmat korjataan välittömästi koodin generoinnin jälkeen
- **Tehokkuus**: Ei tarvetta suorittaa formaattoreita tai lintereitä manuaalisesti tekoälymuokkausten jälkeen
- **Standardit**: Varmistaa, että tekoälyn tuottama koodi noudattaa aina projektin vaatimuksia
- **Siistit commitit**: Koodi on asianmukaisesti muotoiltu jo ennen kuin tarkastelet muutoksia

## Mukauttaminen

Ultracite sulautuu älykkäästi olemassa oleviin hook-konfiguraatioihin:

- **Säilyttää olemassa olevat hookit**: Kaikki olemassa olevat hookit ja niiden komennot jäävät ennalleen
- **Välttää duplikaatit**: Jos Ultracite on jo konfiguroitu, sitä ei lisätä uudelleen
- **Erotteluton**: Olemassa olevan hook-konfiguraatiorakenteesi säilyy muuttumattomana

Voit myös mukauttaa hookin käyttäytymistä muokkaamalla hook-konfiguraatiotiedostoja suoraan, jos tarvitset erilaisia muotoilukomentoja tai lisätoimia muokkausten jälkeen.
