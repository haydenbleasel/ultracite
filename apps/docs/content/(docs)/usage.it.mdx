## Integrazione IDE

Ultracite funziona al meglio quando è integrato nel tuo editor, così ricevi feedback immediato mentre scrivi codice. È progettato per essere eseguito automaticamente al salvataggio. Per provarlo, apri un progetto nel tuo IDE. Man mano che modifichi i file, dovresti vedere la formattazione applicata al salvataggio. Prova a introdurre un piccolo errore di stile (come un punto e virgola in più o una riga indentata in modo errato) e premi Salva – Ultracite (Biome) riformatterà istantaneamente il file.

Se introduci un problema comune di lint (come una variabile non usata), potresti vedere una sottolineatura ondulata o un avviso nel Pannello Problemi; al salvataggio, Ultracite tenterà di correggerlo se è auto-riparabile (per esempio, rimuovere un import non usato) oppure lascerà un avviso che dovrai risolvere.

### Formattazione istantanea

Ogni volta che salvi un file, Biome (con la configurazione di Ultracite) formatta il file. Non è necessario eseguire un Prettier separato o preoccuparsi della formattazione — è tutto gestito automaticamente.

Grazie alle impostazioni `codeActionsOnSave`, VS Code applicherà anche l'azione "fix all" di Biome al salvataggio. Questo significa che qualsiasi regola di lint che dispone di un'auto-correzione (come convertire `!=` in `!==`, aggiungere parentesi mancanti, correggere l'ordine degli import, ecc.) verrà applicata. È come se avessi eseguito automaticamente `--fix` con ESLint.

Abbiamo anche abilitato `formatOnPaste` – quindi se incolli codice da un'altra fonte, verrà immediatamente formattato per adeguarsi al tuo stile.

Il vantaggio principale qui è il feedback immediato. Scrivi codice in VS Code e Ultracite lo mantiene continuamente pulito. Col tempo noterai di dedicare poco o nessun tempo a correggere errori di lint — Ultracite o li sistema per te o li segnala precocemente. Questo conduce a un flusso di lavoro di sviluppo fluido in cui lo stile del codice e i problemi minori non si accumulano mai.

### Integrazione con il Pannello Problemi

Eventuali problemi rimanenti (che richiedono la tua attenzione) appariranno nel Pannello Problemi. Per esempio, se Ultracite ha una regola che proibisce qualcosa o trova un errore che non può correggere (ad esempio, hai usato un'API deprecata o hai dimenticato di gestire il rifiuto di una Promise), verrà elencato come errore o avviso. Puoi cliccarlo per saltare alla posizione nel codice.

### Code lenses / Correzioni rapide

In molti casi, l'estensione VS Code per Biome fornisce suggerimenti di correzione rapida. Se vedi una lampadina gialla o un popup di suggerimento, puoi applicare le correzioni manualmente. Per esempio, se un import non è usato, potresti ottenere una correzione rapida per rimuoverlo (se non è già stato rimosso al salvataggio).

## Utilizzo della CLI

Ultracite include una CLI comoda (invocata come `ultracite` quando installata). Ecco i modi più comuni per usarla.

### Verifica del codice

Il comando `check` avvolge il comando `check` di Biome, che esegue il linter senza modificare i file.

```bash title="Terminal"
npx ultracite check
```

Puoi passare il filtro `--diagnostic-level` per mostrare solo errori o avvisi:

```bash title="Terminal"
# Show only errors
npx ultracite check --diagnostic-level error

# Show warnings and above (errors)
npx ultracite check --diagnostic-level warn
```

### Correzione del codice

Il comando `fix` avvolge il comando `check --write` di Biome, che esegue il linter e corregge i file.

```bash title="Terminal"
npx ultracite fix
```

Puoi anche applicare correzioni non sicure eseguendo:

```bash title="Terminal"
npx ultracite fix --unsafe
```

### Validazione dell'installazione

Il comando `doctor` verifica la tua configurazione alla ricerca di problemi e fornisce raccomandazioni. È utile eseguirlo dopo aver installato Ultracite per assicurarti che tutto sia configurato correttamente.

```bash title="Terminal"
npx ultracite doctor
```
