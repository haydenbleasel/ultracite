Ultracite biedt naadloze integratie met AI-codeassistenten door automatisch editorspecifieke regelbestanden te genereren. Deze regels helpen AI-codeassistenten om betere, consistenter geschreven code te produceren die de beste praktijken volgt.

De editorregels werken naast Biome's linting en formattering. Terwijl Biome codeformattering verzorgt en syntaxfouten opvangt, begeleiden de editorregels AI-codeassistenten zodat ze vanaf het begin betere code schrijven, waardoor de noodzaak voor latere correcties afneemt.

Deze dubbele aanpak waarborgt zowel geautomatiseerde codekwaliteit (via Biome) als intelligente codegeneratie (via AI-coderegels).

## Installatie

### Automatische installatie

Wanneer je `npx ultracite init` uitvoert, krijg je een prompt om te selecteren welke editorregels je wilt inschakelen:

```bash
? Which editor rules do you want to enable (optional)?
  ◯ GitHub Copilot (VSCode)
  ◯ Cursor
  ◯ Windsurf
  # ... etcetera
```

Dit maakt het relevante bestand in je projectmap aan.

### Handmatige installatie

Als je handmatig wilt instellen, maak dan het relevante bestand aan in je projectmap (bijvoorbeeld `.cursor/rules` voor Cursor of `.rules` voor Zed). Kopieer en plak vervolgens [deze inhoud](https://github.com/haydenbleasel/ultracite/tree/main/.cursor/rules/ultracite.mdc) in het bestand.

## Voordelen

Het gebruik van Ultracite-editorregels biedt verschillende voordelen:

- **Consistentie**: Alle door AI gegenereerde code volgt dezelfde conventies
- **Kwaliteit**: Regels dwingen beste praktijken af en voorkomen veelvoorkomende fouten
- **Toegankelijkheid**: Ingebouwde toegankelijkheidsrichtlijnen zorgen voor inclusieve code
- **Prestaties**: Regels promoten efficiënte patronen en vermijden antipatronen
- **Beveiliging**: Richtlijnen helpen veelvoorkomende beveiligingskwetsbaarheden te voorkomen

## Aanpassing

Hoewel de standaardregels uitgebreid zijn, kun je ze aanpassen aan je specifieke behoeften:

1. **Bewerk bestaande regels**: Bewerk de regelbestanden direct
2. **Voeg projectspecifieke regels toe**: Voeg aanvullende richtlijnen toe
3. **Schakel regels uit**: Verwijder of commentarieer regels die niet op je project van toepassing zijn
