Ultracite biedt naadloze integratie met AI-codeassistenten door automatisch editor-specifieke regelbestanden te genereren. Deze regels helpen AI-codeassistenten te begeleiden zodat ze betere, meer consistente code schrijven die de beste praktijken volgt.

De editorregels werken naast Biome's linting en formattering. Terwijl Biome codeformattering afhandelt en syntaxfouten opvangt, zorgen de editorregels ervoor dat AI-codeassistenten vanaf het begin betere code genereren, waardoor de behoefte aan latere correcties afneemt.

Deze dubbele aanpak zorgt zowel voor geautomatiseerde codekwaliteit (via Biome) als voor intelligente codegeneratie (via AI-code regels).

## Installatie

### Automatische installatie

Wanneer je `npx ultracite init` uitvoert, wordt je gevraagd welke editorregels je wilt inschakelen:

```bash
? Which editor rules do you want to enable (optional)?
  ◯ GitHub Copilot (VSCode)
  ◯ Cursor
  ◯ Windsurf
  # ... etcetera
```

Dit maakt het relevante bestand in je projectmap aan.

### Handmatige installatie

Als je het handmatig wilt instellen, maak dan het relevante bestand in je projectmap aan (bijvoorbeeld `.cursor/rules` voor Cursor of `.rules` voor Zed). Kopieer en plak vervolgens [deze inhoud](https://github.com/haydenbleasel/ultracite/tree/main/.cursor/rules/ultracite.mdc) in het bestand.

## Voordelen

Het gebruik van Ultracite-editorregels biedt verschillende voordelen:

- **Consistentie**: Alle door AI gegenereerde code volgt dezelfde conventies
- **Kwaliteit**: Regels handhaven beste praktijken en voorkomen veelvoorkomende fouten
- **Toegankelijkheid**: Ingebouwde richtlijnen voor toegankelijkheid zorgen voor inclusieve code
- **Prestaties**: Regels stimuleren efficiënte patronen en vermijden anti-patronen
- **Beveiliging**: Richtlijnen helpen veelvoorkomende beveiligingskwetsbaarheden te voorkomen

## Aanpassing

Hoewel de standaardregels uitgebreid zijn, kun je ze aanpassen aan je specifieke behoeften:

1. **Bestaande regels aanpassen**: Bewerk de regelbestanden direct
2. **Project-specifieke regels toevoegen**: Voeg extra richtlijnen toe
3. **Regels uitschakelen**: Verwijder of commentarieer regels die niet op je project van toepassing zijn
