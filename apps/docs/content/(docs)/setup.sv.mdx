Ultracite kan installeras med alla större pakethanterare. Du behöver Node.js v14.18+ (Node 18+ rekommenderas) och ett projekt med en `package.json`-fil.

## Installation

Det enklaste sättet att komma igång är att köra Ultracites `init`-skript, som kommer att installera nödvändiga paket och skapa konfigurationsfilen åt dig. Kör i din projektmapp:

```package-install
npx ultracite init
```

Du kan valfritt skicka följande flaggor till `init`-kommandot:

| Flagga              | Beskrivning                                     | Alternativ                                                               |
|---------------------|--------------------------------------------------|--------------------------------------------------------------------------|
| `--pm`              | Den pakethanterare som ska användas.             | `pnpm`, `bun`, `yarn`, `npm`                                              |
| `--frameworks`      | De ramverk som ska konfigureras.                 | Komma-separerade: `react`, `next`, `solid`, `vue`, `qwik`, `angular`, `remix`, `svelte` |
| `--editors`         | De redigerare som ska konfigureras.              | Komma-separerade: `vscode`, `zed`                                         |
| `--agents`          | De AI-agenter som ska konfigureras.              | Komma-separerade: `vscode-copilot`, `cursor`, `windsurf`, `zed`, `claude`, `codex`, `kiro`, `cline`, `amp`, `aider`, `firebase-studio`, `open-hands`, `gemini-cli`, `junie`, `augmentcode`, `kilo-code`, `goose`, `roo-code`, `warp`, `droid` |
| `--hooks`           | Agent hooks som ska konfigureras.                | Komma-separerade: `cursor`, `claude`                                      |
| `--integrations`    | Ytterligare integrationer att konfigurera.        | Komma-separerade: `husky`, `lefthook`, `lint-staged`                      |
| `--migrate`         | Migrationverktyg som ska tas bort.               | Komma-separerade: `eslint`, `prettier` (tar bort beroenden, konfigurationsfiler och editorinställningar) |
| `--skip-install`    | Hoppa över installation av beroenden.             | (inget värde krävs)                                                       |
| `--quiet`           | Undertryck interaktiva prompts och visuell utdata.| (inget värde behövs, aktiveras automatiskt i CI)                          |

### Programmatisk användning

För verktyg som behöver anropa Ultracite programmässigt (t.ex. scaffolding-CLI:er), kan du använda flaggan `--quiet` för att undertrycka alla interaktiva prompts och visuell utdata:

```bash title="Terminal"
npx ultracite init --quiet --pm npm
```

Flaggan `--quiet` aktiveras automatiskt när miljövariabeln `CI` är satt till `true` eller `1`, vilket gör den lämplig för CI/CD-pipelines:

```bash title="Terminal"
CI=true npx ultracite init --pm npm
```

I tyst läge kommer Ultracite att:
- Hoppa över alla interaktiva prompts
- Använda rimliga standardvärden för alla alternativ
- Undertrycka ASCII-art, snurror och loggmeddelanden
- Endast skapa den grundläggande Biome-konfigurationen
- Avsluta rent med kod 0 vid framgång eller 1 vid fel

## Tillägg

Om du använder Visual Studio Code, installera det officiella [Biome](https://marketplace.visualstudio.com/items?itemName=biomejs.biome)-tillägget, som möjliggör formatering/fixning vid sparande:

```bash title="Terminal"
code --install-extension biomejs.biome
```

Dessutom, om du använder Tailwind CSS, rekommenderas att installera [Tailwind CSS IntelliSense](https://marketplace.visualstudio.com/items?itemName=bradlc.vscode-tailwindcss)-tillägget för klassortering och förslag:

```bash title="Terminal"
code --install-extension bradlc.vscode-tailwindcss
```
