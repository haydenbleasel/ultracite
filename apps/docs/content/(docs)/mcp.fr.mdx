Le **Protocole de Contexte de Modèle (MCP)** est une norme ouverte qui permet aux assistants IA comme Claude, Cursor et autres outils alimentés par l'IA de se connecter en toute sécurité à des sources de données et systèmes externes. Pensez-y comme à une "télécommande universelle" qui permet à vos outils IA d'accéder à des données et fonctionnalités réelles.

Ultracite prend en charge MCP pour optimiser votre flux de travail de développement IA.

## Guide d'installation

### Étape 1 : Choisissez votre outil d'IA

Tout d'abord, assurez-vous d'utiliser un outil de développement IA qui prend en charge MCP :

- [Claude Desktop](https://claude.ai/download) (Gratuit - recommandé pour les débutants)
- [Cursor](https://www.cursor.com/) (Éditeur de code assisté par IA)
- [Windsurf by Codeium](https://windsurf.com/) (Plateforme de développement IA)
- Autres outils compatibles MCP

### Étape 2 : Localisez votre fichier de configuration

Selon votre outil d'IA, vous devrez modifier l'un de ces fichiers :

- **Claude Desktop**: `~/Library/Application\ Support/Claude/claude_desktop_config.json` pour macOS et `%APPDATA%\Claude\claude_desktop_config.json` pour Windows
- **Cursor**: `.cursor/mcp.json`
- **Windsurf**: `.codeium/windsurf/mcp_config.json`
- **Autres outils**: Consultez la documentation MCP de votre outil

### Étape 3 : Ajoutez la configuration Ultracite

Copiez et collez cette configuration dans votre fichier de configuration MCP :

```json
{
  "mcpServers": {
    "ultracite": {
      "command": "npx",
      "args": [
        "-y",
        "mcp-remote",
        "https://www.ultracite.ai/api/mcp/mcp"
      ]
    }
  }
}
```

### Étape 4 : Redémarrez votre outil d'IA

Fermez et rouvrez votre application IA pour que les modifications prennent effet.

### Étape 5 : Vérifiez la connexion

Testez l'intégration en demandant à votre assistant IA :

> Quelles règles Ultracite sont disponibles ?

En cas de succès, votre IA devrait pouvoir lister et expliquer les règles d'Ultracite !

## Plusieurs serveurs MCP

Vous pouvez utiliser Ultracite aux côtés d'autres serveurs MCP :

```json
{
  "mcpServers": {
    "ultracite": {
      "command": "npx",
      "args": ["-y", "mcp-remote", "https://www.ultracite.ai/api/mcp/mcp"]
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"]
    },
    "filesystem": {
      "command": "npx", 
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/project"]
    }
  }
}
```

## Exemples d'utilisation

### Explorer les règles disponibles

Demandez à votre assistant IA :

> Quelles règles Ultracite propose pour les hooks React ?

La réponse attendue inclura les règles de linting pertinentes pour les hooks React et leurs descriptions.

### Comprendre les catégories de règles

> Montrez-moi toutes les règles d'accessibilité dans Ultracite

Votre IA peut lister et expliquer les règles a11y configurées dans Ultracite.

### Comparer les préréglages

> Quelle est la différence entre ultracite/core et ultracite/react ?

Obtenez des informations détaillées sur les différentes configurations préréglées et quand les utiliser.

## Sécurité et confidentialité

### Traitement des données

- Le serveur MCP d'Ultracite ne fournit que la documentation publique des règles de linting et les informations de configuration
- Aucune donnée personnelle ni aucun code n'est transmis à nos serveurs
- Toute la communication passe par la couche de sécurité de l'outil d'IA que vous avez choisi

### Authentification

- Aucune authentification requise pour l'accès à la documentation publique
- Des fonctionnalités premium futures pourront nécessiter des clés API
- Utilisez toujours les endpoints MCP officiels d'Ultracite
