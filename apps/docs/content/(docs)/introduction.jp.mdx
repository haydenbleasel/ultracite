---

Ultracite は、高速で直感的かつシンプルな開発ツールで、JavaScript / TypeScript プロジェクトに自動のコード整形とリンティングをもたらします。[Biome](https://biomejs.dev/)（Rust ベースのコードフォーマッタ兼リンター）の上に構築されており、Ultracite はモダンなウェブ開発に最適化された堅牢な事前構成セットアップを提供します。

次のコマンドを実行して Ultracite を素早くインストールします:

```package-install
npx ultracite init
```

その後、開始するには [使い方](/usage) ガイドに従うか、より細かい制御が必要な場合は [セットアップ](/setup) を参照してください。

## Ultracite はどのように動作しますか？

セットアップが完了すると、Ultracite は主にバックグラウンドで動作します — ファイルを保存するたびに自動的にコードを整形し、修正を適用します。Biome は非常に高速なため、大規模なプロジェクトでも Ultracite のチェックは極めて高速に実行でき、保存ごとに遅延なく実行することが可能です。これにより、スタイルの手動修正やランタイムエラーのデバッグに時間を割くのではなく、構築とリリースに集中できます。

## Biome とは何ですか？

[Biome](https://biomejs.dev/) は、コード整形とリンティングのためのオールインワンツールチェーンです。Rust で書かれた Biome は、[Prettier](https://prettier.io/)（コードフォーマッタ）、[ESLint](https://eslint.org/)（リンター）、[Stylelint](https://stylelint.io/) のようなツールの機能を単一の高速なバイナリに統合します。Ultracite は Biome の速度と能力を活用し、モダンな TypeScript プロジェクト向けに調整されたルールと設定のプリセットを提供します。

## 目標

開発上の観点から見た Ultracite の目標は次のとおりです：

### 非常に高速なパフォーマンス

Ultracite は [Biome](https://biomejs.dev/) のパフォーマンスと Rust 実装の恩恵を受けており、その状態を維持するよう努めるべきです。

### デフォルトで設定不要

Ultracite はデフォルトで設定不要（zero-config）となるよう設計されています。これにより、新しいユーザーはリンターやフォーマッタを設定することなく迅速に始められ、チームが新しい開発者をオンボーディングする際の時間を節約できます。

目標は、ほとんどのプロジェクトにとって十分に良いデフォルト設定を提供すると同時に、必要な人向けに設定をカスタマイズする方法も提供することです。

### 直感的でシンプル

Ultracite は可能な限り可視性を抑えるべきです。理想的には、ユーザーの設定や操作を必要としないことが望ましいです。必要な場合は、理解しやすく実行しやすい方法でフィードバックを提供するべきです。

### 最大限の型安全性

Ultracite は可能な限り型安全になるよう設計されるべきです。TypeScript の strict モードや追加のリントルールを使用してエラーを検出し、改善のための提案を行います。

### 他のツールと協調する

Ultracite は他のツールやライブラリと共に動作でき、競合しないように設計されるべきです。
