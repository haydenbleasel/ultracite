## Integrare IDE

Ultracite funcționează cel mai bine atunci când este integrat în editorul tău, astfel încât primești feedback imediat pe măsură ce scrii cod. Este conceput să ruleze automat la salvare. Pentru a-l testa, deschide un proiect în IDE-ul tău. Pe măsură ce editezi fișiere, ar trebui să vezi formatarea aplicându-se la salvare. Încearcă să introduci o mică greșeală de stil (cum ar fi un punct și virgulă în plus sau o linie indentată greșit) și salvează — Ultracite (Biome) va reformat instantaneu fișierul.

Dacă introduci o eroare de lint comună (de exemplu o variabilă neutilizată), s-ar putea să vezi o subliniere ondulată sau un avertisment în panoul Problems; la salvare, Ultracite va încerca să o remedieze dacă este auto-remediabilă (de exemplu eliminarea unui import neutilizat) sau, în caz contrar, va lăsa un avertisment pentru a fi abordat de tine.

### Formatare instantanee

De fiecare dată când salvezi un fișier, Biome (cu configurația Ultracite) formatează fișierul. Nu trebuie să rulezi un Prettier separat sau să te îngrijorezi de formatare — se ocupă automat.

Datorită setărilor `codeActionsOnSave`, VS Code va aplica de asemenea acțiunea Biome „fix all” la salvare. Aceasta înseamnă că orice regulă de lint care are o remediere automată (cum ar fi conversia `!=` în `!==`, adăugarea parantezelor lipsă, corectarea ordinii importurilor etc.) va fi aplicată. Este ca și cum ai rula `--fix` cu ESLint în mod automat.

Am activat chiar și `formatOnPaste` — așa că dacă inserezi cod din altă parte, acesta va fi formatat imediat pentru a se potrivi stilului tău.

Principalul avantaj aici este feedback-ul instantaneu. Scrii cod în VS Code, iar Ultracite îl menține curat în mod continuu. În timp, vei observa că petreci puțin sau deloc timp pentru a corecta erorile de lint — Ultracite fie le remediează pentru tine, fie le semnalează devreme. Acest lucru conduce la un flux de lucru de dezvoltare fluid în care stilul codului și problemele minore nu se acumulează.

### Integrarea panoului Problems

Orice probleme rămase (care necesită atenția ta) vor apărea în panoul Problems. De exemplu, dacă Ultracite are o regulă care interzice ceva sau găsește o eroare pe care nu o poate remedia (să zicem că ai folosit o API depreciată sau ai uitat să gestionezi o respingere de Promise), le va lista ca eroare sau avertisment. Poți face clic pe ele pentru a sări la locația din cod.

### Code lenses / Quick fixes

În multe cazuri, extensia VS Code pentru Biome oferă sugestii de remedieri rapide. Dacă vezi o pictogramă galbenă în formă de bec sau un popup de sugestii, poți aplica remedierile și manual. De exemplu, dacă un import este neutilizat, s-ar putea să primești o remediere rapidă pentru a-l elimina (dacă nu a fost deja eliminat la salvare).

## Utilizare CLI

Ultracite vine cu un CLI convenabil (invocat ca `ultracite` când este instalat). Iată câteva moduri comune de utilizare.

### Verificarea codului

Comanda `check` învelește comanda Biome `check`, care rulează linter-ul fără a modifica fișierele.

```bash title="Terminal"
npx ultracite check
```

Poți transmite filtrul `--diagnostic-level` pentru a afișa doar erori sau avertismente:

```bash title="Terminal"
# Show only errors
npx ultracite check --diagnostic-level error

# Show warnings and above (errors)
npx ultracite check --diagnostic-level warn
```

### Remedierea codului

Comanda `fix` învelește comanda Biome `check --write`, care rulează linter-ul și remediază fișierele.

```bash title="Terminal"
npx ultracite fix
```

Poți aplica, de asemenea, remedieri nesigure rulând:

```bash title="Terminal"
npx ultracite fix --unsafe
```

### Validarea configurației

Comanda `doctor` verifică setup-ul tău pentru probleme și oferă recomandări. Este util să o rulezi după instalarea Ultracite pentru a te asigura că totul este configurat corect.

```bash title="Terminal"
npx ultracite doctor
```
