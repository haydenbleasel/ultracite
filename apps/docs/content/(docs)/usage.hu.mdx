## IDE integráció

Az Ultracite akkor működik a legjobban, ha a szerkesztődbe van integrálva, így azonnali visszajelzést kapsz írás közben. Arra tervezték, hogy mentéskor automatikusan fusson. Ha ki szeretnéd próbálni, nyiss meg egy projektet az IDE-dben. Ahogy szerkeszted a fájlokat, a mentéskor meg kell jelennie a formázásnak. Próbálj bevezetni egy kisebb kódstílusbeli hibát (például egy plusz pontosvesszőt vagy egy rosszul behúzott sort), majd mentsd el – az Ultracite (Biome) azonnal átrendezi a fájlt.

Ha bevezetsz egy gyakori lint problémát (például egy nem használt változót), előfordulhat, hogy hullámos aláhúzást vagy figyelmeztetést látsz a Problémák panelen; mentéskor az Ultracite megkísérli kijavítani, ha automatikusan javítható (például eltávolít egy nem használt importot), különben figyelmeztetést hagy, hogy te foglalkozz vele.

### Azonnali formázás

Minden fájl mentésekor a Biome (az Ultracite konfigurációval) formázza a fájlt. Nem kell külön Prettier-t futtatnod vagy a formázáson aggódnod — ez automatikusan megoldódik.

A `codeActionsOnSave` beállításoknak köszönhetően a VS Code mentéskor alkalmazza a Biome „mindent javít” műveletét is. Ez azt jelenti, hogy bármely lint szabály, amelyhez automatikus javítás tartozik (például `!=` `!==`-re cserélése, hiányzó zárójelek hozzáadása, import sorrend javítása stb.), alkalmazásra kerül. Olyan, mintha automatikusan lefuttattad volna az ESLint `--fix` opcióját.

Bekapcsoltuk a `formatOnPaste`-t is — így ha máshonnan beillesztesz kódot, az azonnal a stílusodhoz igazítva lesz formázva.

A kulcsfontosságú előny az azonnali visszajelzés. VS Code-ban írsz kódot, és az Ultracite folyamatosan tisztán tartja azt. Idővel észre fogod venni, hogy szinte semennyi időt sem töltesz lint hibák javításával — az Ultracite vagy kijavítja őket helyetted, vagy korán felhívja rájuk a figyelmet. Ez sima fejlesztési munkafolyamatot eredményez, ahol a kódstílus és az apróbb problémák sosem halmozódnak fel.

### Problémák panel integrációja

Minden olyan probléma, ami továbbra is figyelmet igényel, megjelenik a Problémák panelen. Például ha az Ultracite egy szabálya tilt valamit, vagy olyan hibát talál, amit nem tud automatikusan javítani (például elavult API-t használsz, vagy elfelejtetted kezelni egy Promise elutasítását), azt hibaként vagy figyelmeztetésként listázza. Rákattintva az adott helyre ugorhatsz a kódban.

### Code lens-ek / Gyorsjavítások

Sok esetben a Biome VS Code kiterjesztése gyorsjavítási javaslatokat ad. Ha sárga villanykörte ikont vagy egy javaslat felugró ablakot látsz, kézzel is alkalmazhatod a javításokat. Például ha egy import nincs használatban, kaphatsz egy gyorsjavítást annak eltávolítására (ha azt mentéskor még nem távolította el).

## CLI használata

Az Ultracite egy kényelmes CLI-vel érkezik (telepítés után `ultracite`-ként hívható). Íme néhány gyakori módja a használatának.

### Kód ellenőrzése

A `check` parancs a Biome `check` parancsát csomagolja be, amely a lintert futtatja fájlok javítása nélkül.

```bash title="Terminal"
npx ultracite check
```

Átadhatsz egy `--diagnostic-level` szűrőt, hogy csak hibákat vagy figyelmeztetéseket mutasson:

```bash title="Terminal"
# Csak hibák mutatása
npx ultracite check --diagnostic-level error

# Figyelmeztetések és fölöttük lévők (hibák) mutatása
npx ultracite check --diagnostic-level warn
```

### Kód javítása

A `fix` parancs a Biome `check --write` parancsát csomagolja be, amely a lintert futtatja és javítja a fájlokat.

```bash title="Terminal"
npx ultracite fix
```

Veszélyes javításokat is alkalmazhatsz a következő futtatásával:

```bash title="Terminal"
npx ultracite fix --unsafe
```

### Beállítás ellenőrzése

A `doctor` parancs átvizsgálja a beállításaidat problémák után, és ajánlásokat ad. Hasznos lefuttatni az Ultracite telepítése után, hogy megbizonyosodj arról, minden helyesen van-e konfigurálva.

```bash title="Terminal"
npx ultracite doctor
```
