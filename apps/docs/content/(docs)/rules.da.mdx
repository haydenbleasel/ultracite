Ultracite tilbyder problemfri integration med AI-kodeassistenter ved automatisk at generere editor-specifikke regel-filer. Disse regler hjælper med at vejlede AI-kodeassistenter til at skrive bedre, mere konsekvent kode, der følger bedste praksis.

Editorreglerne fungerer side om side med Biome's linting og formatering. Hvor Biome tager sig af kodeformatering og fanger syntaksfejl, guider editorreglerne AI-kodeassistenter til at skrive bedre kode fra starten, hvilket reducerer behovet for rettelser senere.

Denne dobbelte tilgang sikrer både automatiseret kodekvalitet (gennem Biome) og intelligent kodegenerering (gennem AI-koderegler).

## Opsætning

### Automatisk opsætning

When you run `npx ultracite init`, you'll be prompted to select which editor rules you want to enable:

```bash
? Which editor rules do you want to enable (optional)?
  ◯ GitHub Copilot (VSCode)
  ◯ Cursor
  ◯ Windsurf
  # ... etcetera
```

This will create the relevant file in your project directory.

### Manuel opsætning

Hvis du vil sætte tingene op manuelt, så opret den relevante fil i dit projektmappe (for eksempel `.cursor/rules` for Cursor eller `.rules` for Zed). Kopiér derefter og indsæt [this content](https://github.com/haydenbleasel/ultracite/tree/main/.cursor/rules/ultracite.mdc) i filen.

## Fordele

Ved at bruge Ultracite editorregler får du flere fordele:

- **Konsistens**: Al AI-genereret kode følger de samme konventioner
- **Kvalitet**: Regler håndhæver bedste praksis og forhindrer almindelige fejl
- **Tilgængelighed**: Indbyggede retningslinjer for tilgængelighed sikrer inkluderende kode
- **Ydeevne**: Regler fremmer effektive mønstre og undgår antimønstre
- **Sikkerhed**: Retningslinjer hjælper med at forhindre almindelige sikkerhedssårbarheder

## Tilpasning

Selvom standardreglerne er omfattende, kan du tilpasse dem til dine specifikke behov:

1. **Rediger eksisterende regler**: Rediger regelfilerne direkte
2. **Tilføj projektspecifikke regler**: Tilføj yderligere retningslinjer
3. **Deaktiver regler**: Fjern eller kommenter ud regler, der ikke gælder for dit projekt
