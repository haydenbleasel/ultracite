Ultracite leverer problemfri integration med AI-kodeassistenter ved automatisk at generere editor-specifikke regelfiler. Disse regler hjælper med at vejlede AI-kodeassistenter til at skrive bedre, mere konsistent kode, som følger bedste praksis.

Editorreglerne fungerer sammen med Biomes linting og formatering. Mens Biome håndterer kodeformatering og fanger syntaksfejl, vejleder editorreglerne AI-kodeassistenter til at skrive bedre kode fra starten, hvilket reducerer behovet for rettelser senere.

Denne dobbelte tilgang sikrer både automatiseret kodekvalitet (gennem Biome) og intelligent kodegenerering (gennem regler for AI-kodning).

## Opsætning

### Automatisk opsætning

Når du kører `npx ultracite init`, vil du blive bedt om at vælge, hvilke editorregler du vil aktivere:

```bash
? Which editor rules do you want to enable (optional)?
  ◯ GitHub Copilot (VSCode)
  ◯ Cursor
  ◯ Windsurf
  # ... etcetera
```

Dette vil oprette den relevante fil i din projektmappe.

### Manuel opsætning

Hvis du ønsker at opsætte ting manuelt, skal du oprette den relevante fil i din projektmappe (for eksempel `.cursor/rules` for Cursor eller `.rules` for Zed`). Kopier derefter og indsæt [dette indhold](https://github.com/haydenbleasel/ultracite/tree/main/.cursor/rules/ultracite.mdc) i filen.

## Fordele

Brug af Ultracite editorregler giver flere fordele:

- **Konsistens**: Al AI-genereret kode følger de samme konventioner
- **Kvalitet**: Regler håndhæver bedste praksis og forebygger almindelige fejl
- **Tilgængelighed**: Indbyggede retningslinjer for tilgængelighed sikrer inkluderende kode
- **Ydeevne**: Regler fremmer effektive mønstre og undgår anti-mønstre
- **Sikkerhed**: Retningslinjer hjælper med at forhindre almindelige sikkerhedssårbarheder

## Tilpasning

Selvom standardreglerne er omfattende, kan du tilpasse dem til dine specifikke behov:

1. **Rediger eksisterende regler**: Rediger regelfilerne direkte
2. **Tilføj projektspecifikke regler**: Tilføj yderligere retningslinjer
3. **Deaktiver regler**: Fjern eller kommenter regler, som ikke gælder for dit projekt
