Ultracite tilbyder problemfri integration med AI-kodeassistenter ved automatisk at generere editor-specifikke regel-filer. Disse regler hjælper med at vejlede AI-kodeassistenter til at skrive bedre, mere konsekvent kode, der følger best practices.

Editorreglerne fungerer sammen med Biome's linting og formatering. Mens Biome håndterer kodeformatering og opdager syntaksfejl, vejleder editorreglerne AI-kodeassistenter til at skrive bedre kode fra starten, hvilket reducerer behovet for rettelser senere.

Denne dobbelte tilgang sikrer både automatiseret kodekvalitet (via Biome) og intelligent kodegenerering (gennem AI-kodningsregler).

## Opsætning

### Automatisk opsætning

Når du kører `npx ultracite init`, bliver du bedt om at vælge, hvilke editorregler du vil aktivere:

```bash
? Which editor rules do you want to enable (optional)?
  ◯ GitHub Copilot (VSCode)
  ◯ Cursor
  ◯ Windsurf
  # ... etcetera
```

Dette vil oprette den relevante fil i dit projektkatalog.

### Manuel opsætning

Hvis du vil sætte tingene op manuelt, skal du oprette den relevante fil i dit projektkatalog (for eksempel `.cursor/rules` for Cursor eller `.rules` for Zed). Kopiér og indsæt derefter [dette indhold](https://github.com/haydenbleasel/ultracite/tree/main/.cursor/rules/ultracite.mdc) i filen.

## Fordele

Brug af Ultracite-editorregler giver flere fordele:

- **Konsistens**: Al AI-genereret kode følger de samme konventioner
- **Kvalitet**: Regler håndhæver best practices og forhindrer almindelige fejl
- **Tilgængelighed**: Indbyggede retningslinjer for tilgængelighed sikrer inkluderende kode
- **Ydeevne**: Regler fremmer effektive mønstre og undgår anti-mønstre
- **Sikkerhed**: Retningslinjer hjælper med at forhindre almindelige sikkerhedssårbarheder

## Tilpasning

Selvom standardreglerne er omfattende, kan du tilpasse dem til dine specifikke behov:

1. **Ændr eksisterende regler**: Rediger regel-filerne direkte
2. **Tilføj projekt-specifikke regler**: Tilføj yderligere retningslinjer
3. **Deaktiver regler**: Fjern eller kommenter regler, som ikke gælder for dit projekt
