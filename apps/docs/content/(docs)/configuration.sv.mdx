En av Ultracites bästa funktioner är att den kräver ingen konfiguration från början – du kan helt enkelt utöka `ultracite`-presettet och komma igång. Däremot är varje projekt lite annorlunda, och du kanske vill anpassa vissa regler eller anpassa Ultracite för olika ramverk. Detta avsnitt förklarar standardkonfigurationen och hur du kan utöka eller åsidosätta den.

Ultracite tillhandahåller ramverksspecifika konfigurationer som du kan utöka utöver baskonfigurationen. Detta gör att du kan lägga till ramverksspecifika lint-regler utan att göra baskonfigurationen tung eller hantera irrelevanta regler.

Du hittar den fullständiga listan över tillgängliga ramverkskonfigurationer i avsnittet [Förinställningar](/preset/core).

### Användning

För att använda en ramverksspecifik konfiguration, lägg till den i din `extends`-array:

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/solid"]
}
```

Du kan kombinera flera konfigurationer om du arbetar i ett multi-ramverksprojekt eller använder något som Astro som stöder flera ramverk.

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/react", "ultracite/remix"]
}
```

## Standardkonfiguration

När du inkluderar `"extends": ["ultracite/core"]` i din `biome.jsonc` hämtar du Ultracites baspresetkonfiguration. Detta preset innehåller ramverksoberoende regler och inställningar. Några anmärkningsvärda aspekter av standardkonfigurationen:

- **TypeScripts strikthet**: Ultracite aktiverar TypeScripts strikta kontroller och lint-regler som kräver robust typning. Till exempel avråder det från användning av `any`, kräver hantering av `null/undefined` och föredrar explicita typer i vissa situationer för att undvika tvetydighet.
- **ESLint "Recommended"-motsvarigheter**: Vanliga bästa praxis-regler (inga oanvända variabler, ingen explicita eval, ingen proto-pollution, etc.) ingår.
- **Tillgänglighet**: Regler motsvarande tillgänglighetsstandarder är aktiverade som standard (ARIA-attribut, semantisk HTML, tangentbordsnavigering, etc.).
- **Node.js**: Miljö-specifika egenheter hanteras. Till exempel känner systemet igen globala variabler som tillhandahålls av Node (som `module`, `process`) så att de inte felaktigt markeras som odefinierade.
- **Formatkonventioner**: Ultracites formatteringsinställningar (till stor del ärvda från Biome/Prettier-standarder) inkluderar 2-mellanslagsindentering, max radbredd på 80 tecken, semikolon i slutet av satser, avslutande kommatecken där det är giltigt (t.ex. i flerradiga arrayer/objekt), dubbla citattecken för strängar, etc. Dessa har valts för att matcha vanliga konventioner och säkerställa rena diffar.

Alla dessa standardinställningar syftar till att upprätthålla konsekvent stil och förhindra vanliga fel utan att du behöver konfigurera varje regel manuellt.

För en fullständig referens över alla lint-regler som är konfigurerade i Ultracites baskonfiguration, se dokumentationen för [Kärnkonfiguration](/preset/core).

## Åsidosätta konfigurationen

### Ändra `biome.jsonc`

Du kan inaktivera regler genom att ändra filen `biome.jsonc`.

Till exempel, för att inaktivera regeln `noAutofocus` kan du lägga till följande:

```jsonc title="biome.jsonc"
{
  "linter": {
    "rules": {
      "a11y": {
        "noAutofocus": "off"
      }
    }
  }
}
```

### Inline-kommentarer

Du kan också inaktivera regler per rad genom att lägga till en kommentar i slutet av raden:

```tsx title="page.tsx"
// biome-ignore lint/security/noDangerouslySetInnerHtml: This is a valid use case.
<div dangerouslySetInnerHTML={{ ... }} />
```
