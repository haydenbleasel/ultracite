Ultracite proporciona integración sin fisuras con asistentes de codificación IA generando automáticamente archivos de reglas específicos para cada editor. Estas reglas ayudan a guiar a los asistentes de codificación IA para que escriban código mejor, más consistente y que siga las mejores prácticas.

Las reglas del editor funcionan junto con el linting y el formateo de Biome. Mientras que Biome se encarga del formateo del código y de detectar errores de sintaxis, las reglas del editor guían a los asistentes de codificación IA para que escriban mejor código desde el principio, reduciendo la necesidad de correcciones posteriores.

Este enfoque dual garantiza tanto la calidad automatizada del código (a través de Biome) como la generación inteligente de código (mediante reglas de codificación asistida por IA).

## Configuración

### Configuración automática

Cuando ejecutes `npx ultracite init`, se te pedirá que selecciones qué reglas del editor quieres habilitar:

```bash
? Which editor rules do you want to enable (optional)?
  ◯ GitHub Copilot (VSCode)
  ◯ Cursor
  ◯ Windsurf
  # ... etcetera
```

Esto creará el archivo correspondiente en el directorio de tu proyecto.

### Configuración manual

Si deseas configurar las cosas manualmente, crea el archivo correspondiente en el directorio de tu proyecto (por ejemplo `.cursor/rules` para Cursor o `.rules` para Zed). Luego, copia y pega [este contenido](https://github.com/haydenbleasel/ultracite/tree/main/.cursor/rules/ultracite.mdc) en el archivo.

## Ventajas

Usar las reglas de editor de Ultracite ofrece varias ventajas:

- **Consistencia**: Todo el código generado por IA sigue las mismas convenciones
- **Calidad**: Las reglas imponen las mejores prácticas y previenen errores comunes
- **Accesibilidad**: Las directrices integradas de accesibilidad aseguran código inclusivo
- **Rendimiento**: Las reglas promueven patrones eficientes y evitan antipatrones
- **Seguridad**: Las directrices ayudan a prevenir vulnerabilidades de seguridad comunes

## Personalización

Aunque las reglas por defecto son exhaustivas, puedes personalizarlas para tus necesidades específicas:

1. **Modificar reglas existentes**: Edita los archivos de reglas directamente
2. **Agregar reglas específicas del proyecto**: Añade directrices adicionales
3. **Deshabilitar reglas**: Elimina o comenta las reglas que no apliquen a tu proyecto
