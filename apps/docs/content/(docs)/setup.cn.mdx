title: 安装
description: 如何在你的项目中安装 Ultracite。
---

Ultracite 可以通过所有主流包管理器安装。你需要 Node.js v14.18+（建议使用 Node 18+）以及包含 `package.json` 文件的项目。

## 安装

开始的最简方法是运行 Ultracite 的 `init` 脚本，它会为你安装必要的包并为你设置配置文件。在你的项目目录中运行：

```package-install
npx ultracite init
```

你可以可选地向 `init` 命令传递以下标志：

| Flag                | Description                                      | Options                                                                 |
|---------------------|--------------------------------------------------|-------------------------------------------------------------------------|
| `--pm`              | 要使用的包管理器。                                | `pnpm`, `bun`, `yarn`, `npm`                                            |
| `--frameworks`      | 要配置的框架。                                    | 逗号分隔：`react`, `next`, `solid`, `vue`, `qwik`, `angular`, `remix`, `svelte` |
| `--editors`         | 要配置的编辑器。                                  | 逗号分隔：`vscode`, `zed`                                                |
| `--agents`          | 要配置的 AI 代理。                                | 逗号分隔：`vscode-copilot`, `cursor`, `windsurf`, `zed`, `claude`, `codex`, `kiro`, `cline`, `amp`, `aider`, `firebase-studio`, `open-hands`, `gemini-cli`, `junie`, `augmentcode`, `kilo-code`, `goose`, `roo-code`, `warp`, `droid` |
| `--hooks`           | 要配置的代理钩子。                                | 逗号分隔：`cursor`, `claude`                                             |
| `--integrations`    | 要设置的额外集成。                                | 逗号分隔：`husky`, `lefthook`, `lint-staged`                             |
| `--migrate`         | 要移除的迁移工具。                                | 逗号分隔：`eslint`, `prettier`（移除依赖、配置文件和编辑器设置）              |
| `--skip-install`    | 跳过安装依赖项。                                  | （不需要值）                                                              |
| `--quiet`           | 抑制交互提示和可视输出。                          | （不需要值，在 CI 中会自动启用）                                          |

### 以编程方式使用

对于需要以编程方式调用 Ultracite（例如脚手架 CLI）的工具，可以使用 `--quiet` 标志来抑制所有交互提示和可视输出：

```bash title="Terminal"
npx ultracite init --quiet --pm npm
```

当环境变量 `CI` 被设置为 `true` 或 `1` 时，`--quiet` 标志会自动启用，使其适用于 CI/CD 管道：

```bash title="Terminal"
CI=true npx ultracite init --pm npm
```

在静默模式下，Ultracite 将会：
- 跳过所有交互式提示
- 对所有选项使用合理的默认值
- 抑制 ASCII 艺术、加载指示器和日志消息
- 仅创建核心 Biome 配置
- 在成功时以代码 0 干净退出，失败时以代码 1 退出

## 扩展

如果你使用 Visual Studio Code，请安装官方的 [Biome 扩展](https://marketplace.visualstudio.com/items?itemName=biomejs.biome)，它允许在保存时进行格式化/修复：

```bash title="Terminal"
code --install-extension biomejs.biome
```

此外，如果你使用 Tailwind CSS，建议安装 [Tailwind CSS IntelliSense 扩展](https://marketplace.visualstudio.com/items?itemName=bradlc.vscode-tailwindcss) 以获得类排序和提示：

```bash title="Terminal"
code --install-extension bradlc.vscode-tailwindcss
```
