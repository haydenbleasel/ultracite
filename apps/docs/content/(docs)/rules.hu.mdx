Ultracite zökkenőmentes integrációt biztosít az AI kódolási asszisztensekkel azáltal, hogy automatikusan generál szerkesztőspecifikus szabályfájlokat. Ezek a szabályok segítik az AI kódolási asszisztenseket abban, hogy jobb, következetesebb kódot írjanak, amely megfelel a bevett gyakorlatoknak.

A szerkesztői szabályok a Biome lintelésével és formázásával együtt működnek. Míg a Biome a kód formázását végzi és elkapja a szintaxis hibákat, a szerkesztői szabályok arra irányítják az AI kódolási asszisztenseket, hogy már a kezdetektől jobb kódot generáljanak, így kevesebb javításra lesz szükség később.

Ez a kettős megközelítés biztosítja az automatizált kódminőséget (a Biome-on keresztül) és az intelligens kódgenerálást (az AI kódolási szabályokon keresztül).

## Beállítás

### Automatikus beállítás

When you run `npx ultracite init`, you'll be prompted to select which editor rules you want to enable:

```bash
? Which editor rules do you want to enable (optional)?
  ◯ GitHub Copilot (VSCode)
  ◯ Cursor
  ◯ Windsurf
  # ... etcetera
```

This will create the relevant file in your project directory.

### Kézi beállítás

Ha kézzel szeretné beállítani a dolgokat, hozza létre a megfelelő fájlt a projekt könyvtárában (például `.cursor/rules` Cursor esetén vagy `.rules` Zed esetén). Ezután másolja és illessze be [ezt a tartalmat](https://github.com/haydenbleasel/ultracite/tree/main/.cursor/rules/ultracite.mdc) a fájlba.

## Előnyök

Az Ultracite szerkesztői szabályok használata több előnnyel jár:

- **Konzisztencia**: Minden AI által generált kód ugyanazokat a konvenciókat követi
- **Minőség**: A szabályok érvényesítik a bevett gyakorlatokat és megelőzik a gyakori hibákat
- **Hozzáférhetőség**: A beépített akadálymentességi irányelvek inkluzív kódot biztosítanak
- **Teljesítmény**: A szabályok elősegítik a hatékony mintákat és elkerülik az anti-patterneket
- **Biztonság**: Az irányelvek segítenek megelőzni a gyakori biztonsági sebezhetőségeket

## Testreszabás

Bár az alapértelmezett szabályok átfogóak, testreszabhatja őket a saját igényei szerint:

1. **Módosítsa a meglévő szabályokat**: Szerkessze közvetlenül a szabályfájlokat
2. **Projekt-specifikus szabályok hozzáadása**: Fűzzön hozzá további irányelveket
3. **Szabályok letiltása**: Távolítson el vagy kommenteljen ki olyan szabályokat, amelyek nem vonatkoznak a projektjére
