Ultracite zökkenőmentes integrációt biztosít AI kódolási asszisztensekkel azáltal, hogy automatikusan generál szerkesztőspecifikus szabályfájlokat. Ezek a szabályok segítik az AI kódolási asszisztenseket abban, hogy jobb, következetesebb kódot írjanak, amely a legjobb gyakorlatokat követi.

A szerkesztői szabályok a Biome lintelésével és formázásával együtt működnek. Míg a Biome a kód formázását végzi és észleli a szintaktikai hibákat, addig a szerkesztői szabályok arra irányítják az AI kódolási asszisztenseket, hogy már a kezdetektől jobb kódot generáljanak, csökkentve ezzel a későbbi javítások szükségességét.

Ez a kettős megközelítés biztosítja mind az automatizált kódminőséget (Biome révén), mind az intelligens kódgenerálást (AI kódolási szabályok révén).

## Beállítás

### Automatikus beállítás

When you run `npx ultracite init`, you'll be prompted to select which editor rules you want to enable:

```bash
? Which editor rules do you want to enable (optional)?
  ◯ GitHub Copilot (VSCode)
  ◯ Cursor
  ◯ Windsurf
  # ... etcetera
```

This will create the relevant file in your project directory.

### Kézi beállítás

Ha kézzel szeretnéd beállítani, hozd létre a megfelelő fájlt a projekt könyvtárában (például `.cursor/rules` a Cursorhoz vagy `.rules` a Zedhez). Ezután másold és illeszd be [ezt a tartalmat](https://github.com/haydenbleasel/ultracite/tree/main/.cursor/rules/ultracite.mdc) a fájlba.

## Előnyök

Az Ultracite szerkesztői szabályainak használata több előnnyel jár:

- **Következetesség**: Minden AI által generált kód ugyanazokat a konvenciókat követi
- **Minőség**: A szabályok érvényesítik a legjobb gyakorlatokat és megakadályozzák az általános hibákat
- **Akadálymentesség**: A beépített akadálymentességi irányelvek inkluzív kódot biztosítanak
- **Teljesítmény**: A szabályok hatékony mintákat ösztönöznek és elkerülik az anti-patternöket
- **Biztonság**: Az irányelvek segítenek megelőzni a gyakori biztonsági sebezhetőségeket

## Testreszabás

Bár az alapértelmezett szabályok átfogóak, testreszabhatod őket a konkrét igényeid szerint:

1. **Módosítsd a meglévő szabályokat**: Szerkeszd közvetlenül a szabályfájlokat
2. **Adj hozzá projektspecifikus szabályokat**: Fűzz hozzá további irányelveket
3. **Tilts szabályokat**: Távolítsd el vagy kommenteld ki azokat a szabályokat, amelyek nem alkalmazhatók a projektedre
