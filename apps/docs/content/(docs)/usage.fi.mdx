## IDE-integraatio

Ultracite toimii parhaiten, kun se on integroituna editoriisi, jolloin saat välitöntä palautetta kirjoittaessasi koodia. Se on suunniteltu ajettavaksi automaattisesti tallennuksen yhteydessä. Kokeillaksesi avaa projekti IDE:ssäsi. Muokatessasi tiedostoja näet muotoilun astuvan voimaan tallennuksen yhteydessä. Yritä lisätä pieni koodityylivirhe (kuten ylimääräinen puolipiste tai väärin sisennetty rivi) ja paina Tallenna – Ultracite (Biome) muotoilee tiedoston välittömästi uudelleen.

Jos syntyy yleinen lint-ongelma (kuten käyttämätön muuttuja), saatat nähdä aaltoilevan alleviivauksen tai varoituksen Problems-paneelissa; tallennuksen yhteydessä Ultracite yrittää korjata sen, jos se on automaattisesti korjattavissa (esimerkiksi poistaa käyttämättömän importin) tai jättää muuten varoituksen korjattavaksi.

### Välitön muotoilu

Joka kerta kun tallennat tiedoston, Biome (Ultraciten konfiguraation kanssa) muotoilee tiedoston. Sinun ei tarvitse ajaa erillistä Prettieriä tai huolehtia muotoilusta – se hoituu automaattisesti.

Kiitos `codeActionsOnSave`-asetuksille, VS Code soveltaa myös Biomen “korjaa kaikki” -toimintoa tallennuksen yhteydessä. Tämä tarkoittaa, että kaikki lint-säännöt, joilla on automaattinen korjaus (kuten `!=` muuttaminen `!==`:ksi, puuttuvien sulkujen lisääminen, import-järjestyksen korjaaminen jne.), sovelletaan. Se on kuin ajaisit `--fix` ESLintin kanssa automaattisesti.

Olemme jopa ottaneet käyttöön `formatOnPaste` – joten jos liität koodia muualta, se muotoillaan välittömästi tyylisi mukaiseksi.

Keskeinen etu tässä on välitön palaute. Kirjoitat koodia VS Codessa, ja Ultracite pitää sen jatkuvasti siistinä. Ajan myötä huomaat käyttäväsi vähän tai ei lainkaan aikaa lint-virheiden korjaamiseen – Ultracite joko korjaa ne puolestasi tai ilmoittaa niistä ajoissa. Tämä johtaa sujuvaan kehitystyönkulkuun, jossa koodityyli ja pienet ongelmat eivät kasaannu.

### Ongelmat-paneelin integrointi

Kaikki ongelmat, jotka vaativat huomiotasi, näkyvät Problems-paneelissa. Esimerkiksi jos Ultracitella on sääntö, joka kieltää jonkin toiminnon, tai se löytää virheen jota se ei voi korjata (esimerkiksi käytit poistettua APIa tai unohdit käsitellä Promisen hylkäyksen), se listaa sen virheenä tai varoituksena. Voit klikata merkintää siirtyäksesi kohtaan koodissa.

### Code lenses / Pikakorjaukset

Monissa tapauksissa Biomen VS Code -laajennus tarjoaa pikakorjausehdotuksia. Jos näet keltaisen hehkulampun tai ehdotuksen ponnahdusikkunan, voit soveltaa korjauksia myös manuaalisesti. Esimerkiksi, jos import on käyttämätön, saatat saada pikakorjauksen sen poistamiseksi (jos sitä ei jo poistettu tallennuksen yhteydessä).

## CLI-käyttö

Ultracite sisältää kätevän komentorivityökalun (kutsutaan `ultracite`-komennolla, kun se on asennettu). Tässä yleisiä tapoja käyttää sitä.

### Koodin tarkistus

`check`-komento käärii Biomen `check`-komennon, joka suorittaa lintin ilman tiedostojen korjaamista.

```bash title="Terminal"
npx ultracite check
```

Voit käyttää `--diagnostic-level`-suodatinta näyttääksesi vain virheet tai varoitukset:

```bash title="Terminal"
# Show only errors
npx ultracite check --diagnostic-level error

# Show warnings and above (errors)
npx ultracite check --diagnostic-level warn
```

### Koodin korjaus

`fix`-komento käärii Biomen `check --write` -komennon, joka suorittaa lintin ja korjaa tiedostot.

```bash title="Terminal"
npx ultracite fix
```

Voit myös soveltaa ei-turvallisia korjauksia ajamalla:

```bash title="Terminal"
npx ultracite fix --unsafe
```

### Asennuksen tarkistus

`doctor`-komento tarkistaa asetuksesi ongelmien varalta ja antaa suosituksia. Tätä on hyödyllistä ajaa Ultraciten asentamisen jälkeen varmistaaksesi, että kaikki on konfiguroitu oikein.

```bash title="Terminal"
npx ultracite doctor
```
