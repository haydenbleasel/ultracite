title: Migrera från Biome
description: Hur du migrerar ditt projekt från fristående Biome till Ultracite.
---

Om du redan använder Biome och vill byta till Ultracites förkonfigurerade setup, hjälper denna guide dig att migrera samtidigt som du bevarar din befintliga konfiguration.

## Varför migrera till Ultracite?

- **Ingen konfiguration**: Ultracite tillhandahåller en färdigkonfigurerad förinställning av regler
- **Redigerarintegration**: Inbyggt stöd för AI-drivna redigerare (Cursor, Windsurf, GitHub Copilot)
- **Konsekvent arbetsflöde**: Standardiserad setup över projekt och team
- **Ytterligare verktyg**: Valfria Git hooks, integration med lint-staged
- **Underhållna regler**: Regelbundna uppdateringar med nya bästa praxis

## Innan du börjar

Se till att du har:

- Ett befintligt projekt som använder Biome
- Node.js v14.18+ (Node 18+ rekommenderas)
- En `biome.json` eller `biome.jsonc`-fil

## Migreringsalternativ

### Alternativ 1: Snabb migrering (Rekommenderas)

Det snabbaste sättet är att köra [automatiskt installationsskript](/setup#automatic-installation).

Detta kommer att:

- Installera Ultracite som ett beroende
- Slå samman din befintliga `biome.json` / `biome.jsonc` med Ultracites förinställning
- Slå samman din befintliga `.vscode/settings.json` med Ultracites förinställning
- Aktivera `strictNullChecks` i din `tsconfig.json`-fil (eller skapa en om den inte finns)
- Bevara dina anpassade regler och inställningar
- Konfigurera redigerarintegrationer

Efter uppgraderingen kanske du vill granska din `biome.json` / `biome.jsonc`-fil för att ta bort eventuella överskrivningar. Starta dessutom om din redigerare för att säkerställa att den nya konfigurationen tillämpas.

### Alternativ 2: Manuell migrering

Om du föredrar mer kontroll över processen, följ stegen för [Manuell migrering](/setup#manual-installation) genom att:

- Installera Ultracite och Biome manuellt
- Uppdatera din `biome.json`-fil för att ärva från Ultracite
- Skapa eller uppdatera `.vscode/settings.json`-filen för att aktivera Biome-tillägget
- Aktivera `strictNullChecks` i din `tsconfig.json`-fil (eller skapa en om den inte finns)

Efter migreringen, starta om din redigerare för att säkerställa att den nya konfigurationen tillämpas.
