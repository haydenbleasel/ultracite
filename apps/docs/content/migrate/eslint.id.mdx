Jika Anda menggunakan ESLint dan ingin beralih ke pengaturan pra-konfigurasi Ultracite, panduan ini akan membantu Anda memigrasikan sambil mempertahankan standar kualitas kode.

## Mengapa Migrasi ke Ultracite?

- **Kecepatan Kilat**: Biome (fondasi Ultracite) berbasis Rust, jauh lebih cepat daripada ESLint
- **Tanpa Konfigurasi**: Ratusan aturan pra-konfigurasi dibandingkan dengan pengaturan ESLint manual
- **Alat Gabungan**: Menggantikan baik ESLint maupun Prettier dengan satu alat tunggal
- **Integrasi AI**: Dukungan bawaan untuk editor bertenaga AI (Cursor, Windsurf, GitHub Copilot)
- **Keamanan Tipe**: Aturan TypeScript yang komprehensif dengan strict null checks
- **Standar Modern**: Praktik terbaik JavaScript/TypeScript terbaru

## Sebelum Anda Mulai

Pastikan Anda memiliki:

- Proyek yang sudah ada yang menggunakan ESLint
- Node.js v14.18+ (disarankan Node 18+)
- File `package.json`

## Pengaturan Otomatis

Jalankan [skrip pengaturan otomatis](/setup#automatic-installation).

```package-install
npx ultracite init
```

Ini akan:

- Menginstal Ultracite sebagai dependensi
- Menggabungkan `biome.json` / `biome.jsonc` Anda yang ada dengan preset Ultracite
- Menggabungkan `.vscode/settings.json` Anda yang ada dengan preset Ultracite
- Mengaktifkan `strictNullChecks` di file `tsconfig.json` Anda (atau membuatnya jika belum ada)
- Mempertahankan aturan dan pengaturan kustom Anda
- Menyiapkan integrasi editor

Pastikan Anda memilih untuk menghapus konfigurasi `ESLint` yang ada. Ini akan:

- Menghapus ESLint dan dependensi terkait
- Menghapus semua file konfigurasi ESLint
- Menghapus perintah terkait ESLint dari file `.vscode/settings.json` Anda

Setelah peningkatan, Anda mungkin ingin meninjau file `biome.json` / `biome.jsonc` Anda untuk menghapus override apa pun. Selain itu, restart editor Anda untuk memastikan konfigurasi baru diterapkan.
