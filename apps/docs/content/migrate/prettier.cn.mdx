---
title: 从 Prettier 迁移
description: 如何将您的项目从 Prettier 迁移到 Ultracite。
---

如果您正在使用 Prettier 并希望切换到 Ultracite 的预配置设置，本指南将帮助您在保持代码质量标准的同时进行迁移。

## 为什么要迁移到 Ultracite？

- **闪电般的速度**: Biome（Ultracite 的基础）基于 Rust，比 Prettier 快得多
- **零配置**: 数百条预配置规则，而不是手动设置 Prettier
- **组合工具**: 用单一工具替代 ESLint 和 Prettier
- **AI 集成**: 内置对 AI 驱动编辑器的支持（Cursor、Windsurf、GitHub Copilot）
- **类型安全**: 全面的 TypeScript 规则，包含 strict null checks
- **现代标准**: 最新的 JavaScript/TypeScript 最佳实践

## 在开始之前

确保您拥有：

- 一个当前使用 Prettier 的现有项目
- Node.js v14.18+（推荐 Node 18+）
- 一个 `package.json` 文件

## 自动设置

运行 [自动安装脚本](/setup#automatic-installation)。

```package-install
npx ultracite init
```

这将：

- 安装 Ultracite 作为依赖项
- 将现有的 `biome.json` / `biome.jsonc` 与 Ultracite 的预设合并
- 将现有的 `.vscode/settings.json` 与 Ultracite 的预设合并
- 在您的 `tsconfig.json` 文件中启用 `strictNullChecks`（如果不存在则创建一个）
- 保留您自定义的规则和设置
- 设置编辑器集成

确保您选择移除现有的 `Prettier` 配置。这将：

- 移除 Prettier 以及任何相关依赖项
- 移除所有 Prettier 配置文件
- 从您的 `.vscode/settings.json` 文件中移除所有与 Prettier 相关的命令

升级完成后，您可能需要检查您的 `biome.json` / `biome.jsonc` 文件以移除任何覆盖项。此外，重启编辑器以确保新的配置已生效。
