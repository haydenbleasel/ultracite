---
title: Prettier からの移行
description: Prettier から Ultracite へプロジェクトを移行する方法。
---

もし Prettier を使用していて Ultracite の事前設定済みセットアップに切り替えたい場合、このガイドはコード品質基準を維持しながら移行する手順を案内します。

## なぜ Ultracite に移行するのか？

- **高速**: Biome（Ultracite の基盤）は Rust ベースで、Prettier よりはるかに高速です
- **設定不要**: 手動での Prettier 設定に対して、何百もの事前設定済みルールを提供
- **統合ツール**: ESLint と Prettier の両方を単一のツールで置き換えます
- **AI 統合**: AI 対応エディタ（Cursor、Windsurf、GitHub Copilot）への組み込みサポート
- **型の安全性**: 厳格な null チェックを含む包括的な TypeScript ルール
- **最新の標準**: 最新の JavaScript/TypeScript ベストプラクティス

## 開始前に

以下を確認してください:

- Prettier を使用している既存のプロジェクト
- Node.js v14.18+（Node 18+ を推奨）
- `package.json` ファイル

## 自動セットアップ

[自動セットアップスクリプト](/setup#automatic-installation) を実行してください。

```package-install
npx ultracite init
```

これにより:

- Ultracite を依存関係としてインストールします
- 既存の `biome.json` / `biome.jsonc` を Ultracite のプリセットとマージします
- 既存の `.vscode/settings.json` を Ultracite のプリセットとマージします
- `tsconfig.json` ファイルで `strictNullChecks` を有効にします（存在しない場合は作成します）
- カスタムルールと設定を保持します
- エディタ統合を設定します

既存の `Prettier` 設定を削除することを選択していることを確認してください。これにより:

- Prettier と関連する依存関係が削除されます
- すべての Prettier 設定ファイルが削除されます
- `.vscode/settings.json` ファイルから Prettier に関連するコマンドが削除されます

アップグレード後、オーバーライドを削除するために `biome.json` / `biome.jsonc` ファイルを確認することをおすすめします。さらに、新しい設定が適用されるようにエディタを再起動してください。
