title: Migrera från Prettier
description: Hur du migrerar ditt projekt från Prettier till Ultracite.
---

Om du använder Prettier och vill byta till Ultracites förkonfigurerade setup hjälper den här guiden dig att migrera samtidigt som kodkvalitetsstandarderna upprätthålls.

## Varför migrera till Ultracite?

- **Blixtrande snabbhet**: Biome (Ultracites grund) är baserat på Rust, mycket snabbare än Prettier
- **Ingen konfiguration**: Hundratals förkonfigurerade regler jämfört med manuell Prettier-inställning
- **Kombinerat verktyg**: Ersätter både ESLint och Prettier med ett enda verktyg
- **AI-integration**: Inbyggt stöd för AI-drivna redigerare (Cursor, Windsurf, GitHub Copilot)
- **Typsäkerhet**: Omfattande TypeScript-regler med strikta nullkontroller
- **Moderna standarder**: Senaste bästa praxis för JavaScript/TypeScript

## Innan du börjar

Se till att du har:

- Ett befintligt projekt som använder Prettier
- Node.js v14.18+ (Node 18+ rekommenderas)
- En `package.json`-fil

## Automatisk installation

Kör [det automatiska installationsskriptet](/setup#automatic-installation).

```package-install
npx ultracite init
```

Detta kommer att:

- Installera Ultracite som ett beroende
- Slå samman din befintliga `biome.json` / `biome.jsonc` med Ultracites förinställning
- Slå samman din befintliga `.vscode/settings.json` med Ultracites förinställning
- Aktivera `strictNullChecks` i din `tsconfig.json`-fil (eller skapa en om den inte finns)
- Behålla dina anpassade regler och inställningar
- Konfigurera integrationer i redigeraren

Se till att du väljer att ta bort den befintliga `Prettier`-konfigurationen. Detta kommer att:

- Ta bort Prettier och alla relaterade beroenden
- Ta bort eventuella Prettier-konfigurationsfiler
- Ta bort eventuella Prettier-relaterade kommandon från din `.vscode/settings.json`-fil

Efter uppgraderingen kan du vilja granska din `biome.json` / `biome.jsonc`-fil för att ta bort eventuella överskrivningar. Starta dessutom om din editor för att säkerställa att den nya konfigurationen tillämpas.
