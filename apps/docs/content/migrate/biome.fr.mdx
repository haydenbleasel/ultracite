---
title: Migrer depuis Biome
description: Comment migrer votre projet de Biome autonome vers Ultracite.
---

Si vous utilisez déjà Biome et souhaitez passer au préréglage préconfiguré d'Ultracite, ce guide vous aidera à migrer tout en préservant votre configuration existante.

## Pourquoi migrer vers Ultracite ?

- **Aucune configuration**: Ultracite fournit un préréglage de règles hautement configuré
- **Intégration éditeur**: Prise en charge intégrée des éditeurs alimentés par l'IA (Cursor, Windsurf, GitHub Copilot)
- **Flux de travail cohérent**: Configuration standardisée entre projets et équipes
- **Outils supplémentaires**: Hooks Git optionnels, intégration lint-staged
- **Règles maintenues**: Mises à jour régulières avec de nouvelles bonnes pratiques

## Avant de commencer

Assurez-vous d'avoir :

- Un projet existant utilisant Biome
- Node.js v14.18+ (Node 18+ recommandé)
- Un fichier `biome.json` ou `biome.jsonc`

## Options de migration

### Option 1 : Migration rapide (recommandée)

La façon la plus rapide est d'exécuter le [script d'installation automatique](/setup#automatic-installation).

Cela permettra de :

- Installer Ultracite en tant que dépendance
- Fusionner votre `biome.json` / `biome.jsonc` existant avec le préréglage d'Ultracite
- Fusionner votre `.vscode/settings.json` existant avec le préréglage d'Ultracite
- Activer `strictNullChecks` dans votre fichier `tsconfig.json` (ou en créer un s'il n'existe pas)
- Conserver vos règles et paramètres personnalisés
- Configurer les intégrations d'éditeur

Après la mise à niveau, vous pouvez vouloir revoir votre fichier `biome.json` / `biome.jsonc` pour supprimer toute surcharge. De plus, redémarrez votre éditeur pour vous assurer que la nouvelle configuration est appliquée.

### Option 2 : Migration manuelle

Si vous préférez un contrôle plus précis du processus, suivez les étapes de la [Migration manuelle](/setup#manual-installation) en :

- Installant Ultracite et Biome manuellement
- Mettant à jour votre fichier `biome.json` pour étendre Ultracite
- Créant ou mettant à jour un fichier `.vscode/settings.json` pour activer l'extension Biome
- Activant `strictNullChecks` dans votre fichier `tsconfig.json` (ou en créant un s'il n'existe pas)

Après la migration, redémarrez votre éditeur pour vous assurer que la nouvelle configuration est appliquée.
