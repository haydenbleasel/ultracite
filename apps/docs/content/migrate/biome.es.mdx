---
title: Migrar desde Biome
description: Cómo migrar tu proyecto desde Biome independiente a Ultracite.
---

Si ya estás usando Biome y quieres cambiar a la configuración preconfigurada de Ultracite, esta guía te ayudará a migrar preservando tu configuración existente.

## ¿Por qué migrar a Ultracite?

- **Cero configuración**: Ultracite proporciona un preajuste altamente configurado de reglas
- **Integración con editores**: Soporte integrado para editores potenciados por IA (Cursor, Windsurf, GitHub Copilot)
- **Flujo de trabajo consistente**: Configuración estandarizada entre proyectos y equipos
- **Herramientas adicionales**: Hooks de Git opcionales, integración con lint-staged
- **Reglas mantenidas**: Actualizaciones periódicas con nuevas mejores prácticas

## Antes de comenzar

Asegúrate de tener:

- Un proyecto existente que use Biome
- Node.js v14.18+ (se recomienda Node 18+)
- Un archivo `biome.json` o `biome.jsonc`

## Opciones de migración

### Opción 1: Migración rápida (Recomendada)

La forma más rápida es ejecutar el [script de instalación automática](/setup#automatic-installation).

Esto hará:

- Instalar Ultracite como una dependencia
- Fusionar tu `biome.json` / `biome.jsonc` existente con la preconfiguración de Ultracite
- Fusionar tu `.vscode/settings.json` existente con la preconfiguración de Ultracite
- Habilitar `strictNullChecks` en tu archivo `tsconfig.json` (o crear uno si no existe)
- Conservar tus reglas y configuraciones personalizadas
- Configurar las integraciones con el editor

Tras la actualización, puede que desees revisar tu archivo `biome.json` / `biome.jsonc` para eliminar cualquier anulación. Además, reinicia tu editor para asegurarte de que se aplique la nueva configuración.

### Opción 2: Migración manual

Si prefieres tener más control sobre el proceso, sigue los pasos de la [Migración manual](/setup#manual-installation) realizando:

- Instalar Ultracite y Biome manualmente
- Actualizar tu archivo `biome.json` para extender Ultracite
- Crear o actualizar un archivo `.vscode/settings.json` para habilitar la extensión de Biome
- Habilitar `strictNullChecks` en tu archivo `tsconfig.json` (o crear uno si no existe)

Después de la migración, reinicia tu editor para asegurarte de que se aplique la nueva configuración.
