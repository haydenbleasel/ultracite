---

Ultracite v6 introduit des [préréglages spécifiques au framework](/configuration#framework-specific-configurations) qui vous donnent plus de contrôle sur les règles qui s'appliquent à votre projet. **Cette mise à niveau n'entraîne pas de rupture** pour les utilisateurs Next.js/React.

## Ce qui a changé

v6 introduit des [préréglages spécifiques au framework](/configuration#framework-specific-configurations) :

- [`ultracite/core`](/preset/core) - Règles de base pour JavaScript/TypeScript
- [`ultracite/react`](/preset/react) - Règles spécifiques à React
- [`ultracite/next`](/preset/next) - Règles spécifiques à Next.js
- [`ultracite/solid`](/preset/solid) - Règles pour Solid.js
- [`ultracite/vue`](/preset/vue) - Règles pour Vue.js
- [`ultracite/svelte`](/preset/svelte) - Règles pour Svelte
- [`ultracite/qwik`](/preset/qwik) - Règles pour Qwik
- [`ultracite/angular`](/preset/angular) - Parseur HTML pour Angular
- [`ultracite/remix`](/preset/remix) - Prise en charge du routage basé sur les fichiers

## Migration

### Projets Next.js/React

Mettez à jour votre `biome.jsonc` pour étendre les préréglages appropriés :

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/react", "ultracite/next"]
}
```

### Projets non-React

Passez au préréglage core si vous n'utilisez pas React :

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core"]
}
```

### Autres frameworks

Utilisez le préréglage approprié pour votre framework :

```jsonc title="biome.jsonc"
{
  // Solid.js
  "extends": ["ultracite/core", "ultracite/solid"]

  // Vue.js
  "extends": ["ultracite/core", "ultracite/vue"]

  // Svelte
  "extends": ["ultracite/core", "ultracite/svelte"]

  // Qwik
  "extends": ["ultracite/core", "ultracite/qwik"]

  // Angular
  "extends": ["ultracite/core", "ultracite/angular"]
}
```

### Projets multi-framework

Combinez les préréglages selon les besoins :

```jsonc title="biome.jsonc"
{
  "extends": ["ultracite/core", "ultracite/react", "ultracite/solid"]
}
```
